## 2. フレームワーク別特徴

### ⚡ Next.js（`next/`・`next-*`）

```
得意なこと:
  - SSR / SSG / ISR を柔軟に切り替えられる
  - App Router で Server Components / Server Actions が使える
  - Vercel への 1 クリックデプロイ
  - エコシステムが最大規模

苦手なこと:
  - 設定・概念が多くて学習コストが高い
  - React の知識が前提

採用規模: 🏢🏢🏢🏢🏢（業界最大規模）
```

### 🟢 Nuxt（`nuxt/`）

```
得意なこと:
  - Vue エコシステムとの親和性が高い
  - Nitro サーバーエンジンで高パフォーマンス
  - auto-imports で開発体験が優れる
  - ファイルベースルーティングが直感的

苦手なこと:
  - Next.js より情報が少ない（日本語リソースは多め）
  - Vue の知識が前提

採用規模: 🏢🏢🏢（日本では人気）
```

### 🎸 Remix（`remix/`）

```
得意なこと:
  - Web 標準（Form / fetch）に忠実な設計
  - ネストされたルーティングとデータ取得が強力
  - エラーバウンダリが使いやすい
  - Prisma との相性が良い

苦手なこと:
  - 独自の概念（loader / action）の学習が必要
  - Next.js より情報が少ない

採用規模: 🏢🏢（成長中）
```

### 🍀 SvelteKit（`sveltekit/`）

```
得意なこと:
  - Svelte の構文がシンプルで学習コストが低い
  - バンドルサイズが小さく高速
  - Svelte 5 の Runes API が革新的
  - Prisma との相性が良い

苦手なこと:
  - React/Vue に比べてエコシステムが小さい
  - 企業採用事例がまだ少ない

採用規模: 🏢🏢（注目度急上昇）
```

### ⚛️ React + Vite（`react/`）

```
得意なこと:
  - SPA（シングルページアプリケーション）に最適
  - 最も広いエコシステム
  - 採用実績が最大
  - ライブラリ選択の自由度が高い

苦手なこと:
  - SSR / SEO はフレームワーク追加が必要
  - ルーティングは別途 react-router 等が必要

採用規模: 🏢🏢🏢🏢🏢（業界最大規模）
```

### 💚 Vue 3 + Vite（`vue/`）

```
得意なこと:
  - 学習コストが低く、HTML/CSS の知識で入りやすい
  - Options API / Composition API どちらも使える
  - 日本語ドキュメントが充実

苦手なこと:
  - SSR は Nuxt が必要
  - React より採用規模がやや小さい

採用規模: 🏢🏢🏢（日本では人気）
```

### 🚀 Astro（`astro/`）

```
得意なこと:
  - コンテンツ重視サイトに最適（ブログ・LP・ドキュメント）
  - 複数フレームワークのコンポーネントを混在可能
  - デフォルトで JavaScript をほぼ出力しない（超高速）

苦手なこと:
  - 動的な SPA には向かない
  - インタラクティブな機能は Islands Architecture が必要

採用規模: 🏢🏢（コンテンツサイト特化）
```

## 4. ユースケース別推奨

### 📰 コンテンツサイト・ブログ・LP

```
第 1 推奨: Astro
  → JavaScript をほぼ出力しないため超高速。SEO に最適。

第 2 推奨: Next.js（SSG モード）
  → 既存の React 知識が活かせる。
```

### 🛒 EC サイト・一般的な Web アプリ

```
第 1 推奨: Next.js
  → App Router + Server Actions でフルスタック開発が可能。
    Supabase / Prisma との相性◎。

第 2 推奨: Nuxt（Vue が好みなら）
  → 日本語リソースが充実。Vue エコシステムが使える。
```

### 📊 管理画面・ダッシュボード（SPA でよい場合）

```
第 1 推奨: React + Vite
  → シンプルな SPA。UI ライブラリ（shadcn/ui 等）が豊富。

第 2 推奨: Vue 3 + Vite
  → 学習コストが低い。
```

### 🎯 フルスタック（BFF なしでバックエンドも書きたい）

```
第 1 推奨: Next.js（App Router + Server Actions）
  → フロント・バックを 1 プロジェクトで完結。

第 2 推奨: SvelteKit
  → シンプルな構文。form actions が直感的。
```

### 📱 モバイルアプリ

```
推奨: React Native（`react-native/` テンプレート）
  → iOS / Android を 1 つのコードベースで開発。
```

## 5. 技術選定フローチャート

```
チームに Vue の経験者が多い？
  → YES: Nuxt を選択
  → NO:
     ↓
React を使いたい？
  → YES:
     SSR / SEO が必要？
       → YES: Next.js を選択
       → NO: React + Vite を選択
  → NO:
     ↓
     学習コストを最小化したい？
       → YES: SvelteKit を選択（最もシンプル）
       → NO:
          Web 標準に忠実な設計がよい？
            → YES: Remix を選択
            → NO:
               コンテンツサイト（ブログ・LP）？
                 → YES: Astro を選択
```

## 6. 各テンプレートのセットアップ

### Next.js（`next/`）

```bash
cd next
cp .env.example .env
docker compose up -d
docker compose exec app npx prisma migrate dev
# → http://localhost:3000
```

### Nuxt（`nuxt/`）

```bash
cd nuxt
docker compose up -d
# → http://localhost:3000
```

### Remix（`remix/`）

```bash
cd remix
cp .env.example .env  # .env.example があれば
docker compose up -d
docker compose exec app npx prisma migrate dev
# → http://localhost:3000
```

### SvelteKit（`sveltekit/`）

```bash
cd sveltekit
docker compose up -d
docker compose exec app npx prisma migrate dev
# → http://localhost:5173
```

### React + Vite（`react/`）

```bash
cd react
docker compose up -d
# → http://localhost:5173
```

### Vue 3 + Vite（`vue/`）

```bash
cd vue
docker compose up -d
# → http://localhost:5173
```
