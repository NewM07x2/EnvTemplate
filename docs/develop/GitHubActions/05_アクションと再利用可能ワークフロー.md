## 1. アクション（Action）とは

**アクション** は、GitHub Actions で再利用可能な処理の単位です。

```
ステップ内で使用:

steps:
  - uses: actions/checkout@v4
       ↑
    これがアクション
```

**アクションの3つの種類**:

| 種類 | 実装言語 | 用途 | 例 |
|------|---------|------|-----|
| **JavaScript アクション** | JavaScript/TypeScript | 軽量な処理 | checkout, setup-node |
| **Docker アクション** | 任意の言語 | 複雑な処理 | コード分析、ビルド |
| **複合アクション** | YAML | ステップの再利用 | 複数ステップをまとめる |

## 2. GitHub 公式アクション

GitHub が提供する公式アクションは、`actions/` というプレフィックスが付いています。

### 2.1 checkout

ソースコードをランナーにチェックアウトします。**ほぼすべてのワークフローで最初に使用**。

```yaml
steps:
  - name: Checkout code
    uses: actions/checkout@v4
    with:
      # ブランチを指定
      ref: main
      
      # フェッチする履歴数（1 = 最新のコミットのみ）
      fetch-depth: 1
      
      # サブモジュールも取得
      submodules: true
```

### 2.2 setup-node

Node.js をセットアップしてキャッシュします。

```yaml
steps:
  - uses: actions/checkout@v4
  
  - name: Setup Node.js
    uses: actions/setup-node@v4
    with:
      # Node.js バージョン
      node-version: '18'
      
      # または複数バージョン
      # node-version: ['16', '18', '20']
      
      # npm キャッシュを有効化
      cache: 'npm'
      
      # yarn の場合
      # cache: 'yarn'
```

### 2.3 upload-artifact

ビルド成果物や一時ファイルをアップロードして保存します。

```yaml
steps:
  - run: npm run build
  
  - name: Upload build artifacts
    uses: actions/upload-artifact@v3
    with:
      # アーティファクト名
      name: build-output
      
      # アップロード対象のパス
      path: dist/
      
      # 保存期間（日数、デフォルト 90 日）
      retention-days: 30
```

### 2.4 download-artifact

前のジョブやワークフローでアップロードしたアーティファクトをダウンロード。

```yaml
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - run: npm run build
      - uses: actions/upload-artifact@v3
        with:
          name: build-output
          path: dist/
  
  deploy:
    needs: build
    runs-on: ubuntu-latest
    steps:
      - name: Download artifacts
        uses: actions/download-artifact@v3
        with:
          name: build-output
          path: ./dist
      
      - run: npm run deploy
```

### 2.5 github-script

GitHub API を JavaScript で直接操作します。

```yaml
steps:
  - name: Comment on PR
    uses: actions/github-script@v7
    with:
      script: |
        github.rest.issues.createComment({
          issue_number: context.issue.number,
          owner: context.repo.owner,
          repo: context.repo.repo,
          body: 'Hello, pull request!'
        })
```

### 2.6 その他の重要なアクション

```yaml
# キャッシュ
- uses: actions/cache@v3
  with:
    path: ~/.npm
    key: ${{ runner.os }}-npm-${{ hashFiles('**/package-lock.json') }}

# 環境変数のセット
- uses: actions/setup-go@v4
  with:
    go-version: '1.20'

# Docker ログイン
- uses: docker/login-action@v2
  with:
    username: ${{ secrets.DOCKER_USERNAME }}
    password: ${{ secrets.DOCKER_PASSWORD }}
```

## 3. Marketplace アクション

### 3.1 Marketplace の検索

GitHub Marketplace には、コミュニティが作成したアクションが数千個あります。

📍 [GitHub Marketplace](https://github.com/marketplace?type=actions)

**検索のコツ**:
- 使用言語やツール名で検索（例: "Python", "Docker", "Slack"）
- 「Stars」数が多いもの
- 最近更新されているもの
- 公式サポートマークが付いているもの

### 3.2 人気の Marketplace アクション

| アクション | 説明 | 使用例 |
|-----------|------|--------|
| **codecov/codecov-action** | カバレッジレポート送信 | テストカバレッジ追跡 |
| **actions/labeler** | PR にラベルを自動付与 | 自動分類 |
| **docker/build-push-action** | Docker イメージビルド | コンテナ化 |
| **softprops/action-gh-release** | GitHub Release 作成 | 自動リリース |
| **slackapi/slack-github-action** | Slack 通知 | 通知 |

### 3.3 アクションの評価と選択

```yaml
# ✅ 公式または信頼できるユーザー作成
uses: actions/checkout@v4
uses: docker/build-push-action@v5
uses: codecov/codecov-action@v3

# ✅ 定期的に更新されている
# GitHub で「Last updated」を確認

# ❌ 古いバージョン（非推奨）
uses: actions/checkout@v1  # 4 が最新

# ❌ バージョン指定なし（危険）
uses: some-action  # 避ける
```

## 4. Docker コンテナアクション

Docker コンテナでアクションを実行する場合：

```yaml
steps:
  - name: Run Docker action
    uses: docker://node:18
    with:
      args: npm test
```

**Docker イメージ指定**:

```yaml
uses: docker://ubuntu:22.04
# または
uses: docker://myregistry.azurecr.io/myimage:latest
```

## 5. カスタムアクション（カスタム実装）

リポジトリ固有の処理は、カスタムアクションとして実装できます。

### 5.1 JavaScript アクション

**必要なファイル**:

```
.github/
├── actions/
│   └── my-action/
│       ├── action.yml        ← アクション定義
│       ├── index.js           ← 処理実装
│       └── package.json       ← Node.js 依存
```

### 5.2 カスタムアクションの構造

ディレクトリを作成：

```powershell
mkdir .github/actions/greet-action
cd .github/actions/greet-action
```

### 5.3 action.yml の定義

`.github/actions/greet-action/action.yml`:

```yaml
name: 'Greet Action'
description: 'Greet someone'
author: 'Your Name'

# 入力パラメータ
inputs:
  name:
    description: 'Name to greet'
    required: true
    default: 'World'

# 出力値
outputs:
  greeting:
    description: 'Greeting message'
    value: ${{ steps.greet.outputs.greeting }}

# 実行設定
runs:
  using: 'node20'
  main: 'index.js'
```

### 5.4 カスタムアクション実装例

`.github/actions/greet-action/index.js`:

```javascript
const core = require('@actions/core');
const github = require('@actions/github');

try {
  // 入力パラメータを取得
  const name = core.getInput('name');
  const greeting = `Hello, ${name}!`;
  
  // 出力を設定
  core.setOutput('greeting', greeting);
  
  // ログ出力
  core.info(`Greeting: ${greeting}`);
  core.info(`Event payload: ${JSON.stringify(github.context, null, 2)}`);
  
} catch (error) {
  core.setFailed(error.message);
}
```

`.github/actions/greet-action/package.json`:

```json
{
  "name": "greet-action",
  "version": "1.0.0",
  "description": "Greet someone",
  "main": "index.js",
  "dependencies": {
    "@actions/core": "^1.10.0",
    "@actions/github": "^5.1.0"
  }
}
```

### 5.5 カスタムアクションの利用

ワークフローで使用：

```yaml
jobs:
  greet:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Run custom action
        uses: ./.github/actions/greet-action
        with:
          name: GitHub Actions
      
      - name: Use output
        run: echo "Output: ${{ steps.greet.outputs.greeting }}"
```

## 6. 再利用可能ワークフロー

### 6.1 再利用可能ワークフロー vs カスタムアクション

| 特性 | カスタムアクション | 再利用可能ワークフロー |
|------|------------------|-------------------|
| **複雑度** | 簡単な処理 | 複雑なマルチステップ |
| **言語** | JavaScript 推奨 | YAML のみ |
| **ステップ数** | 1 ステップ | 複数ステップ |
| **使用例** | ファイル変換、検証 | CI/CD 全体 |

### 6.2 再利用可能ワークフローの定義

`.github/workflows/test.yml`:

```yaml
name: Test

# これが再利用可能ワークフローの合図
on:
  workflow_call:
    inputs:
      node-version:
        required: false
        type: string
        default: '18'
    
    secrets:
      npm-token:
        required: false

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - uses: actions/setup-node@v4
        with:
          node-version: ${{ inputs.node-version }}
          cache: 'npm'
      
      - run: npm install
      
      - run: npm test
```

### 6.3 再利用可能ワークフローの利用

別ワークフロー（`.github/workflows/main.yml`）から呼び出し：

```yaml
name: Main CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  # 再利用可能ワークフローを呼び出し
  test:
    uses: ./.github/workflows/test.yml
    with:
      node-version: '20'
    secrets:
      npm-token: ${{ secrets.NPM_TOKEN }}
  
  # その他のジョブ
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
      - run: npm install
      - run: npm run lint
```

### 6.4 実践例

**共有ワークフロー例**（`.github/workflows/reusable-build.yml`）:

```yaml
name: Build

on:
  workflow_call:
    inputs:
      environment:
        type: string
        required: true
        description: 'Environment (dev, staging, prod)'
    
    secrets:
      aws-access-key-id:
        required: true
      aws-secret-access-key:
        required: true

jobs:
  build:
    runs-on: ubuntu-latest
    environment: ${{ inputs.environment }}
    
    steps:
      - uses: actions/checkout@v4
      
      - uses: actions/setup-node@v4
        with:
          node-version: '18'
          cache: 'npm'
      
      - run: npm install
      
      - run: npm run build
      
      - name: Configure AWS credentials
        uses: aws-actions/configure-aws-credentials@v2
        with:
          aws-access-key-id: ${{ secrets.aws-access-key-id }}
          aws-secret-access-key: ${{ secrets.aws-secret-access-key }}
          aws-region: us-east-1
      
      - name: Deploy to S3
        run: aws s3 sync dist/ s3://my-bucket/${{ inputs.environment }}/
```

**このワークフローを利用**:

```yaml
name: Deploy

on:
  push:
    branches: [main]

jobs:
  deploy-staging:
    uses: ./.github/workflows/reusable-build.yml
    with:
      environment: staging
    secrets:
      aws-access-key-id: ${{ secrets.AWS_ACCESS_KEY_ID }}
      aws-secret-access-key: ${{ secrets.AWS_SECRET_ACCESS_KEY }}
  
  deploy-prod:
    needs: deploy-staging
    uses: ./.github/workflows/reusable-build.yml
    with:
      environment: prod
    secrets:
      aws-access-key-id: ${{ secrets.AWS_ACCESS_KEY_ID }}
      aws-secret-access-key: ${{ secrets.AWS_SECRET_ACCESS_KEY }}
```

## 7. よくあるアクション利用パターン

### 7.1 Node.js プロジェクトのテスト & ビルド

```yaml
name: Node.js CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        node-version: [16, 18, 20]
    
    steps:
      - uses: actions/checkout@v4
      
      - uses: actions/setup-node@v4
        with:
          node-version: ${{ matrix.node-version }}
          cache: 'npm'
      
      - run: npm ci
      - run: npm run lint
      - run: npm test
      - run: npm run build
      
      - uses: actions/upload-artifact@v3
        with:
          name: build-node-${{ matrix.node-version }}
          path: dist/
```

### 7.2 Python プロジェクトのテスト

```yaml
name: Python CI

on:
  push:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        python-version: ['3.9', '3.10', '3.11']
    
    steps:
      - uses: actions/checkout@v4
      
      - uses: actions/setup-python@v4
        with:
          python-version: ${{ matrix.python-version }}
          cache: 'pip'
      
      - run: pip install -r requirements.txt
      - run: pytest
      - run: black --check .
      - run: flake8 .
```

### 7.3 Docker ビルド & プッシュ

```yaml
name: Build and Push Docker image

on:
  push:
    branches: [main]
    tags: ['v*']

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - uses: docker/setup-buildx-action@v2
      
      - uses: docker/login-action@v2
        with:
          username: ${{ secrets.DOCKER_USERNAME }}
          password: ${{ secrets.DOCKER_PASSWORD }}
      
      - uses: docker/build-push-action@v5
        with:
          context: .
          push: true
          tags: myrepo/myimage:${{ github.sha }}
          cache-from: type=registry,ref=myrepo/myimage:buildcache
          cache-to: type=registry,ref=myrepo/myimage:buildcache,mode=max
```

### 7.4 Slack 通知

```yaml
name: Notify Slack

on:
  push:
    branches: [main]

jobs:
  notify:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - run: npm test
      
      # 成功時
      - uses: slackapi/slack-github-action@v1
        if: success()
        with:
          webhook-url: ${{ secrets.SLACK_WEBHOOK }}
          payload: |
            {
              "text": "✅ Build succeeded",
              "blocks": [
                {
                  "type": "section",
                  "text": {
                    "type": "mrkdwn",
                    "text": "*Build Successful* 🎉\n${{ github.server_url }}/${{ github.repository }}/commit/${{ github.sha }}"
                  }
                }
              ]
            }
      
      # 失敗時
      - uses: slackapi/slack-github-action@v1
        if: failure()
        with:
          webhook-url: ${{ secrets.SLACK_WEBHOOK }}
          payload: |
            {
              "text": "❌ Build failed",
              "color": "danger"
            }
```

### 7.5 GitHub にコメントを投稿

```yaml
name: Add PR Comment

on:
  pull_request:

jobs:
  comment:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm test
      
      - uses: actions/github-script@v7
        with:
          script: |
            github.rest.issues.createComment({
              issue_number: context.issue.number,
              owner: context.repo.owner,
              repo: context.repo.repo,
              body: '✅ Tests passed! Ready to review.'
            })
```
