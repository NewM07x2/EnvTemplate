## 1. ローカル環境構築

### 1.1 Git のインストール

#### Windows

**方法 1: Scoop を使う（推奨）**

```powershell
# Scoop がインストールされていない場合
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
iex (new-object net.webclient).downloadstring('https://get.scoop.sh')

# Git をインストール
scoop install git
```

**方法 2: 公式サイトからインストール**

1. [Git 公式サイト](https://git-scm.com/) にアクセス
2. Windows 用インストーラーをダウンロード
3. 指示に従ってインストール（デフォルト設定で OK）

**確認**：

```powershell
git --version
# Output: git version 2.43.0.windows.1
```

#### macOS

```bash
# Homebrew を使用
brew install git

# 確認
git --version
```

#### Linux（Ubuntu/Debian）

```bash
sudo apt update
sudo apt install git

# 確認
git --version
```

### 1.2 GitHub CLI のインストール

GitHub CLI は、ターミナルから GitHub を操作できるツールです。

#### Windows

```powershell
# Scoop を使う場合
scoop install gh

# Chocolatey を使う場合
choco install gh

# 確認
gh --version
```

#### macOS

```bash
brew install gh

# 確認
gh --version
```

#### Linux

```bash
curl -fsSL https://cli.github.com/packages/githubcli-archive-keyring.gpg | sudo dd of=/usr/share/keyrings/githubcli-archive-keyring.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/githubcli-archive-keyring.gpg] https://cli.github.com/packages stable main" | sudo tee /etc/apt/sources.list.d/github-cli.list > /dev/null
sudo apt update
sudo apt install gh

# 確認
gh --version
```

### 1.3 認証設定

#### GitHub アカウント作成（未登録の場合）

1. [GitHub](https://github.com) にアクセス
2. 「Sign up」から登録
3. メール認証を完了

#### GitHub CLI で認証

```powershell
gh auth login
```

対話形式で以下を選択：

```
? What account do you want to log into?
  GitHub.com
  GitHub Enterprise Server

? What is your preferred protocol for Git operations?
  HTTPS  ← 推奨
  SSH

? Authenticate Git with your GitHub credentials?
  Yes  ← 推奨

? How would you like to authenticate GitHub CLI?
  Login with a web browser  ← 推奨
  Paste an authentication token
```

ブラウザが開くので、GitHub にログインして認可します。

**確認**：

```powershell
gh auth status
# Output: 
# github.com
#   ✓ Logged in to github.com as your-username
```

## 2. リポジトリの準備

### 2.1 テスト用リポジトリの作成

#### GitHub UI から作成

1. GitHub にログイン
2. 右上の「+」 → 「New repository」
3. 以下を設定：
   - **Repository name**: `github-actions-test`
   - **Description**: `GitHub Actions の練習用`
   - **Public**: チェック（パブリックリポジトリは無料実行無制限）
   - **Add a README file**: チェック
4. 「Create repository」をクリック

#### GitHub CLI から作成

```powershell
gh repo create github-actions-test `
  --public `
  --source=. `
  --remote=origin `
  --clone
```

### 2.2 ローカルへのクローン

GitHub UI から作成した場合：

```powershell
cd ~\Documents  # または任意のディレクトリ
git clone https://github.com/YOUR_USERNAME/github-actions-test.git
cd github-actions-test
```

## 3. ワークフローファイルの作成

### 3.1 ファイル構造

ワークフローファイルは以下の場所に配置します：

```
github-actions-test/
├── .github/
│   └── workflows/
│       └── test.yml          ← ここにワークフローファイルを作成
├── src/
│   └── index.js
├── package.json
└── README.md
```

### 3.2 最初のワークフロー

#### Step 1: ディレクトリを作成

```powershell
mkdir .github
mkdir .github/workflows
```

#### Step 2: ワークフローファイルを作成

`.github/workflows/test.yml` を作成：

```yaml
# ワークフロー名
name: Hello GitHub Actions

# 実行トリガー
on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

# ジョブの定義
jobs:
  hello:
    runs-on: ubuntu-latest
    steps:
      - name: Hello World
        run: echo "Hello, GitHub Actions!"
      
      - name: Show current time
        run: date
      
      - name: Show working directory
        run: pwd
```

**各要素の説明**：

| キー | 説明 |
|------|------|
| `name` | ワークフロー名（GitHub UI に表示） |
| `on` | 実行トリガー（いつ実行するか） |
| `jobs` | ジョブの一覧 |
| `runs-on` | 実行環境（`ubuntu-latest`, `windows-latest`, `macos-latest`） |
| `steps` | ジョブ内の処理ステップ |
| `run` | シェルコマンドを実行 |

#### Step 3: Node.js プロジェクトの最小セットアップ

実際のテストを行うため、簡単な Node.js プロジェクトを作成：

**package.json**:

```json
{
  "name": "github-actions-test",
  "version": "1.0.0",
  "scripts": {
    "test": "echo \"Tests passed!\""
  }
}
```

**test.yml を更新（Node.js テスト例）**:

```yaml
name: Node.js Test

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    
    steps:
      # Step 1: ソースコードをチェックアウト
      - name: Checkout code
        uses: actions/checkout@v4
      
      # Step 2: Node.js のセットアップ
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '18'
      
      # Step 3: 依存関係をインストール
      - name: Install dependencies
        run: npm install
      
      # Step 4: テストを実行
      - name: Run tests
        run: npm test
```

## 4. ワークフローの実行

### 4.1 ファイルを Push する

```powershell
# 変更をステージング
git add .

# コミット
git commit -m "Add GitHub Actions workflow"

# Push
git push origin main
```

### 4.2 実行結果を確認

#### 方法 1: GitHub UI から確認

1. リポジトリを GitHub で開く
2. 「Actions」タブをクリック
3. 左側に実行済みのワークフローが表示される
4. クリックして詳細を確認

```
Actions
├── Latest workflows
│   └── Node.js Test
│       ├── Test run #1 ✅ Completed
│       │   └── test (ubuntu-latest)
│       │       ├── Checkout code ✓
│       │       ├── Setup Node.js ✓
│       │       ├── Install dependencies ✓
│       │       └── Run tests ✓
```

#### 方法 2: GitHub CLI から確認

```powershell
# 最新の実行結果を確認
gh run list

# 詳細を表示
gh run view <run-id>

# ログを表示
gh run view <run-id> --log
```

**成功時の出力例**:

```
✓ Node.js Test - main
  Completed successfully
  
  ✓ test
    ✓ Checkout code
    ✓ Setup Node.js
    ✓ Install dependencies
    ✓ Run tests
```

## 5. よくあるトラブル

### ❌ "Workflow file not found"

**原因**: ワークフローファイルのパスが間違っている

**解決策**:
```
正: .github/workflows/test.yml
誤: .github/workflow/test.yml
誤: workflows/test.yml
```

### ❌ "No shell type is allowed to run a script"

**原因**: Windows ランナーでシェルコマンドが実行できない

**解決策**:
```yaml
# 方法 1: bash を明示的に指定
- name: Run command
  shell: bash
  run: echo "Hello"

# 方法 2: PowerShell を使う
- name: Run command
  shell: pwsh
  run: Write-Host "Hello"
```

### ❌ "Permission denied"

**原因**: スクリプトファイルの実行権限がない

**解決策**:
```yaml
- name: Run script
  run: |
    chmod +x ./script.sh
    ./script.sh
```

### ❌ ワークフローが実行されない

**原因**: トリガー条件が満たされていない

**確認方法**:
```yaml
on:
  push:
    branches: [main]  # main ブランチへの push のみ
```

main ではなく develop ブランチに Push している場合は実行されません。
