## 1. トリガーの種類

ワークフローを実行するトリガーは以下の通りです：

| トリガー | 説明 | 用途 |
|---------|------|------|
| **push** | コードを Push したとき | テスト・ビルド・デプロイ |
| **pull_request** | PR を作成・更新したとき | PR のチェック・レビュー |
| **schedule** | 定時実行（cron） | 定期メンテナンス・レポート |
| **workflow_dispatch** | 手動トリガー | 緊急デプロイ・テスト実行 |
| **release** | リリース作成時 | リリースビルド・デプロイ |
| **issues** | Issue 作成・更新時 | 自動ラベル付与・通知 |
| **workflow_run** | 別ワークフロー完了時 | ワークフロー間の連携 |

## 2. push トリガー

コードをリポジトリに Push したときにワークフローを実行します。

### 2.1 基本的な使い方

```yaml
# すべての push で実行
on: push

# または明示的に
on:
  push:
    branches: [main, develop]
```

### 2.2 ブランチの指定

```yaml
on:
  push:
    # 指定したブランチへの push のみ
    branches: [main, develop]
```

**ブランチマッチング**:

```yaml
# 単一ブランチ
branches: [main]

# 複数ブランチ
branches: [main, develop]

# ワイルドカード使用
branches: ['release/**']      # release/* を除く
branches: ['release/**', '!release/**-alpha']  # 除外

# マッチパターン
branches:
  - main
  - 'feature/*'         # feature/xxx
  - 'v[0-9]+.[0-9]+'    # v1.0 など（正規表現）
```

### 2.3 パスの指定

特定のファイルやディレクトリが変更された時のみ実行：

```yaml
on:
  push:
    branches: [main]
    paths:
      - 'src/**'
      - 'package.json'
      - 'package-lock.json'
```

**除外パス**:

```yaml
on:
  push:
    branches: [main]
    paths:
      - 'src/**'
      - '!src/test/**'    # src/test は除外
      - '!**.md'          # Markdown は除外
```

**実用例**:

```yaml
# フロントエンド変更時のみ実行
on:
  push:
    branches: [main]
    paths:
      - 'frontend/**'
      - '.github/workflows/frontend-test.yml'

# ドキュメント変更は無視
on:
  push:
    branches: [main]
    paths-ignore:
      - 'docs/**'
      - '**.md'
```

### 2.4 タグの指定

```yaml
on:
  push:
    tags:
      - 'v*'            # v で始まるタグ（v1.0.0 など）
      - 'release/**'    # release/ で始まるタグ
```

**リリース時の実行例**:

```yaml
on:
  push:
    tags:
      - 'v[0-9]+.[0-9]+.[0-9]+'  # セマンティックバージョニング

jobs:
  release:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm run build
      - name: Create Release
        run: echo "Release ${{ github.ref }} created"
```

## 3. pull_request トリガー

Pull Request の作成・更新時にワークフローを実行します。

### 3.1 基本的な使い方

```yaml
# すべての PR で実行
on: pull_request

# または
on:
  pull_request:
    branches: [main, develop]
```

### 3.2 PR 作成のタイプ

```yaml
on:
  pull_request:
    # 特定のアクティビティタイプのみ
    types: [opened, synchronize, reopened]
```

**よく使うタイプ**:

| タイプ | 説明 |
|--------|------|
| `opened` | PR が開かれた |
| `synchronize` | PR にコミットが追加された |
| `reopened` | クローズされた PR が再度開かれた |
| `edited` | PR のタイトル・説明が編集された |
| `ready_for_review` | ドラフト PR がレビュー準備完了 |
| `labeled` | ラベルが追加された |

```yaml
# PR オープン時のみ実行（コミット push では実行されない）
on:
  pull_request:
    types: [opened]
```

### 3.3 ブランチの指定

```yaml
on:
  pull_request:
    branches: [main]  # main への PR のみ
```

### 3.4 パスの指定

```yaml
on:
  pull_request:
    branches: [main]
    paths:
      - 'src/**'
      - 'package.json'
```

## 4. schedule トリガー（定時実行）

Cron 形式で定時実行を指定します。

### 4.1 Cron 式の基本

```
┌─────────────── 分 (0 - 59)
│ ┌──────────── 時 (0 - 23) UTC
│ │ ┌────────── 日 (1 - 31)
│ │ │ ┌──────── 月 (1 - 12)
│ │ │ │ ┌────── 曜日 (0 - 6) 0 = 日, 6 = 土
│ │ │ │ │
│ │ │ │ │
* * * * *
```

**例**:

```yaml
on:
  schedule:
    # 毎日 UTC 0:00 に実行
    - cron: '0 0 * * *'
```

### 4.2 よくある実行パターン

```yaml
on:
  schedule:
    # 毎日 9:00（JST は UTC+9 なので UTC 0:00）
    - cron: '0 0 * * *'
    
    # 毎週月曜 9:00（日本時間）
    - cron: '0 0 * * 1'
    
    # 毎月 1 日 9:00（日本時間）
    - cron: '0 0 1 * *'
    
    # 平日毎日 9:00（月〜金）
    - cron: '0 0 * * 1-5'
    
    # 1 時間ごと
    - cron: '0 * * * *'
    
    # 30 分ごと
    - cron: '*/30 * * * *'
    
    # 6 時間ごと（0:00, 6:00, 12:00, 18:00）
    - cron: '0 */6 * * *'
```

### 4.3 複数の定時実行

```yaml
on:
  schedule:
    # 平日朝 9:00（JST）
    - cron: '0 0 * * 1-5'
    
    # 土日朝 10:00（JST）
    - cron: '0 1 * * 0,6'
    
    # 毎月 1 日
    - cron: '0 0 1 * *'
```

**実用例**:

```yaml
name: Daily maintenance

on:
  schedule:
    - cron: '0 0 * * *'  # UTC 0:00（日本時間 9:00）

jobs:
  cleanup:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm run cleanup
  
  backup:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm run backup
```

## 5. その他のトリガー

### 5.1 workflow_dispatch（手動トリガー）

GitHub UI から手動でワークフローを実行できます。

```yaml
on:
  workflow_dispatch:
    inputs:
      environment:
        description: 'Environment to deploy'
        required: true
        default: 'staging'
        type: choice
        options:
          - staging
          - production
      
      message:
        description: 'Deploy message'
        required: false
        type: string

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - run: |
          echo "Environment: ${{ inputs.environment }}"
          echo "Message: ${{ inputs.message }}"
```

**GitHub UI での実行**:

1. リポジトリの「Actions」タブ
2. ワークフロー名をクリック
3. 「Run workflow」ボタン
4. パラメータを入力
5. 「Run workflow」実行

### 5.2 release（リリース時）

GitHub Release が作成・公開されたときに実行：

```yaml
on:
  release:
    types: [published]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: |
          echo "Release: ${{ github.event.release.tag_name }}"
          echo "Version: ${{ github.event.release.name }}"
```

### 5.3 issues（Issue 作成時）

Issue が作成・編集されたときに実行：

```yaml
on:
  issues:
    types: [opened, labeled]

jobs:
  auto_comment:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/github-script@v7
        with:
          script: |
            github.rest.issues.createComment({
              issue_number: context.issue.number,
              owner: context.repo.owner,
              repo: context.repo.repo,
              body: 'Thank you for creating an issue!'
            })
```

### 5.4 workflow_run（別ワークフロー完了時）

別のワークフローが完了した後に実行：

```yaml
on:
  workflow_run:
    workflows: [build.yml]  # build.yml ワークフローの完了後
    types: [completed]
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    if: ${{ github.event.workflow_run.conclusion == 'success' }}
    steps:
      - run: echo "Deploying after successful build"
```

## 6. フィルタリングとアクティビティタイプ

### 6.1 push/pull_request での詳細フィルタ

```yaml
on:
  push:
    branches:
      - main
      - 'release/**'
    paths:
      - 'src/**'
      - 'package.json'
    tags:
      - 'v*'
  
  pull_request:
    branches:
      - main
    paths:
      - 'src/**'
```

**除外フィルタ**:

```yaml
on:
  push:
    branches:
      - main
    paths-ignore:
      - 'docs/**'
      - '**.md'
```

### 6.2 アクティビティタイプ（types）

```yaml
on:
  pull_request:
    types:
      - opened        # PR が開かれた
      - synchronize   # 新規コミットが Push された
      - reopened      # 再度開かれた
```

## 7. 実践的な例

### 7.1 本番デプロイは main ブランチのみ

```yaml
name: Deploy to production

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Build
        run: npm run build
      - name: Deploy to production
        run: npm run deploy:prod
```

### 7.2 フロントエンド変更時のみテスト

```yaml
name: Frontend test

on:
  push:
    branches: [main, develop]
    paths:
      - 'frontend/**'
      - 'package.json'
  
  pull_request:
    branches: [main, develop]
    paths:
      - 'frontend/**'
      - 'package.json'

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
      - run: npm install
      - run: npm run test:frontend
```

### 7.3 毎日定時実行

```yaml
name: Daily security scan

on:
  schedule:
    # 毎日 UTC 0:00（日本時間 9:00）
    - cron: '0 0 * * *'

jobs:
  scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm audit
      - name: Generate report
        run: npm run generate:security-report
```

### 7.4 複数のトリガー条件を組み合わせ

```yaml
name: Comprehensive CI/CD

on:
  # PR やコミット時
  push:
    branches: [main, develop]
    paths:
      - 'src/**'
      - 'tests/**'
      - 'package.json'
  
  pull_request:
    branches: [main, develop]
    paths:
      - 'src/**'
      - 'tests/**'
  
  # 手動トリガー
  workflow_dispatch:
  
  # 定時実行（毎日 0:00）
  schedule:
    - cron: '0 0 * * *'

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
      - run: npm install
      - run: npm test
  
  # main ブランチ && 成功時のみデプロイ
  deploy:
    needs: test
    if: |
      github.ref == 'refs/heads/main' && 
      github.event_name == 'push' && 
      success()
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm run deploy
```
