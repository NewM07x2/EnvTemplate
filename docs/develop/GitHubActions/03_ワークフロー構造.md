## 1. YAML ファイルの基本

### 1.1 YAML の構文

YAML は人間が読みやすいデータ形式です。

**基本ルール**:

```yaml
# コメント（# 以降）
key: value

# ネストはスペース（タブは NG）
parent:
  child1: value1
  child2: value2

# リスト
items:
  - item1
  - item2
  - item3
```

**GitHub Actions で使う時の注意**:

```yaml
# ❌ タブを使わない（必ずスペース）
	key: value  ← エラー

# ✅ スペースを使う
  key: value  ← 正しい

# ❌ インデント不一致もエラー
key1: value1
  key2: value2  ← エラー（インデントレベルが違う）

# ✅ インデントを揃える
key1: value1
key2: value2
```

### 1.2 データ型

```yaml
# 文字列
string: "Hello World"

# 数値
number: 42

# 真偽値
boolean: true

# null
nullable: null

# リスト
list:
  - item1
  - item2

# 辞書
dict:
  key1: value1
  key2: value2

# 複数行テキスト（| は改行を保持）
multiline: |
  line1
  line2
  line3

# 複数行テキスト（> は改行をスペースに変換）
folded: >
  This is a very long
  text that will be folded
  into a single line.
```

## 2. ワークフロー全体の構造

```yaml
name: Workflow Name                   ← ワークフロー名

on:                                   ← トリガー条件
  push:
    branches: [main]
  pull_request:
    branches: [main]

env:                                  ← グローバル環境変数
  NODE_VERSION: '18'

jobs:                                 ← ジョブの定義
  job1:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm install
      - run: npm test
  
  job2:
    runs-on: ubuntu-latest
    needs: job1                       ← job1 の完了後に実行
    steps:
      - uses: actions/checkout@v4
      - run: npm run build
```

## 3. ワークフロー最上位のキー

### 3.1 name（ワークフロー名）

```yaml
name: Build and Test
```

- GitHub UI の Actions タブに表示される
- 省略可（その場合、ファイル名が表示される）
- 最大 255 文字

### 3.2 on（トリガー）

ワークフローを実行するきっかけを指定します。詳細は `04_トリガーと実行条件.md` で説明。

```yaml
on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]
  schedule:
    - cron: '0 0 * * *'  # 毎日 UTC 0:00
```

### 3.3 env（環境変数）

ワークフロー全体で使える環境変数を定義します。

```yaml
env:
  NODE_ENV: production
  API_URL: https://api.example.com

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - run: echo ${{ env.NODE_ENV }}
```

**スコープ**:
- グローバル（`env:` で定義）: すべてのジョブとステップで利用可能
- ジョブレベル（`jobs.<job_id>.env:`）: そのジョブ内のみ
- ステップレベル（`jobs.<job_id>.steps[].env:`）: そのステップのみ

### 3.4 jobs（ジョブの定義）

ワークフロー内で実行する複数のジョブを定義します。

```yaml
jobs:
  # ジョブ ID（ユニークである必要がある）
  build:
    runs-on: ubuntu-latest
    steps:
      - run: echo "Building..."
  
  # 別のジョブ
  test:
    runs-on: ubuntu-latest
    needs: build  # build ジョブの完了を待つ
    steps:
      - run: echo "Testing..."
```

## 4. ジョブの詳細設定

### 4.1 runs-on（実行環境）

```yaml
jobs:
  build:
    # GitHub ホステッド ランナー
    runs-on: ubuntu-latest
    # または: windows-latest, macos-latest
    
    # 複数の環境で実行
    strategy:
      matrix:
        os: [ubuntu-latest, windows-latest, macos-latest]
    runs-on: ${{ matrix.os }}
    
    # セルフホステッド ランナー
    runs-on: [self-hosted, linux, x64]
```

**利用可能な環境**:

```yaml
ubuntu-latest     # Ubuntu 22.04 / 20.04
windows-latest    # Windows Server 2022
macos-latest      # macOS 13 / 12
ubuntu-22.04      # Ubuntu 22.04 指定
ubuntu-20.04      # Ubuntu 20.04 指定
windows-2022      # Windows Server 2022 指定
windows-2019      # Windows Server 2019 指定
macos-13          # macOS 13 指定
macos-12          # macOS 12 指定
```

### 4.2 strategy（マトリックス）

複数の環境やバージョンでテストを並列実行します。

```yaml
jobs:
  test:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        node-version: [16, 18, 20]
        os: [ubuntu-latest, windows-latest]
    
    steps:
      - uses: actions/setup-node@v4
        with:
          node-version: ${{ matrix.node-version }}
      
      - run: npm test
```

**実行パターン**: 3 × 2 = 6 個のジョブが並列実行される

```
test (node 16, ubuntu-latest)
test (node 16, windows-latest)
test (node 18, ubuntu-latest)
test (node 18, windows-latest)
test (node 20, ubuntu-latest)
test (node 20, windows-latest)
```

### 4.3 steps（処理ステップ）

各ジョブ内で実行する個別の処理を定義します。詳細は「5. ステップの詳細」を参照。

```yaml
steps:
  - name: Checkout code
    uses: actions/checkout@v4
  
  - name: Run script
    run: npm install
  
  - name: Run command
    run: npm test
```

### 4.4 needs（ジョブの依存関係）

別のジョブの完了を待ってから実行します。

```yaml
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - run: npm run build
  
  test:
    runs-on: ubuntu-latest
    needs: build  # build の完了後に実行
    steps:
      - run: npm test
  
  deploy:
    runs-on: ubuntu-latest
    needs: [build, test]  # 複数の依存関係
    steps:
      - run: npm run deploy
```

**実行順序**:
```
build
  ↓ (完了後)
test
  ↓ (完了後)
deploy
```

### 4.5 if（条件分岐）

特定の条件でジョブをスキップします。

```yaml
jobs:
  deploy:
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main'  # main ブランチのみ実行
    steps:
      - run: npm run deploy
```

**よく使う条件**:

```yaml
# ブランチで分岐
if: github.ref == 'refs/heads/main'

# イベントタイプで分岐
if: github.event_name == 'push'

# Pull Request で分岐
if: github.event_name == 'pull_request'

# ジョブの成功/失敗で分岐
if: success()
if: failure()
```

### 4.6 timeout-minutes（タイムアウト）

```yaml
jobs:
  build:
    runs-on: ubuntu-latest
    timeout-minutes: 30  # 30 分でタイムアウト（デフォルト 360 分）
    steps:
      - run: npm test
```

## 5. ステップの詳細

### 5.1 run（コマンド実行）

シェルコマンドを実行します。

```yaml
steps:
  # 単行コマンド
  - name: Install dependencies
    run: npm install
  
  # 複数行コマンド
  - name: Build and test
    run: |
      npm run build
      npm test
      npm run lint
  
  # 環境別にコマンドを変える
  - name: Setup
    run: |
      if [ "${{ runner.os }}" = "Windows" ]; then
        echo "Windows setup"
      else
        echo "Linux/macOS setup"
      fi
```

**利用可能なシェル**:

```yaml
# Bash（デフォルト、Linux/macOS）
- run: echo "Hello"
  shell: bash

# PowerShell（Windows）
- run: Write-Host "Hello"
  shell: pwsh

# Cmd（Windows）
- run: echo Hello
  shell: cmd

# Python
- run: print('Hello')
  shell: python
```

### 5.2 uses（アクション実行）

再利用可能なアクション（GitHub 公式またはコミュニティ作成）を実行します。

```yaml
steps:
  # GitHub 公式アクション
  - uses: actions/checkout@v4
  
  # 他のバージョンを指定
  - uses: actions/setup-node@v3
  
  # GitHub Marketplace のアクション
  - uses: codecov/codecov-action@v3
  
  # リポジトリ内のカスタムアクション
  - uses: ./.github/actions/custom-action
```

### 5.3 with（パラメータ指定）

アクションにパラメータを渡します。

```yaml
steps:
  - name: Setup Node.js
    uses: actions/setup-node@v4
    with:
      node-version: '18'
      cache: 'npm'
  
  - name: Upload coverage
    uses: codecov/codecov-action@v3
    with:
      files: ./coverage.xml
      flags: unittests
      name: codecov-umbrella
      fail_ci_if_error: true
```

### 5.4 name と id

```yaml
steps:
  # name: 出力に表示される名前
  - name: Test step
    id: test  # id: 後のステップから参照可能
    run: npm test
  
  # id を使用して前のステップの出力を参照
  - name: Show result
    run: echo "Test status: ${{ steps.test.outcome }}"
```

### 5.5 if（ステップの条件分岐）

```yaml
steps:
  - name: Run tests
    run: npm test
  
  # テストが失敗した時のみ実行
  - name: Upload failure logs
    if: failure()
    run: |
      echo "Tests failed!"
      cat test.log
  
  # 成功した時のみ実行
  - name: Deploy
    if: success()
    run: npm run deploy
  
  # 特定の条件で実行
  - name: Deploy to production
    if: github.ref == 'refs/heads/main' && success()
    run: npm run deploy
```

**利用可能な関数**:

| 関数 | 説明 |
|------|------|
| `success()` | 前のステップがすべて成功 |
| `failure()` | 前のステップで 1 つ以上失敗 |
| `always()` | 成功/失敗に関わらず実行 |
| `cancelled()` | ワークフローがキャンセルされた |

### 5.6 env（ステップレベルの環境変数）

```yaml
steps:
  - name: Deploy
    env:
      API_KEY: ${{ secrets.API_KEY }}
      ENVIRONMENT: production
    run: |
      echo $API_KEY
      echo $ENVIRONMENT
```

### 5.7 continue-on-error（エラー無視）

```yaml
steps:
  # このステップでエラーが発生しても、ワークフロー続行
  - name: Run tests
    run: npm test
    continue-on-error: true
  
  # 次のステップは必ず実行される
  - name: Generate report
    run: npm run report
```

## 6. 複雑なワークフロー例

### 6.1 複数ジョブの依存関係

```yaml
name: Build, Test, and Deploy

on:
  push:
    branches: [main]

jobs:
  # ステップ 1: ビルド
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '18'
      - run: npm install
      - run: npm run build
      - uses: actions/upload-artifact@v3
        with:
          name: build-artifacts
          path: dist/
  
  # ステップ 2: テスト（build の完了後）
  test:
    runs-on: ubuntu-latest
    needs: build
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '18'
      - run: npm install
      - run: npm test
  
  # ステップ 3: デプロイ（build と test の完了後）
  deploy:
    runs-on: ubuntu-latest
    needs: [build, test]
    if: github.ref == 'refs/heads/main'
    steps:
      - uses: actions/checkout@v4
      - uses: actions/download-artifact@v3
        with:
          name: build-artifacts
      - run: npm run deploy
```

### 6.2 マトリックステスト

```yaml
name: Multi-Version Test

on: [push, pull_request]

jobs:
  test:
    runs-on: ${{ matrix.os }}
    strategy:
      matrix:
        os: [ubuntu-latest, windows-latest, macos-latest]
        node-version: [16, 18, 20]
        exclude:
          # Windows + Node 16 は skip
          - os: windows-latest
            node-version: 16
    
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: ${{ matrix.node-version }}
      - run: npm install
      - run: npm test
```

### 6.3 条件付き実行

```yaml
name: Conditional Workflow

on: [push, pull_request]

jobs:
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm install
      - run: npm run lint
  
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm install
      - run: npm test
  
  deploy:
    runs-on: ubuntu-latest
    needs: [lint, test]
    # main ブランチ && PR ではない && すべて成功
    if: |
      github.ref == 'refs/heads/main' && 
      github.event_name == 'push' && 
      success()
    steps:
      - uses: actions/checkout@v4
      - run: npm run deploy
```

## 7. よくあるパターン

### 7.1 失敗時の通知

```yaml
name: Build with notification

on: [push]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm test
      
      # テスト失敗時に通知
      - name: Notify failure
        if: failure()
        uses: actions/github-script@v7
        with:
          script: |
            github.rest.issues.createComment({
              issue_number: context.issue.number,
              owner: context.repo.owner,
              repo: context.repo.repo,
              body: '❌ Tests failed! Please check the logs.'
            })
```

### 7.2 環境別のワークフロー

```yaml
name: Deploy to multiple environments

on:
  push:
    branches:
      - main      # 本番へデプロイ
      - develop   # ステージングへデプロイ

jobs:
  deploy:
    runs-on: ubuntu-latest
    env:
      ENVIRONMENT: ${{ github.ref == 'refs/heads/main' && 'production' || 'staging' }}
      API_URL: ${{ github.ref == 'refs/heads/main' && 'https://api.example.com' || 'https://staging-api.example.com' }}
    
    steps:
      - uses: actions/checkout@v4
      - name: Deploy to ${{ env.ENVIRONMENT }}
        run: |
          echo "Deploying to ${{ env.ENVIRONMENT }}"
          echo "API URL: ${{ env.API_URL }}"
```
