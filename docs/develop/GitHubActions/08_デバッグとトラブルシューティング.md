## 1. ログの確認方法

### 1.1 GitHub UI でログを見る

**Web ブラウザから確認**:

1. リポジトリを開く
2. 「Actions」タブをクリック
3. 左側でワークフロー実行を選択
4. ジョブをクリック
5. ステップを展開してログを確認

```
Actions
├── All workflows
│   └── Your Workflow Name
│       └── Run #123 (Completed)
│           ├── job-name
│           │   ├── ✓ Checkout code
│           │   ├── ✓ Setup Node.js
│           │   ├── ✗ Run tests (ここをクリック)
│           │   └── ✓ Build
```

### 1.2 GitHub CLI でログを見る

```powershell
# 最新の実行を確認
gh run list

# 特定の実行のログを表示
gh run view <run-id> --log

# リアルタイムでログを確認
gh run watch <run-id>

# 特定のジョブのログのみ
gh run view <run-id> --job <job-id> --log

# ステップごとの実行時間を表示
gh run view <run-id> --verbose
```

### 1.3 ログの見方

```
2024-03-01T10:30:00.000Z Checking out the repository...
2024-03-01T10:30:02.000Z ✓ Checkout complete
2024-03-01T10:30:02.500Z Run: npm install
2024-03-01T10:30:05.000Z > npm install
2024-03-01T10:30:15.000Z added 500 packages
2024-03-01T10:30:15.500Z ✓ npm install completed
                        ↑
                        各行にタイムスタンプがある
```

**エラー行の見分け方**:

```
❌ = ステップ失敗（赤）
⚠️  = 警告（黄）
✓  = 成功（緑）
⊘  = スキップ
```

## 2. デバッグモードの有効化

### 2.1 Runner debug logging

GitHub Actions の内部ログを有効化：

**リポジトリシークレットに追加**:

1. Settings → Secrets → New repository secret
2. **Name**: `ACTIONS_STEP_DEBUG`
3. **Value**: `true`
4. 「Add secret」をクリック

その後のワークフロー実行でデバッグログが表示されます。

```powershell
# CLI から設定
gh secret set ACTIONS_STEP_DEBUG --body "true"
```

**デバッグモードで出力される追加情報**:

```
[debug] Current working directory: /home/runner/work/repo/repo
[debug] Environment variables:
[debug]   GITHUB_ACTION_PATH=/home/runner/work/...
[debug]   RUNNER_OS=Linux
[debug]   PATH=/home/runner/...
```

### 2.2 ステップレベルでのデバッグ出力

```yaml
jobs:
  debug:
    runs-on: ubuntu-latest
    steps:
      # コアコマンドでデバッグメッセージを出力
      - name: Debug output
        run: |
          echo "::debug::This is a debug message"
          echo "::notice::This is a notice"
          echo "::warning::This is a warning"
          echo "::error::This is an error"
```

**コアコマンドの種類**:

```bash
echo "::debug::Debug message"
echo "::notice::Notice message"
echo "::warning::Warning message"
echo "::error::Error message"
echo "::group::Group Title"
echo "  Grouped content"
echo "::endgroup::"
```

## 3. よくあるエラーと解決策

### 3.1 シークレットが認識されない

**症状**: `${{ secrets.API_KEY }}` が空文字列

**原因**: 
- シークレット名のスペルミス
- 外部 fork の PR（セキュリティ対策）

**解決策**:

```powershell
# シークレット名を確認
gh secret list

# 正しい名前で参照
# ❌ ${{ secrets.API_KAY }}    （"KAY" は誤字）
# ✅ ${{ secrets.API_KEY }}    （正しい）
```

```yaml
# 確認用のデバッグステップ
- name: Check if secret exists
  run: |
    if [ -z "${{ secrets.API_KEY }}" ]; then
      echo "❌ Secret not found!"
      exit 1
    else
      echo "✅ Secret is set"
    fi
```

### 3.2 権限エラー（Permission denied）

**症状**: `Permission denied` エラー

**原因**: 
- スクリプトファイルに実行権限がない
- 外部アクションへのアクセス権限がない

**解決策**:

```yaml
# シェルスクリプトに実行権限を付与
- name: Run script
  run: |
    chmod +x ./scripts/deploy.sh
    ./scripts/deploy.sh

# または
- name: Run script
  shell: bash
  run: bash ./scripts/deploy.sh
```

### 3.3 ファイルが見つからない

**症状**: `No such file or directory`

**原因**: 
- ワーキングディレクトリが違う
- ファイルパスが相対パス

**解決策**:

```yaml
- name: Debug working directory
  run: |
    pwd  # 現在のディレクトリを表示
    ls -la  # ファイル一覧

- name: Run with correct path
  run: |
    ls ./src/index.js  # 存在確認
    npm test
```

### 3.4 依存関係のインストール失敗

**症状**: `npm ERR!`, `pip error` など

**原因**: 
- Lock ファイルが古い
- ネットワーク接続の問題
- レジストリが使用不可

**解決策**:

```yaml
# npm の場合
- name: Clear npm cache
  run: npm cache clean --force

- name: Install with verbose output
  run: npm install --verbose

# pip の場合
- name: Upgrade pip
  run: python -m pip install --upgrade pip

- name: Install with error output
  run: pip install -r requirements.txt --verbose
```

### 3.5 タイムアウト

**症状**: ジョブが 360 分でタイムアウト

**原因**: 
- テストやビルドが長い
- プロセスが応答しない

**解決策**:

```yaml
jobs:
  build:
    timeout-minutes: 120  # デフォルト 360
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      # 長時間プロセスをバックグラウンドに
      - name: Long running task
        run: npm run build
        timeout-minutes: 60

      # または条件付きスキップ
      - name: Conditional long task
        if: github.event_name == 'push'
        run: npm run expensive-operation
```

### 3.6 メモリ不足

**症状**: `Out of memory` エラー

**原因**: 
- 大きなファイルのビルド
- メモリリーク

**解決策**:

```yaml
- name: Build with limited memory
  run: |
    # Node.js のメモリ上限を設定
    NODE_OPTIONS="--max-old-space-size=4096" npm run build

- name: Check memory usage
  run: |
    free -h  # Linux
    Get-ComputerInfo | Select CsTotalPhysicalMemory  # Windows
```

### 3.7 Docker イメージが見つからない

**症状**: `no such file or directory` (Docker 実行時)

**原因**: 
- イメージがビルドされていない
- イメージ名が違う

**解決策**:

```yaml
- name: Build Docker image
  run: docker build -t my-image:latest .

- name: List images
  run: docker images

- name: Run with correct image
  run: docker run --rm my-image:latest npm test
```

## 4. ステップごとのデバッグ

### 4.1 echo でデバッグ出力

```yaml
- name: Debug environment
  run: |
    echo "=== Environment Debug ==="
    echo "GitHub SHA: ${{ github.sha }}"
    echo "GitHub Ref: ${{ github.ref }}"
    echo "GitHub Event: ${{ github.event_name }}"
    echo "Current directory: $(pwd)"
    echo "Environment variables:"
    env | grep -E "NODE|PYTHON" || true
    echo "Node version: $(node --version)"
    echo "npm version: $(npm --version)"
```

### 4.2 actions/github-script でデバッグ

```yaml
- uses: actions/github-script@v7
  with:
    script: |
      console.log('Event:', context.eventName)
      console.log('Payload:', JSON.stringify(context.payload, null, 2))
      console.log('Actor:', context.actor)
      console.log('Ref:', context.ref)
```

### 4.3 アーティファクトでデバッグ情報を保存

```yaml
- name: Run tests
  run: npm test || true  # 失敗しても続行

- name: Upload debug logs
  if: always()  # 成功・失敗に関わらず実行
  uses: actions/upload-artifact@v3
  with:
    name: test-logs
    path: |
      logs/
      coverage/
      test-results.json
    retention-days: 30
```

## 5. ワークフロー構文エラー

### 5.1 YAML 構文エラー

**症状**: `Workflow is invalid` エラー

**原因**: YAML インデント違い

```yaml
# ❌ 間違い: インデント不一致
jobs:
  build:
  runs-on: ubuntu-latest  # インデント違う
  steps:
    - run: npm test

# ✅ 正しい
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - run: npm test
```

**チェック方法**:

```powershell
# YAML バリデーター（オンライン）
# https://www.yamllint.com/

# または jq でローカルチェック
python -m json.tool < .github/workflows/test.yml
```

### 5.2 コンテキスト参照エラー

**症状**: `Unrecognized named-value` エラー

**原因**: 存在しないコンテキストを参照

```yaml
# ❌ 間違い
run: echo ${{ github.user }}  # "user" は存在しない

# ✅ 正しい
run: echo ${{ github.actor }}
```

**利用可能なコンテキスト**:

```yaml
github.ref          # ブランチ名
github.sha          # コミット SHA
github.actor        # ユーザー名
github.event_name   # トリガー名
github.repository   # リポジトリ名
github.workspace    # ワーキングディレクトリ
```

## 6. 実行環境の問題

### 6.1 OS ごとのコマンド違い

```yaml
jobs:
  test:
    runs-on: ${{ matrix.os }}
    strategy:
      matrix:
        os: [ubuntu-latest, windows-latest, macos-latest]
    
    steps:
      - name: OS-specific command
        run: |
          if [ "${{ runner.os }}" = "Windows" ]; then
            echo "Running on Windows"
            dir
          else
            echo "Running on Linux/macOS"
            ls -la
          fi
        shell: bash  # bash を明示的に指定
```

### 6.2 環境変数の確認

```yaml
- name: Show environment
  run: |
    echo "=== System Info ==="
    uname -a
    
    echo "=== GitHub Environment ==="
    echo "GITHUB_WORKSPACE: $GITHUB_WORKSPACE"
    echo "GITHUB_EVENT_PATH: $GITHUB_EVENT_PATH"
    
    echo "=== All Environment Variables ==="
    env | sort
```

### 6.3 ランナーのリソース確認

```yaml
- name: Check runner resources
  run: |
    echo "=== CPU ==="
    nproc  # CPU コア数
    
    echo "=== Memory ==="
    free -h
    
    echo "=== Disk ==="
    df -h
```

## 7. キャッシュの問題

### 7.1 キャッシュが無視される

**症状**: キャッシュが使用されない

**原因**: 
- キーが毎回変わる
- パスが存在しない

```yaml
# ❌ 問題のあるキャッシュキー
- uses: actions/cache@v3
  with:
    path: ~/.npm
    key: ${{ runner.os }}-npm-${{ github.sha }}  # 毎回異なる
    restore-keys: |
      ${{ runner.os }}-npm-

# ✅ 正しいキャッシュキー
- uses: actions/cache@v3
  with:
    path: ~/.npm
    key: ${{ runner.os }}-npm-${{ hashFiles('**/package-lock.json') }}
    restore-keys: |
      ${{ runner.os }}-npm-
```

### 7.2 キャッシュをクリア

Web UI でキャッシュを削除：

1. Settings → Actions → Caches
2. 削除したいキャッシュの「Delete」をクリック

## 8. ジョブ実行の問題

### 8.1 ジョブが実行されない

**症状**: ワークフローが実行されない

**原因**: 
- トリガー条件が満たされていない
- ワークフロー無効

**確認方法**:

```yaml
# トリガー条件を確認
on:
  push:
    branches: [main]  # main へ push した？
  pull_request:
    branches: [main]  # main への PR を作成した？

# GitHub Actions が無効になっていないか確認
# Settings → Actions → General → All actions allowed
```

### 8.2 ジョブが遅い

**症状**: ジョブの実行が遅い

**原因**: 
- 依存関係のインストールが遅い
- テストが多い
- キャッシュを使用していない

**最適化**:

```yaml
- uses: actions/setup-node@v4
  with:
    node-version: '18'
    cache: 'npm'  # キャッシュを有効化

- name: Install with ci
  run: npm ci  # npm install より高速

- name: Run only changed tests
  run: npm test -- --changedSince=origin/main
```

### 8.3 並列実行数を制限

```yaml
jobs:
  test:
    runs-on: ubuntu-latest
    strategy:
      max-parallel: 2  # 最大 2 つまで並列実行
      matrix:
        test-suite: [unit, integration, e2e]
    
    steps:
      - run: npm run test:${{ matrix.test-suite }}
```

## 9. Pull Request でのデバッグ

### 9.1 PR で実行されないワークフロー

**原因**: `pull_request` トリガーが設定されていない

```yaml
on:
  push:
    branches: [main]
  pull_request:  # PR トリガーを追加
    branches: [main]
```

### 9.2 fork から PR が実行されない

**これは意図的なセキュリティ対策です**

fork 元の PR では、シークレットにアクセスできません。

**対策**:

```yaml
on:
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      # シークレットを使わないテスト
      - uses: actions/checkout@v4
      - run: npm test

  deploy:
    # push のみで実行
    if: github.event_name == 'push'
    needs: test
    runs-on: ubuntu-latest
    steps:
      # シークレットを使う処理
      - run: npm run deploy
        env:
          API_KEY: ${{ secrets.API_KEY }}
```

## 10. 実用的なデバッグテンプレート

```yaml
name: Debug Workflow

on: [push, pull_request]

jobs:
  debug:
    runs-on: ubuntu-latest
    
    steps:
      - uses: actions/checkout@v4
      
      # 環境情報を出力
      - name: Debug environment
        run: |
          echo "=== GitHub Context ==="
          echo "SHA: ${{ github.sha }}"
          echo "Ref: ${{ github.ref }}"
          echo "Event: ${{ github.event_name }}"
          echo "Actor: ${{ github.actor }}"
          
          echo "=== System Info ==="
          uname -a
          node --version
          npm --version
          
          echo "=== Working Directory ==="
          pwd
          ls -la
      
      # ステップの成功・失敗を記録
      - name: Run tests
        id: test
        continue-on-error: true
        run: npm test
      
      # デバッグアーティファクトを保存
      - name: Save debug artifacts
        if: always()
        uses: actions/upload-artifact@v3
        with:
          name: debug-logs
          path: |
            logs/
            coverage/
            .npm-debug.log
          retention-days: 7
      
      # 失敗時のサマリーを出力
      - name: Test result summary
        if: failure()
        run: |
          echo "Test failed!"
          echo "See debug artifacts for details"
```
