## 3. ツール・拡張機能

### 3.1 ローカル実行ツール

| ツール | 説明 | インストール |
|--------|------|-----------|
| **act** | ローカルで GitHub Actions ワークフロー実行 | `brew install act` or `choco install act` |
| **GitHub CLI** | コマンドラインから GitHub 操作 | `brew install gh` or `choco install gh` |
| **GitHub Copilot** | AI が ワークフロー生成支援 | VS Code 拡張 |

**act の使用例**:

```bash
# ローカルでワークフロー実行
act push

# 特定のジョブのみ実行
act -j build

# コンテナイメージを指定
act -P ubuntu-latest=ghcr.io/catthehacker/ubuntu:full-latest

# 詳細: https://github.com/nektos/act
```

### 3.2 YAML バリデータ

| ツール | 説明 | URL |
|--------|------|-----|
| **yamllint** | YAML 構文チェック | [yamllint.com](https://www.yamllint.com/) |
| **YAML Validator** | オンライン YAML チェック | [codebeautify.org](https://codebeautify.org/yaml-validator) |
| **GitHub Actions Syntax Checker** | ワークフロー構文チェック | リポジトリ内で自動実行 |

### 3.3 VS Code 拡張

| 拡張 | ID | 説明 |
|------|-----|------|
| **GitHub Actions** | `github.vscode-github-actions` | ワークフロー構文チェック・実行 |

**インストール**:

```bash
code --install-extension github.vscode-github-actions
```

## 9. ナレッジベース

### 9.1 FAQ

**Q1: ワークフローが実行されない場合は？**
A: `Actions` タブで実行状況確認。トリガー条件（ブランチ名など）が正しいか確認。

**Q2: シークレットはどこに保存される？**
A: GitHub が暗号化して安全に保存。ログには表示されない（自動マスク）。

**Q3: キャッシュはどのくらい保存される？**
A: 7 日間未使用で削除。リポジトリあたり 5GB まで。

**Q4: 本番環境へのデプロイは承認が必要？**
A: `environment` 設定で承認ルール（required reviewers）を設定可能。

**Q5: セルフホストランナーは使える？**
A: はい。組織内に専用マシンを配置して使用可能。

### 9.2 よくあるエラー

**「Permission denied」エラー**:
- 原因: スクリプト実行権限がない
- 対策: `chmod +x script.sh` または `shell: bash` 指定

**「Workflow is invalid」エラー**:
- 原因: YAML 構文エラー
- 対策: インデント確認、バリデータ使用

**「Secrets is empty」エラー**:
- 原因: シークレット名のスペルミス
- 対策: `gh secret list` で確認

**「Timeout after 360 minutes」エラー**:
- 原因: ジョブが長時間実行
- 対策: `timeout-minutes` 設定、並列化
