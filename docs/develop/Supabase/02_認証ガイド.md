## Supabase 認証の仕組み

### 認証フロー

```
┌─────────────────┐
│  ユーザー       │
│（ブラウザ）     │
└────────┬────────┘
         │ メール & パスワード送信
         ▼
┌─────────────────────────────────┐
│  Supabase Authentication Service │
├─────────────────────────────────┤
│ 1. メール・パスワード検証        │
│ 2. JWT トークン生成              │
│ 3. JWT をクライアントに返却      │
└────────┬────────────────────────┘
         │ JWT トークン返却
         ▼
┌─────────────────┐
│  ユーザー       │
│（ブラウザ）     │
│ JWT 保存        │
└────────┬────────┘
         │ 以降のリクエストに JWT を含める
         ▼
┌─────────────────────────────────┐
│  Supabase API / Database         │
├─────────────────────────────────┤
│ JWT 検証 → アクセス許可 / 拒否    │
└─────────────────────────────────┘
```

### JWT とは

JWT（JSON Web Token）は、ユーザーが認証済みであることを証明するトークンです。

```
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJzdWIiOiI...
```

構成：
```
Header.Payload.Signature
─────────────────────────
[暗号化されたユーザー情報] + [署名]
```

## ステップ 2: JavaScript/TypeScript での実装

### 2-1. ライブラリをインストール

```bash
npm install @supabase/supabase-js
```

### 2-2. Supabase クライアント初期化

`lib/supabase.ts` (または `lib/supabaseClient.js`) を作成：

```typescript
import { createClient } from '@supabase/supabase-js'

const supabaseUrl = process.env.NEXT_PUBLIC_SUPABASE_URL!
const supabaseAnonKey = process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!

export const supabase = createClient(supabaseUrl, supabaseAnonKey)
```

### 2-3. 環境変数を設定

`.env.local` ファイルに以下を追加：

```env
NEXT_PUBLIC_SUPABASE_URL=https://xxxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

⚠️ 注意：
- `NEXT_PUBLIC_` プレフィックスで始まるため、**クライアント側に公開される**
- API キーは秘密ではなく、RLS ポリシーで保護する
- サーバー側の秘密キーは `.env` に記載（GitHub に含めない）

## ステップ 3: ユーザー登録

### 実装例

```typescript
import { supabase } from '@/lib/supabase'

async function signUp(email: string, password: string, name: string) {
  try {
    // 1. Supabase で認証ユーザーを作成
    const { data: authData, error: authError } = 
      await supabase.auth.signUp({
        email,
        password
      })

    if (authError) throw authError

    // 2. users テーブルにプロフィール情報を保存
    const { data: userData, error: userError } = 
      await supabase
        .from('users')
        .insert([
          {
            id: authData.user!.id,  // 認証ユーザー ID を使用
            email,
            name
          }
        ])

    if (userError) throw userError

    return { success: true, user: userData }
  } catch (error) {
    console.error('登録エラー:', error)
    return { success: false, error }
  }
}
```

### React コンポーネント例

```typescript
import { useState } from 'react'
import { signUp } from '@/lib/auth'

export function SignUpForm() {
  const [email, setEmail] = useState('')
  const [password, setPassword] = useState('')
  const [name, setName] = useState('')
  const [loading, setLoading] = useState(false)
  const [message, setMessage] = useState('')

  const handleSignUp = async (e: React.FormEvent) => {
    e.preventDefault()
    setLoading(true)

    const result = await signUp(email, password, name)

    if (result.success) {
      setMessage('✅ 登録完了！メール確認のため、メールをチェックしてください。')
    } else {
      setMessage('❌ 登録に失敗しました：' + result.error?.message)
    }

    setLoading(false)
  }

  return (
    <form onSubmit={handleSignUp}>
      <input
        type="text"
        placeholder="名前"
        value={name}
        onChange={(e) => setName(e.target.value)}
        required
      />
      <input
        type="email"
        placeholder="メールアドレス"
        value={email}
        onChange={(e) => setEmail(e.target.value)}
        required
      />
      <input
        type="password"
        placeholder="パスワード"
        value={password}
        onChange={(e) => setPassword(e.target.value)}
        required
      />
      <button type="submit" disabled={loading}>
        {loading ? '登録中...' : '登録'}
      </button>
      {message && <p>{message}</p>}
    </form>
  )
}
```

## ステップ 4: ログイン

### 実装例

```typescript
async function signIn(email: string, password: string) {
  try {
    const { data, error } = await supabase.auth.signInWithPassword({
      email,
      password
    })

    if (error) throw error

    // ログイン成功
    console.log('✅ ログイン成功:', data.user.email)
    return { success: true, user: data.user }
  } catch (error) {
    console.error('❌ ログインエラー:', error)
    return { success: false, error }
  }
}
```

### React コンポーネント例

```typescript
import { useState } from 'react'
import { useRouter } from 'next/navigation'
import { signIn } from '@/lib/auth'

export function LoginForm() {

  const [loading, setLoading] = useState(false)

        type="email"
        placeholder="メールアドレス"
        value={email}
        onChange={(e) => setEmail(e.target.value)}
        required
      />
      <input
        type="password"
        placeholder="パスワード"
        value={password}
        onChange={(e) => setPassword(e.target.value)}
        required
      />
      <button type="submit" disabled={loading}>
        {loading ? 'ログイン中...' : 'ログイン'}
      </button>
    </form>
  )
}
```

## ステップ 5: 現在のユーザー情報取得

### 実装例

```typescript
async function getCurrentUser() {
  try {
    const { data: { user }, error } = await supabase.auth.getUser()

    if (error) throw error
    
    return user  // ユーザー情報が返される
  } catch (error) {
    console.error('ユーザー情報取得エラー:', error)
    return null
  }
}
```

### React Hook 例

```typescript
import { useEffect, useState } from 'react'
import { supabase } from '@/lib/supabase'

export function useAuthUser() {
  const [user, setUser] = useState(null)
  const [loading, setLoading] = useState(true)

  useEffect(() => {
    // 現在のセッションを確認
    const { data: { subscription } } = supabase.auth.onAuthStateChange(
      (event, session) => {
        setUser(session?.user ?? null)
        setLoading(false)
      }
    )

    return () => subscription?.unsubscribe()
  }, [])

  return { user, loading }
}
```

### 使用例

```typescript
export function Dashboard() {
  const { user, loading } = useAuthUser()

  if (loading) return <div>読み込み中...</div>
  if (!user) return <div>ログインしてください</div>

  return <div>ようこそ、{user.email}さん！</div>
}
```

## ステップ 7: パスワードリセット

### 1️⃣ リセットメール送信

```typescript
async function resetPassword(email: string) {
  try {
    const { error } = await supabase.auth.resetPasswordForEmail(email)
    
    if (error) throw error
    
    console.log('✅ リセットメールを送信しました')
    return { success: true }
  } catch (error) {
    console.error('❌ エラー:', error)
    return { success: false, error }
  }
}
```

### 2️⃣ メール内のリンクで新パスワード設定

メール内のリンクをクリックすると、リセット画面（`/auth/callback` など）へリダイレクトされます。

```typescript
async function updatePassword(newPassword: string) {
  try {
    const { error } = await supabase.auth.updateUser({
      password: newPassword
    })
    
    if (error) throw error
    
    console.log('✅ パスワードを更新しました')
    return { success: true }
  } catch (error) {
    console.error('❌ エラー:', error)
    return { success: false, error }
  }
}
```

## セキュリティのベストプラクティス

### 🔐 Do's（推奨）

- ✅ **パスワードは最小 8 文字** — Supabase で強制可能
- ✅ **HTTPS のみを使用** — 本番環境では必須
- ✅ **環境変数で API キーを管理** — GitHub に含めない
- ✅ **RLS ポリシーで行レベルのアクセス制御** — 次のガイド参照
- ✅ **JWT トークンは自動更新** — Supabase が自動管理

### 🚫 Don'ts（禁止）

- ❌ **API キー（anon_key）を秘密と思わない** — クライアント公開前提で設計
- ❌ **パスワードをログに記録** — 絶対に記録しない
- ❌ **ユーザーの認証なしに個人情報を返す** — RLS で保護する
- ❌ **トークン情報をクライアントで手動管理** — Supabase に任せる

## 🆘 よくあるエラー

### Q: 「Invalid login credentials」エラー

**答え**: メールアドレスまたはパスワードが間違っています。確認してください。

### Q: ユーザー登録後、メール確認ができない

**答え**: Supabase ダッシュボード → Authentication → Emails で、メール送信設定を確認してください。開発環境では自動確認も可能です。

### Q: JWT トークンの有効期限は？

**答え**: デフォルトは 1 時間。Supabase が自動更新します。設定は `Settings → JWT expiry` で変更可能。
