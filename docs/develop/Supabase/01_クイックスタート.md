## ステップ 1: Supabase アカウント作成

### 1-1. サイトにアクセス

[https://supabase.com](https://supabase.com) にアクセスしてください。

### 1-2. GitHub でサインアップ（推奨）

「Sign Up」→「Continue with GitHub」をクリック。

```
┌─────────────────────────────────┐
│ Supabase ログイン画面            │
├─────────────────────────────────┤
│ □ Email アドレスでサインアップ   │
│ □ GitHub でサインアップ ✅推奨  │
│ □ Google でサインアップ         │
└─────────────────────────────────┘
```

GitHub 認証を許可して、サインアップを完了してください。

## ステップ 2: 新規プロジェクト作成

### 2-1. ダッシュボードから「New Project」をクリック

![Supabase ダッシュボード](../images/supabase-dashboard.png)

### 2-2. プロジェクト情報を入力

| 項目 | 入力例 | 説明 |
|------|--------|------|
| **Organization** | My Organization | プロジェクトが属する組織（初回は自動作成） |
| **Project Name** | my-first-app | プロジェクト名。わかりやすい名前を付ける |
| **Database Password** | (生成) | 自動生成パスワード。**コピーして保存** |
| **Region** | Tokyo / Singapore | リージョン選択。日本なら Tokyo がおすすめ |

```
┌──────────────────────────────────────────┐
│ New Project                              │
├──────────────────────────────────────────┤
│ Organization: [My Organization      ▼] │
│ Project Name: [my-first-app________]  │
│ Database Password: [••••••••••••••••]  │
│               Copy to Clipboard     [📋] │
│ Region: [Tokyo             ▼]          │
│                                       │
│                   [Create new project] │
└──────────────────────────────────────────┘
```

### 2-3. 「Create new project」をクリック

プロジェクトが作成されるまで **1〜2 分待機** してください。

## ステップ 3: テーブル作成

### 3-1. ダッシュボードの「SQL Editor」を開く

左メニュー → 「SQL Editor」をクリック。

### 3-2. 新規クエリを実行

以下の SQL をコピーして、SQL Editor に貼り付けて実行してください：

```sql
-- ユーザー管理用テーブル
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email VARCHAR UNIQUE NOT NULL,
  name VARCHAR,
  created_at TIMESTAMP DEFAULT NOW()
);

-- 投稿管理用テーブル
CREATE TABLE posts (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id) ON DELETE CASCADE,
  title VARCHAR NOT NULL,
  content TEXT,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);
```

実行後、以下のメッセージが表示されれば成功：
```
✅ Query executed successfully
```

### 3-3. テーブルが作成されたか確認

左メニュー → 「Table Editor」をクリック。

以下の 2 つのテーブルが表示されていれば OK：
- `users`
- `posts`

## ステップ 4: テストデータ挿入

### 4-1. Table Editor から「users」テーブルを開く

左メニュー → 「Table Editor」 → 「users」をクリック。

### 4-2. 「Insert row」をクリック

```
┌─────────────────────────────────────────┐
│ users テーブル                          │
├─────────────────────────────────────────┤
│ [+ Insert row]                          │
├─────────────────────────────────────────┤
│ id | email | name | created_at         │
│────┼───────┼──────┼─────────────────────│
│    |       |      |                    │
└─────────────────────────────────────────┘
```

### 4-3. データを入力

新しい行が表示されるので、以下を入力：

| カラム | 入力値 |
|--------|--------|
| **email** | user@example.com |
| **name** | 山田太郎 |

**id** と **created_at** は自動生成されるため入力不要です。

### 4-4. 同様に posts テーブルにもデータを追加

「posts」テーブルを開いて、以下を入力：

| カラム | 入力値 |
|--------|--------|
| **user_id** | 先ほど作成したユーザーの ID |
| **title** | 最初の投稿 |
| **content** | これは Supabase のテスト投稿です。 |

## ステップ 5: フロントエンドからの接続テスト

### 5-1. API キーを取得

左メニュー → 「Settings」 → 「API」をクリック。

以下の 2 つのキーが表示されます：
- **Project URL** — API の URL（例: `https://xxxxx.supabase.co`）
- **anon public** — 公開用 API キー（クライアント側で使用）

これらを **コピーして保存** してください。

```
Project URL: https://xxxxx.supabase.co
anon public: eyJhbGciOiJIUzI1NiIsInR5c...
```

### 5-2. Node.js でテスト（オプション）

Next.js や Node.js プロジェクトで、以下のコードでテスト接続できます：

```javascript
// npm install @supabase/supabase-js
import { createClient } from '@supabase/supabase-js'

const supabase = createClient(
  'https://xxxxx.supabase.co',  // Project URL
  'YOUR_ANON_KEY'                // API キー
)

// ユーザー一覧を取得
const { data, error } = await supabase
  .from('users')
  .select('*')

console.log('ユーザー:', data)
console.log('エラー:', error)
```

**期待される出力:**
```json
{
  "data": [
    {
      "id": "uuid-xxx",
      "email": "user@example.com",
      "name": "山田太郎",
      "created_at": "2026-02-28T10:00:00Z"
    }
  ],
  "error": null
}
```

### 5-3. REST API でテスト（オプション）

curl コマンドでテストすることもできます：

```bash
curl -H "apikey: YOUR_ANON_KEY" \
  https://xxxxx.supabase.co/rest/v1/users \
  -H "Content-Type: application/json"
```

## 🆘 トラブルシューティング

### Q: プロジェクト作成後、ダッシュボードが真っ白のまま

**答え**: ブラウザキャッシュをクリアして再読み込みしてください。
```
Ctrl + Shift + R （Windows）
Cmd + Shift + R （Mac）
```

### Q: SQL 実行時に「Permission denied」エラー

**答え**: ダッシュボード上で実行していることを確認してください。ユーザーが持つ権限の制限ではなく、SQL Editor の権限です。

### Q: テーブルが「Table Editor」に表示されない

**答え**: 左メニューで「All tables」を選択していることを確認してください。
