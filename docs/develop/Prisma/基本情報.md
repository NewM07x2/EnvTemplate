## 1. Prisma とは何か

**Prisma** は、Node.js / TypeScript 向けの **次世代 ORM（Object-Relational Mapper）** です。

ORM とは、プログラミング言語のオブジェクトとデータベースのテーブルを対応付け、SQL を直接書かずにデータベース操作を行う仕組みです。

```
┌─────────────────┐      ┌─────────────────┐      ┌─────────────────┐
│  TypeScript      │      │  Prisma Client   │      │  PostgreSQL     │
│  コード          │ ──→  │  (自動生成)       │ ──→  │  データベース     │
│                  │      │                  │      │                  │
│  prisma.user     │      │  型安全な SQL     │      │  SELECT * FROM  │
│  .findMany()     │      │  クエリ生成       │      │  "User" ...     │
└─────────────────┘      └─────────────────┘      └─────────────────┘
```

### 一言でまとめると

> **「スキーマファイルを書くだけで、型安全な DB クライアントが自動生成される」**

## 2. 従来の DB 操作との比較

### ❌ 従来の方法（生 SQL）

```typescript
// 型情報なし・スペルミスは実行時エラー
const result = await db.query(
  'SELECT * FROM users WHERE email = $1',
  ['alice@example.com']
)
const user = result.rows[0] // any 型 → ランタイムエラーの温床
```

**問題点:**
- 型情報がない（`any` 型）
- SQL のスペルミスはコンパイル時に検出できない
- テーブル名やカラム名の変更時に修正漏れが起きやすい

### ✅ Prisma の方法

```typescript
// 完全な型安全・補完が効く・SQL を書かない
const user = await prisma.user.findUnique({
  where: { email: 'alice@example.com' }
})
// user の型は `User | null` → TypeScript が自動で推論
```

**利点:**
- 完全な型推論（自動補完が効く）
- テーブル名やカラム名の変更はスキーマを修正 → `prisma generate` で検出
- SQL の知識がなくても直感的に使える

## 3. Prisma の構成要素

Prisma は 4 つの主要コンポーネントで構成されています。

```
┌──────────────────────────────────────────────────────────┐
│                    Prisma エコシステム                      │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  📄 Prisma Schema (schema.prisma)                        │
│   └── DB の構造（テーブル・カラム・リレーション）を宣言的に定義  │
│                                                          │
│  🔧 Prisma CLI                                           │
│   └── マイグレーション・クライアント生成・DB 管理用コマンド      │
│                                                          │
│  📦 Prisma Client (@prisma/client)                       │
│   └── スキーマから自動生成される型安全な DB クライアント        │
│                                                          │
│  🖥️ Prisma Studio                                        │
│   └── DB の内容をブラウザで閲覧・編集できる GUI ツール         │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

### 各コンポーネントの詳細

| コンポーネント | パッケージ名 | 役割 |
|-------------|------------|------|
| **Prisma Schema** | - | `schema.prisma` ファイルでモデル（テーブル）を定義 |
| **Prisma CLI** | `prisma` (devDependencies) | `npx prisma` コマンドで操作 |
| **Prisma Client** | `@prisma/client` (dependencies) | アプリケーションコードから import して使用 |
| **Prisma Studio** | CLI に同梱 | `npx prisma studio` で起動 |

### インストール時のパッケージの違い

```bash
npm install @prisma/client    # 本番でも使う（dependencies）
npm install -D prisma         # 開発時のみ使う（devDependencies）
```

> **注意:** `prisma`（CLI）と `@prisma/client`（ランタイム）は**別パッケージ**です。両方必要です。

## 4. 対応データベース

| データベース | サポート状況 | 本リポジトリでの使用 |
|------------|------------|-------------------|
| **PostgreSQL** | ✅ 完全サポート | ✅ 使用中 |
| **MySQL / MariaDB** | ✅ 完全サポート | - |
| **SQLite** | ✅ 完全サポート | - |
| **SQL Server** | ✅ 完全サポート | - |
| **MongoDB** | ✅ サポート | - |
| **CockroachDB** | ✅ サポート | - |

### データベースごとの接続文字列例

```env
# PostgreSQL
DATABASE_URL="postgresql://user:password@localhost:5432/mydb?schema=public"

# MySQL
DATABASE_URL="mysql://user:password@localhost:3306/mydb"

# SQLite（ファイルベース）
DATABASE_URL="file:./dev.db"

# MongoDB
DATABASE_URL="mongodb+srv://user:password@cluster.mongodb.net/mydb"
```

## 5. Prisma のメリット・デメリット

### ✅ メリット

| # | メリット | 詳細 |
|---|---------|------|
| 1 | **型安全性** | スキーマから TypeScript 型が自動生成され、コンパイル時にエラーを検出 |
| 2 | **自動補完** | VS Code での入力補完が非常に強力（テーブル名・カラム名・フィルタ条件） |
| 3 | **直感的な API** | `findMany`, `create`, `update` など、命名が分かりやすい |
| 4 | **マイグレーション管理** | スキーマの変更から SQL マイグレーションファイルを自動生成 |
| 5 | **GUI ツール** | Prisma Studio で DB の内容をブラウザから確認・編集 |
| 6 | **活発な開発** | 頻繁なアップデートと充実したドキュメント |
| 7 | **エコシステム** | Prisma Accelerate（接続プーリング）、Prisma Pulse（リアルタイム）等 |

### ⚠️ デメリット・注意点

| # | デメリット | 詳細 | 対処法 |
|---|-----------|------|--------|
| 1 | **複雑な SQL** | 複雑な集計・ウィンドウ関数は直接書けない | `$queryRaw` で生 SQL を使用 |
| 2 | **学習コスト** | スキーマ言語（PSL）の学習が必要 | 公式ドキュメント参照 |
| 3 | **パフォーマンス** | 極端に最適化されたクエリが必要な場合は不向き | 生 SQL + 型定義で対応 |
| 4 | **クライアント再生成** | スキーマ変更のたびに `prisma generate` が必要 | postinstall スクリプトで自動化 |
| 5 | **バンドルサイズ** | サーバーレス環境ではクライアントサイズに注意 | Prisma Accelerate の利用 |

## 6. 他の ORM との比較

| 特徴 | Prisma | TypeORM | Drizzle | Sequelize |
|------|--------|---------|---------|-----------|
| **型安全性** | ◎ 自動生成 | △ デコレータ依存 | ◎ スキーマ推論 | ✕ 手動定義 |
| **学習曲線** | ○ 中程度 | △ やや急 | ○ 中程度 | ○ 中程度 |
| **スキーマ定義** | 独自言語（PSL） | TypeScript デコレータ | TypeScript オブジェクト | JavaScript オブジェクト |
| **マイグレーション** | ◎ 自動生成 | ○ 自動生成 | ○ 手動 + 自動 | ○ CLI で生成 |
| **生 SQL** | ○ `$queryRaw` | ○ `query()` | ◎ ネイティブ対応 | ○ `sequelize.query()` |
| **GUI ツール** | ◎ Prisma Studio | ✕ なし | ◎ Drizzle Studio | ✕ なし |
| **パフォーマンス** | ○ 良好 | △ やや重い | ◎ 軽量 | △ やや重い |
| **コミュニティ** | ◎ 活発 | ○ 成熟 | ○ 成長中 | ○ 成熟 |
| **npm DL/週** | 〜400万+ | 〜200万+ | 〜100万+ | 〜150万+ |

### 選定ガイド

| 使いたい場面 | 推奨 ORM |
|------------|---------|
| 型安全性を最重視 & 自動補完が欲しい | **Prisma** |
| SQL に近い書き方がしたい & 軽量さ重視 | **Drizzle** |
| Active Record パターン & デコレータが好き | **TypeORM** |
| レガシー JS プロジェクト | **Sequelize** |

## 7. Prisma のバージョンについて

### 本リポジトリで使用しているバージョン

| テンプレート | Prisma バージョン |
|------------|-----------------|
| next / nuxt / remix / sveltekit / vue / react / next-go | **^6.4.1** |
| next-python | **^5.8.0** |

### バージョンの確認方法

```bash
# インストール済みバージョンの確認
npx prisma --version

# 出力例:
# prisma                  : 6.4.1
# @prisma/client          : 6.4.1
# Computed binaryTarget    : debian-openssl-3.0.x
```

### バージョンアップ方法

```bash
# 最新バージョンに更新
npm install prisma@latest @prisma/client@latest

# Client を再生成（必ず実行）
npx prisma generate
```

> **⚠️ 重要:** `prisma` と `@prisma/client` のバージョンは**必ず一致させてください**。バージョンが異なるとエラーが発生します。

## 8. 本リポジトリでの利用状況

### テンプレート別の構成

| テンプレート | schema.prisma の場所 | Client の場所 | DB |
|------------|--------------------|--------------|----|
| `next/` | `src/lib/prisma/schema.prisma` | `src/lib/prisma/client.ts` | PostgreSQL |
| `nuxt/` | `lib/prisma/schema.prisma` | `lib/prisma/client.ts` | PostgreSQL |
| `remix/` | `prisma/schema.prisma` | - | PostgreSQL |
| `sveltekit/` | `prisma/schema.prisma` | - | PostgreSQL |
| `vue/` | `src/lib/prisma/schema.prisma` | `src/lib/prisma/client.ts` | PostgreSQL |
| `react/` | `src/lib/prisma/schema.prisma` | `src/lib/prisma/client.ts` | PostgreSQL |
| `next-go/next/` | `src/lib/prisma/schema.prisma` | `src/lib/prisma/client.ts` | PostgreSQL |

### 共通パターン

全テンプレートで以下のパターンを採用しています：

1. **DB**: PostgreSQL 16 (Alpine)
2. **接続方式**: Docker Compose の `db` サービスに接続
3. **Client の初期化**: グローバルシングルトンパターン（開発時のホットリロード対策）
4. **postinstall**: `prisma generate` を自動実行
