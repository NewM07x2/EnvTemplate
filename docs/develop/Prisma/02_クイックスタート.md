## 1. 全体の流れ

```
Step 1: プロジェクト準備（DB 起動）
  ↓
Step 2: Prisma インストール
  ↓
Step 3: Prisma 初期化（schema.prisma + .env 生成）
  ↓
Step 4: DATABASE_URL を設定
  ↓
Step 5: スキーマにモデルを定義
  ↓
Step 6: マイグレーション実行（DB にテーブル作成）
  ↓
Step 7: Prisma Client でデータ操作
  ↓
Step 8: Prisma Studio で確認
```

> **💡 既存テンプレートを使う場合:** 本リポジトリの `next/` や `remix/` などのテンプレートには Prisma がセットアップ済みです。その場合は [ローカル環境構築方法.md](./ローカル環境構築方法.md) を参照してください。

## 2. Step 1: プロジェクトの準備

### A. 既存テンプレートを使う場合（推奨）

```bash
# 例: Next.js テンプレートを使う場合
cd next
docker compose up -d    # PostgreSQL + アプリを起動
```

### B. ゼロから始める場合

```bash
# 新規プロジェクトを作成
mkdir my-app && cd my-app
npm init -y

# TypeScript のセットアップ
npm install -D typescript ts-node @types/node
npx tsc --init
```

PostgreSQL を Docker で起動します：

```yaml
# docker-compose.yml
version: "3.8"
services:
  db:
    image: postgres:16-alpine
    restart: always
    ports:
      - "5432:5432"
    environment:
      - POSTGRES_USER=postgres
      - POSTGRES_PASSWORD=postgres
      - POSTGRES_DB=myapp
    volumes:
      - postgres_data:/var/lib/postgresql/data

volumes:
  postgres_data:
```

```bash
docker compose up -d
```

## 3. Step 2: Prisma のインストール

```bash
# Prisma CLI（開発用）
npm install -D prisma

# Prisma Client（ランタイム用）
npm install @prisma/client
```

### インストールの確認

```bash
npx prisma --version
```

出力例：
```
prisma                  : 6.4.1
@prisma/client          : 6.4.1
Computed binaryTarget    : windows
```

> **⚠️ 注意:** `prisma` と `@prisma/client` のバージョンは一致している必要があります。

## 4. Step 3: Prisma の初期化

```bash
npx prisma init
```

以下の 2 つのファイルが生成されます：

```
my-app/
├── prisma/
│   └── schema.prisma   ← スキーマ定義ファイル（★ここを編集する）
├── .env                ← データベース接続情報
└── package.json
```

### 生成される schema.prisma の初期内容

```prisma
// This is your Prisma schema file,
// learn more about it in the docs: https://pris.ly/d/prisma-schema

generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}
```

## 5. Step 4: データベースの接続設定

`.env` ファイルを編集して、データベースの接続先を設定します。

### Docker Compose を使っている場合

```env
# .env
DATABASE_URL="postgresql://postgres:postgres@localhost:5432/myapp?schema=public"
```

### 接続文字列の構成

```
postgresql://ユーザー名:パスワード@ホスト:ポート/データベース名?schema=スキーマ名
```

| 部分 | 値 | 説明 |
|------|---|------|
| ユーザー名 | `postgres` | docker-compose.yml の `POSTGRES_USER` |
| パスワード | `postgres` | docker-compose.yml の `POSTGRES_PASSWORD` |
| ホスト | `localhost` | ホストマシンから接続する場合。コンテナ内なら `db` |
| ポート | `5432` | PostgreSQL のデフォルトポート |
| データベース名 | `myapp` | docker-compose.yml の `POSTGRES_DB` |
| スキーマ | `public` | PostgreSQL のデフォルトスキーマ |

### 接続テスト

```bash
npx prisma db pull
```

エラーが出なければ接続成功です。初回は「スキーマが空」というメッセージが出ますが正常です。

## 6. Step 5: スキーマの定義

`prisma/schema.prisma` にモデル（テーブル）を定義します。

### 最初のモデル定義例

```prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

// ユーザーテーブル
model User {
  id        Int      @id @default(autoincrement())  // 主キー（自動採番）
  email     String   @unique                        // メール（ユニーク）
  name      String?                                 // 名前（NULL 許容）
  age       Int                                     // 年齢
  createdAt DateTime @default(now())                // 作成日時（自動セット）
  updatedAt DateTime @updatedAt                     // 更新日時（自動更新）
  posts     Post[]                                  // Post との 1対多リレーション
}

// 投稿テーブル
model Post {
  id        Int      @id @default(autoincrement())
  title     String                                  // タイトル
  content   String?                                 // 本文（NULL 許容）
  published Boolean  @default(false)                // 公開フラグ
  authorId  Int                                     // 外部キー
  author    User     @relation(fields: [authorId], references: [id], onDelete: Cascade)
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}
```

### スキーマの読み方

```prisma
model User {
  id    Int    @id @default(autoincrement())
  //│    │      │         └── デフォルト値（自動採番）
  //│    │      └──────────── 主キー
  //│    └─────────────────── データ型（Int = 整数）
  //└──────────────────────── フィールド名（= カラム名）

  email String @unique
  //             └── ユニーク制約（重複不可）

  name  String?
  //          └── ? = NULL 許容（省略するとNOT NULL）
}
```

### よく使うデータ型

| Prisma 型 | PostgreSQL 型 | TypeScript 型 | 用途 |
|----------|--------------|--------------|------|
| `String` | `TEXT` | `string` | 文字列 |
| `Int` | `INTEGER` | `number` | 整数 |
| `Float` | `DOUBLE PRECISION` | `number` | 小数 |
| `Boolean` | `BOOLEAN` | `boolean` | 真偽値 |
| `DateTime` | `TIMESTAMP` | `Date` | 日時 |
| `Json` | `JSONB` | `JsonValue` | JSON データ |
| `BigInt` | `BIGINT` | `bigint` | 大きな整数 |
| `Decimal` | `DECIMAL` | `Decimal` | 高精度小数（金額等） |

### よく使うデコレータ

| デコレータ | 説明 | 例 |
|-----------|------|-----|
| `@id` | 主キーを指定 | `id Int @id` |
| `@default()` | デフォルト値 | `@default(now())`, `@default(false)` |
| `@unique` | ユニーク制約 | `email String @unique` |
| `@updatedAt` | 更新時自動セット | `updatedAt DateTime @updatedAt` |
| `@relation` | リレーション定義 | `@relation(fields: [authorId], references: [id])` |
| `@map()` | DB カラム名を指定 | `@map("created_at")` ← snake_case にマッピング |
| `@@map()` | DB テーブル名を指定 | `@@map("users")` ← テーブル名を小文字に |
| `@@index()` | インデックス作成 | `@@index([authorId, createdAt])` |
| `@@unique()` | 複合ユニーク | `@@unique([email, tenantId])` |

## 7. Step 6: マイグレーションの実行

スキーマを定義したら、マイグレーションで DB にテーブルを作成します。

```bash
npx prisma migrate dev --name init
```

### 実行結果

```
Environment variables loaded from .env
Prisma schema loaded from prisma/schema.prisma
Datasource "db": PostgreSQL database "myapp", schema "public" at "localhost:5432"

Applying migration `20260301000000_init`

The following migration(s) have been created and applied from new schema changes:

migrations/
  └─ 20260301000000_init/
    └─ migration.sql    ← 自動生成された SQL

Your database is now in sync with your schema.

✔ Generated Prisma Client
```

### 生成されるマイグレーション SQL（例）

```sql
-- CreateTable
CREATE TABLE "User" (
    "id" SERIAL NOT NULL,
    "email" TEXT NOT NULL,
    "name" TEXT,
    "age" INTEGER NOT NULL,
    "createdAt" TIMESTAMP(3) NOT NULL DEFAULT CURRENT_TIMESTAMP,
    "updatedAt" TIMESTAMP(3) NOT NULL,

    CONSTRAINT "User_pkey" PRIMARY KEY ("id")
);

-- CreateTable
CREATE TABLE "Post" (
    "id" SERIAL NOT NULL,
    "title" TEXT NOT NULL,
    "content" TEXT,
    "published" BOOLEAN NOT NULL DEFAULT false,
    "authorId" INTEGER NOT NULL,
    "createdAt" TIMESTAMP(3) NOT NULL DEFAULT CURRENT_TIMESTAMP,
    "updatedAt" TIMESTAMP(3) NOT NULL,

    CONSTRAINT "Post_pkey" PRIMARY KEY ("id")
);

-- CreateIndex
CREATE UNIQUE INDEX "User_email_key" ON "User"("email");

-- AddForeignKey
ALTER TABLE "Post" ADD CONSTRAINT "Post_authorId_fkey"
  FOREIGN KEY ("authorId") REFERENCES "User"("id")
  ON DELETE CASCADE ON UPDATE CASCADE;
```

### マイグレーション後のディレクトリ構造

```
prisma/
├── schema.prisma
└── migrations/
    ├── 20260301000000_init/
    │   └── migration.sql
    └── migration_lock.toml
```

## 8. Step 7: Prisma Client の使用

### Client の初期化

```typescript
// src/lib/prisma/client.ts
import { PrismaClient } from '@prisma/client'

const globalForPrisma = global as unknown as { prisma: PrismaClient }

export const prisma =
  globalForPrisma.prisma ||
  new PrismaClient({
    log: process.env.NODE_ENV === 'development'
      ? ['query', 'error', 'warn']
      : ['error'],
  })

if (process.env.NODE_ENV !== 'production') globalForPrisma.prisma = prisma
```

> **💡 なぜグローバル変数にキャッシュするのか？**  
> Next.js 等の開発サーバーはホットリロードのたびにモジュールを再読み込みします。グローバルにキャッシュしないと DB 接続が無限に増えてしまいます。

### 最初のデータ操作

```typescript
import { prisma } from '@/lib/prisma/client'

// ユーザーを作成
const user = await prisma.user.create({
  data: {
    email: 'alice@example.com',
    name: 'Alice',
    age: 25,
  },
})
console.log('作成されたユーザー:', user)
// { id: 1, email: 'alice@example.com', name: 'Alice', age: 25, createdAt: ..., updatedAt: ... }

// 全ユーザーを取得
const users = await prisma.user.findMany()
console.log('全ユーザー:', users)

// ID で 1 件取得
const found = await prisma.user.findUnique({
  where: { id: 1 },
})
console.log('ID=1 のユーザー:', found)
```

### よくある確認コマンド

```bash
# DB の状態を確認
npx prisma migrate status

# スキーマと DB の差分を確認
npx prisma migrate diff --from-schema-datamodel prisma/schema.prisma --to-schema-datasource prisma/schema.prisma
```

```bash

npx prisma db push                        # スキーマを直接 DB に反映（マイグレーションなし）
npx prisma db pull                        # DB からスキーマを逆生成
```
