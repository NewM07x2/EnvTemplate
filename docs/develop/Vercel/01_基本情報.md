## 1. Vercelとは

**Vercel** は、フロントエンド開発者向けのクラウドホスティング・デプロイメントプラットフォームです。

### 1.1 概要

- **開発企業**: Vercel, Inc.（創業: 2015年）
- **主な出身背景**: Next.js の生みの親（Vercel が開発・保守）
- **コンセプト**: 「フロントエンド開発を高速化・簡素化」
- **グローバル展開**: 世界150+ ヶ国でホスティング
- **エッジコンピューティング**: Vercel Edge Network による低遅延配信

### 1.2 実は何をやってくれるのか？

| やってくれること | 説明 |
|---|---|
| **ホスティング** | 静的ファイル・API の配信 |
| **ビルド** | GitHub push 時に自動コンパイル |
| **デプロイ** | 本番環境への自動・手動リリース |
| **CDN** | グローバル配信ネットワーク |
| **API Routes** | Node.js サーバーレスレスの提供 |
| **データベース連携** | PostgreSQL（Vercel Storage）等 |
| **環境管理** | 本番・ステージング・プレビュー環境 |
| **監視・分析** | デプロイ履歴・エラー監視・パフォーマンス分析 |

## 2. 主な特徴

### 2.1 フロントエンド最適化

✅ **Next.js との統合**  
- Next.js の公式ホスティング（最適なパフォーマンス）
- ISR（Incremental Static Regeneration）対応
- Image Optimization 統合

✅ **静的生成 (SSG)**  
- ビルド時に HTML を事前生成 → 高速配信
- CDN キャッシュ効率が最高

✅ **サーバーレス関数**  
- API Routes で Node.js コード実行可能
- 自動スケーリング

### 2.2 開発体験の高さ

✅ **Git 統合**  
- GitHub/GitLab/Bitbucket と連携
- Push するだけで自動デプロイ

✅ **プレビュー環境**  
- PR ごとに本番環境と同じ環境を自動生成
- レビュー時に実際の動作確認可能

✅ **ローカルテスト**  
- `vercel dev` コマンドで本番同等の環境をローカルで再現
- ホットリロード対応

### 2.3 グローバル配信

✅ **Edge Network**  
- 280+ 都市のサーバーから自動配信
- ユーザー位置に応じた最速経路

✅ **自動最適化**  
- 画像フォーマット自動変換（WebP 等）
- CSS/JS の自動圧縮

### 2.4 スケーラビリティ

✅ **自動スケーリング**  
- トラフィック増加に自動対応
- 管理者は何もしなくて OK

✅ **無制限の帯域幅**  
- Pro・Enterprise プランでは転送量制限なし

## 3. 料金プラン

### 3.1 プラン比較表

| 項目 | Free | Pro | Enterprise |
|---|---|---|---|
| **月額** | 無料 | $20 | 要相談 |
| **デプロイ数** | 無制限 | 無制限 | 無制限 |
| **チーム規模** | 1人 | 無制限 | 無制限 |
| **CDN** | ✅ グローバル | ✅ グローバル | ✅ プライベート可 |
| **サーバーレス関数** | 100,000 req/月 | 無制限 | 無制限 |
| **Vercel Postgres** | 試験的 | $15/月 | ✅ |
| **Vercel KV (Redis)** | 試験的 | $10/月 | ✅ |
| **Analytics** | 基本 | ✅ 詳細 | ✅ 詳細 |
| **SLA** | なし | 99.95% | 99.99% |
| **優先サポート** | ❌ | 📧 メール | ☎️ 専任 |

### 3.2 Free プランの制限

```
・サーバーレス関数: 100,000リクエスト/月（超過後は 429 Too Many Requests）
・Database接続: 試験的（制限あり）
・CLS（コア Web Vitals）: 監視対象外
・SLA: なし（ダウンタイムに補償なし）
```

### 3.3 課金のポイント

```
✅ 含まれるもの:
- CDN帯域幅（無制限）
- デプロイ数（無制限）
- チームメンバー数（Pro以上）

❌ 別途課金:
- Vercel Postgres（$15/月から）
- Vercel KV（$10/月から）
- Analytics（Pro）
- Edge Middleware
- ドメイン登録・SSL証明書（通常は無料）
```

## 4. 対応フレームワーク

### 4.1 公式サポート・推奨

| フレームワーク | サポート度 | 推奨度 |
|---|---|---|
| **Next.js** | ⭐⭐⭐⭐⭐ | 最高推奨 |
| **React** | ⭐⭐⭐⭐⭐ | 推奨 |
| **Nuxt** | ⭐⭐⭐⭐ | 推奨 |
| **Svelte / SvelteKit** | ⭐⭐⭐⭐ | 推奨 |
| **Vue** | ⭐⭐⭐⭐ | 推奨 |
| **Remix** | ⭐⭐⭐⭐ | 推奨 |
| **Astro** | ⭐⭐⭐⭐ | 推奨 |
| **Static Site Generators** | ⭐⭐⭐⭐ | 対応 |
| **Express / Node.js** | ⭐⭐⭐ | 要カスタマイズ |
| **Python / Flask** | ⭐⭐ | 限定対応 |
| **PHP / Laravel** | ⭐ | 非推奨 |

### 4.2 フレームワーク別の利用シーン

```
Next.js：最適。SSR/ISR/Image最適化が全て組み込み
React：SPA構成で十分ならOK。ただしSSRはNext.jsが便利
SvelteKit：Vercel公式サポート。パフォーマンス優先なら推奨
Remix：サーバー機能が必要ならおすすめ
Astro：静的生成が主なら高速。ハイブリッド構成も対応
```

## 5. コア概念

### 5.1 デプロイメント

```
┌─────────────┐
│ GitHub等に  │
│   Push      │
└──────┬──────┘
       │
       ▼
┌─────────────────────┐
│ Vercelが自動検知    │
│ ビルドを開始        │
└──────┬──────────────┘
       │
       ▼
┌─────────────────────┐
│ ビルド完了          │
│ CDNに配置           │
└──────┬──────────────┘
       │
       ▼
┌─────────────────────┐
│ 本番リリース        │
│ または Preview URL  │
└─────────────────────┘
```

### 5.2 環境（Environments）

| 環境 | 用途 | トリガー |
|---|---|---|
| **Preview** | PR・Feature Branch 検証 | 自動生成（PR毎） |
| **Staging** | テスト環境 | 手動デプロイまたは staging ブランチ |
| **Production** | 本番環境 | main ブランチへのマージ |

### 5.3 Edge Network（エッジネットワーク）

```
ユーザー → 最寄りの Edge Server → オリジンサーバー
          （キャッシュ・最適化）
```

- **280+ 都市** にエッジサーバー配置
- **自動キャッシュ** HTML/CSS/JS/画像
- **地理的最適化** ユーザー位置に応じた最速配信

### 6.2 選択基準

```
✅ Vercel がおすすめ:
- Next.js / React SPA を使っている
- 開発スピード重視
- 小〜中規模プロジェクト
- 簡単なセットアップ希望

✅ Netlify がおすすめ:
- Vercel との差は小さい
- 価格重視
- 静的サイト主体

✅ AWS Amplify がおすすめ:
- AWS エコシステム活用
- バックエンド処理が複雑
- 大規模スケーラビリティ

✅ Firebase Hosting がおすすめ:
- Firebase（Realtime DB / Firestore）を使う
- Google エコシステム活用
```

## 9. デプロイの流れ

### 9.1 最小構成でのデプロイ

```
1. GitHub に リポジトリ作成
2. Vercel にサインアップ（GitHub連携）
3. プロジェクトをインポート
4. 環境変数設定（必要に応じ）
```

## 11. よくある質問

### Q1: Vercel は本当に無料で使える？

**A**: Free プラン は完全無料です。ただし：
- サーバーレス関数: 100,000 req/月 制限
- SLA（ダウンタイム保証）なし
- チームメンバーは1人のみ

小〜中規模個人プロジェクトなら十分です。

### Q2: 既存プロジェクトをVercelに移行できる？

**A**: ほぼ全ての JavaScript / TypeScript フレームワークに対応。
- Next.js / React / Vue / Svelte ⭐ すぐできる
- Express / Node.js 要カスタマイズ（API Routes で対応）
- PHP / Python △ 限定対応

### Q3: データベースはどうする？

**A**: 複数の選択肢：
- **Vercel Postgres** $15/月〜（最も連携が簡単）
- **Vercel KV** $10/月〜（Redis互換）
- **外部DB** MongoDB / Supabase / Firebase 等 自由に選択

### Q4: デプロイ本当に自動なの？

**A**: はい。GitHub push → 自動ビルド・デプロイ。
- PR ごとに Preview URL 自動生成
- main へのマージ → 本番自動リリース

### Q5: セキュリティは大丈夫？

**A**: Vercel は SOC 2 Type II 準拠。
- SSL/TLS 自動
- DDoS 対策有
- 企業向け監査ログ

ただし Free プラン は SLA なし（ダウンタイム補償なし）。
