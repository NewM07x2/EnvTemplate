## 1. プラットフォーム比較

### 1.1 主要プラットフォーム比較表

| | **Vercel** | **Netlify** | **Heroku** | **Firebase** | **AWS** |
|---|---|---|---|---|---|
| **推奨用途** | Next.js | SPA/静的 | Node.js API | モバイル | エンタープライズ |
| **Next.js サポート** | ⭐⭐⭐⭐⭐ | ⭐⭐ | ⭐⭐ | ✗ | ⭐⭐⭐ |
| **スケーリング** | 自動 | 自動 | 手動 | 自動 | 手動 |
| **データベース** | Postgres/KV | ✗ | PostgreSQL | Firestore | RDS他 |
| **料金** | 低 | 低 | 中 | 低（小） | 高（柔軟） |
| **セットアップ** | 簡単 | 簡単 | 中程度 | 簡単 | 複雑 |
| **カスタマイズ** | 限定的 | 限定的 | 高 | 限定的 | 無制限 |

## 2. Netlify から移行

### 2.1 移行前チェック

```
□ netlify.toml 確認
□ 環境変数一覧表示
□ ビルドコマンド確認
□ リダイレクト・ルーティング確認
□ 関数（AWS Lambda）がある場合は API ルートに変換
```

### 2.2 netlify.toml → vercel.json 変換

**Netlify:**

```toml
# netlify.toml
[build]
command = "npm run build"
publish = "dist"

[[redirects]]
from = "/old"
to = "/new"
status = 301

[[headers]]
for = "/*"
[headers.values]
X-Frame-Options = "DENY"
```

**Vercel:**

```json
// vercel.json
{
  "version": 2,
  "buildCommand": "npm run build",
  "outputDirectory": "dist",
  "routes": [
    {
      "src": "/old",
      "status": 301,
      "headers": {
        "Location": "/new"
      }
    }
  ],
  "headers": [
    {
      "source": "/(.*)",
      "headers": [
        {
          "key": "X-Frame-Options",
          "value": "DENY"
        }
      ]
    }
  ]
}
```

### 2.3 Netlify Functions → Vercel API Routes

**Netlify:**

```javascript
// netlify/functions/hello.js
exports.handler = async (event) => {
  return {
    statusCode: 200,
    body: JSON.stringify({ message: 'Hello' }),
  };
};
```

**Vercel:**

```typescript
// pages/api/hello.ts
import type { NextApiRequest, NextApiResponse } from 'next';

export default function handler(req: NextApiRequest, res: NextApiResponse) {
  res.status(200).json({ message: 'Hello' });
}
```

### 2.4 環境変数移行

```bash
# 1. Netlify から環境変数エクスポート
# Netlify UI > Site settings > Build & deploy > Environment

# 2. Vercel に追加
vercel env add VAR_NAME "value"

# または Vercel UI > Settings > Environment Variables
```

### 2.5 ドメイン移行

**Netlify ドメイン:**

```
Project > Settings > Domain management
```

DNS 設定を Vercel に指す。

**Vercel:**

```
Project > Settings > Domains > Add Domain
```

## 3. Heroku から移行

### 3.1 移行前チェック

```
□ Procfile / runtime.txt 確認
□ Add-ons (PostgreSQL / Redis) 確認
□ 環境変数エクスポート
□ ビルドパック確認
```

### 3.2 Procfile → vercel.json

**Heroku:**

```
# Procfile
web: node server.js
worker: node worker.js
```

**Vercel:**

```json
// vercel.json
{
  "version": 2,
  "builds": [
    {
      "src": "server.js",
      "use": "@vercel/node"
    }
  ],
  "routes": [
    {
      "src": "/(.*)",
      "dest": "server.js"
    }
  ]
}
```

### 3.3 ポートバインディング

**Heroku:**

```javascript
// server.js
const port = process.env.PORT || 3000;
app.listen(port);
```

**Vercel:**

```javascript
// Vercel は PORT を自動設定
const port = process.env.PORT || 3000;
app.listen(port);
// 同じコード動作 ✅
```

### 3.4 Add-ons 移行

**Heroku Postgres → Vercel Postgres:**

```bash
# 1. Heroku から DB バックアップ
pg_dump $DATABASE_URL > backup.sql

# 2. Vercel Postgres 作成
# Project > Storage > Create > Postgres

# 3. リストア
psql $POSTGRES_URL < backup.sql
```

**Heroku Redis → Vercel KV:**

```bash
# 1. Heroku Redis から export
redis-cli -u $REDIS_URL KEYS '*' > keys.txt

# 2. Vercel KV に移行
# 手動再構築 (Redis 互換なので同じコードで動作)
```

### 3.5 環境変数移行

```bash
# 1. Heroku から取得
heroku config -a my-app

# 2. Vercel に追加
for var in $(heroku config -a my-app | cut -d: -f1 | tail -n +1); do
  value=$(heroku config:get $var -a my-app)
  vercel env add $var "$value"
done
```

## 4. Firebase から移行

### 4.1 Firebase Hosting → Vercel

**Firebase から出力:**

```bash
# firebase.json → vercel.json に変換
firebase.json
{
  "hosting": {
    "public": "dist",
    "cleanUrls": true,
    "rewrites": [
      {
        "source": "**",
        "destination": "/index.html"
      }
    ]
  }
}
```

**Vercel 設定:**

```json
// vercel.json
{
  "version": 2,
  "buildCommand": "npm run build",
  "outputDirectory": "dist",
  "routes": [
    {
      "src": "/(.*)",
      "dest": "/index.html"
    }
  ]
}
```

### 4.2 Cloud Functions → Vercel API Routes

**Firebase:**

```typescript
// functions/index.ts
import * as functions from 'firebase-functions';

export const hello = functions.https.onRequest((req, res) => {
  res.json({ message: 'Hello' });
});
```

**Vercel:**

```typescript
// pages/api/hello.ts
export default function handler(req, res) {
  res.json({ message: 'Hello' });
}
```

### 4.3 Firestore → Vercel Postgres

**Firebase Firestore:**

```typescript
// Firebase SDK
const doc = await db.collection('users').doc('user1').get();
```

**Vercel Postgres:**

```typescript
// Vercel Postgres
const result = await sql`SELECT * FROM users WHERE id = $1`, ['user1'];
```

**マイグレーション:**

```bash
# 1. Firestore export
firebase firestore:export ./export

# 2. Postgres にインポート
# JSON → SQL 変換スクリプト実行

# 3. 検証
vercel env pull
psql $POSTGRES_URL -c "SELECT COUNT(*) FROM users;"
```

### 4.4 Realtime Database → Vercel KV

```typescript
// Firebase Realtime Database
const ref = db.ref('messages');
ref.on('value', (snapshot) => {
  console.log(snapshot.val());
});

// Vercel KV (Redis)
import { kv } from '@vercel/kv';

const messages = await kv.lrange('messages', 0, -1);
```

## 5. AWS から移行

### 5.1 EC2 / ECS → Vercel

**AWS 設定:**

```
EC2 インスタンス
↓
+ Load Balancer
↓
+ Auto Scaling
↓
+ RDS (PostgreSQL)
```

**Vercel 簡素化:**

```
Vercel API Routes
↓ (自動スケーリング)
↓ (グローバル CDN)
↓
Vercel Postgres
```

### 5.2 S3 → Vercel Static Assets

**AWS S3:**

```typescript
// S3 にアップロード
s3.putObject({
  Bucket: 'my-bucket',
  Key: 'images/photo.jpg',
  Body: fileBuffer,
});
```

**Vercel:**

```typescript
// public/ フォルダに配置
// /public/images/photo.jpg
// ↓
// https://my-app.vercel.app/images/photo.jpg
```

### 5.3 Lambda → Vercel API Routes

**AWS Lambda:**

```python
# lambda_function.py
def lambda_handler(event, context):
    return {
        'statusCode': 200,
        'body': json.dumps({'message': 'Hello'})
    }
```

**Vercel:**

```typescript
// pages/api/hello.ts
export default function handler(req, res) {
  res.json({ message: 'Hello' });
}
```

### 5.4 RDS → Vercel Postgres

```bash
# 1. RDS からダンプ
pg_dump -h my-rds-instance.amazonaws.com \
  -U admin \
  -d mydatabase \
  > backup.sql

# 2. Vercel にリストア
psql $POSTGRES_URL < backup.sql
```

## 6. 共通移行手順

### 6.1 段階的移行プラン

**Phase 1: 準備（1 週間）**

```
□ 移行対象タスク定義
□ 環境構築
□ ローカルテスト
□ ドキュメント作成
□ チーム研修
```

**Phase 2: 開発環境移行（3 日）**

```
□ リポジトリ設定変更
□ vercel.json 作成
□ ローカルで vercel dev テスト
□ API ルート変換
□ CI/CD パイプライン構築
```

**Phase 3: ステージング検証（3 日）**

```
□ Preview 環境デプロイ
□ エンドツーエンドテスト
□ パフォーマンス計測
□ セキュリティチェック
□ ユーザー受け入れテスト
```

**Phase 4: 本番環境移行（1 日）**

```
□ ドメイン切り替え
□ トラフィック監視
□ インシデント対応体制
□ ロールバック準備
```

**Phase 5: クリーンアップ（1 週間）**

```
□ 旧プラットフォーム リソース削除
□ ドキュメント更新
□ チーム研修
□ ポストモーテム
```

## 7. ダウンタイムなし移行

### 7.1 Blue-Green デプロイ

```
1. 旧プラットフォーム (Blue) 稼働

2. 新プラットフォーム (Green) デプロイ・検証

3. トラフィック 10% を Green へ

4. 24 時間監視

5. トラフィック 50% → 100% 段階的切り替え

6. 旧プラットフォーム停止
```

### 7.2 DNS TTL 管理

```bash
# 移行 1 週間前：TTL を短縮
# 例：3600秒（1時間）→ 60秒（1分）

vercel DNS console で設定

# 移行後：TTL を復元
# 60秒 → 3600秒
```

### 7.3 Vercel ロールバック計画

```bash
# 1. 旧環境を一時保持

# 2. 迅速なロールバック手順文書化
# - DNS 再度指定
# - データベース復旧スクリプト
# - 通知リスト

# 3. ロールバック時間目標：15分以内
```

## 8. トラブルシューティング

### よくある移行問題

| 問題 | 原因 | 解決策 |
|---|---|---|
| ビルド失敗 | ビルドコマンド異なる | vercel.json buildCommand 確認 |
| 404 エラー | ルーティング設定ミス | routes 設定確認 |
| 環境変数なし | 未設定・プレフィックス忘れ | vercel env list で確認 |
| DB 接続できない | 接続文字列形式異なる | 環境変数再確認 |
| DNS 遅延 | TTL 値高い / キャッシュ | TTL 低く / キャッシュクリア |
| パフォーマンス低下 | リージョン選択 | Vercel リージョン設定確認 |

### デバッグコマンド

```bash
# ビルドのテスト
vercel build

# 環境確認
vercel env list
vercel project inspect

# ログ確認
vercel logs --follow

# ローカル再現
vercel dev

# DNS 確認
nslookup my-app.vercel.app
```
