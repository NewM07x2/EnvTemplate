## 1. アカウント作成・GitHub連携

### 1.1 Vercel にサインアップ

```
1. https://vercel.com にアクセス
2. 「Sign Up」をクリック
3. GitHub でサインイン（または Email）
4. 許可を与える
   ↓
Vercel ダッシュボード にアクセス可能 ✅
```

### 1.2 GitHub リポジトリを作成

```bash
# GitHub に新規リポジトリ作成（Web UI）
# または コマンドラインで：

mkdir my-nextjs-app
cd my-nextjs-app
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/my-nextjs-app.git
git push -u origin main
```

## 2. ローカル環境構築

### 2.1 Node.js インストール確認

```bash
node --version
# v18.17.0 以上推奨

npm --version
# 9.0.0 以上推奨
```

### 2.2 新規プロジェクトを作成（Next.js の場合）

```bash
# 最新版 Next.js プロジェクトを生成
npx create-next-app@latest my-nextjs-app --typescript

# プロンプト回答例：
# ✔ Would you like to use TypeScript? › Yes
# ✔ Would you like to use ESLint? › Yes
# ✔ Would you like to use Tailwind CSS? › Yes
# ✔ Would you like to use `src/` directory? › Yes
# ✔ Would you like to use App Router? › Yes
# ✔ Would you like to customize the import alias? › No

cd my-nextjs-app
```

### 2.3 ローカルで動作確認

```bash
npm run dev

# 出力:
# ▲ Next.js 14.x.x
# - Local: http://localhost:3000
# 
# ✓ Ready in 2.5s
```

ブラウザで http://localhost:3000 にアクセス → Welcome to Next.js が表示される ✅

### 2.4 プロジェクト構造

```
my-nextjs-app/
├── .next/                 # ビルド出力（.gitignore）
├── node_modules/          # 依存ライブラリ（.gitignore）
├── public/                # 静的ファイル（画像等）
├── src/
│   ├── app/
│   │   ├── layout.tsx     # ルートレイアウト
│   │   ├── page.tsx       # / ページ
│   │   └── api/           # API Routes
│   └── components/        # コンポーネント
├── .eslintrc.json         # ESLint 設定
├── .gitignore
├── next.config.js         # Next.js 設定
├── package.json
├── package-lock.json
├── tsconfig.json          # TypeScript 設定
└── README.md
```

## 3. 既存プロジェクトのインポート

### 3.1 Vercel Web UI からインポート

```
1. Vercel ダッシュボード にアクセス
   https://vercel.com/dashboard

2. 「Add New」→「Project」をクリック

3. 「Import Git Repository」

4. リポジトリを選択
   ↓
5. 「Import」をクリック
   ↓
6. デプロイが自動開始
   ↓
数秒～数十秒で本番リリース ✅
```

### 3.2 Vercel CLI からインポート

```bash
# Vercel CLI をインストール
npm i -g vercel

# ログイン
vercel login

# プロジェクトをリンク・デプロイ
cd my-project
vercel

# 対話的に質問に回答：
# ? Set up and deploy "~/my-project"? [Y/n] y
# ? Which scope do you want to deploy to? (username)
# ? Link to existing project? [y/N] n
# ? What's your project's name? my-project
# ? In which directory is your code located? ./
# ? Want to modify these settings? [y/N] n
#
# ✅ Production: https://my-project.vercel.app [v1]
```

## 4. Vercel CLI での開発

### 4.1 Vercel CLI のインストール

```bash
# グローバルインストール
npm install -g vercel

# バージョン確認
vercel --version
```

### 4.2 ローカル開発サーバー起動

```bash
# Vercel が本番環境と同じ設定でローカルサーバー起動
vercel dev

# 出力:
# Vercel CLI v33.x.x
# ⏳ Building for local development...
# ✓ Ready! Available at http://localhost:3000
```

**`vercel dev` の利点**:
- 本番環境と同じ環境で動作確認
- API Routes がそのまま動作
- 環境変数を本番から取得（テスト可能）
- Edge Middleware をテスト可能

### 4.3 `npm run dev` との違い

| 観点 | `npm run dev` | `vercel dev` |
|---|---|---|
| 速度 | 高速（ホットリロード） | やや遅い |
| 本番再現 | 部分的 | 完全再現 |
| 環境変数 | ローカルのみ | 本番から取得可 |
| API Routes | ローカル実行 | 本番同等実行 |
| Edge Middleware | ❌ 未対応 | ✅ 対応 |
| 推奨用途 | 開発速度重視 | 本番前確認 |

**推奨の使い分け**:
```bash
# 開発中：高速リロード重視
npm run dev

# 本番前確認：本番環境再現
vercel dev
```

## 6. 環境変数の設定

### 6.1 ローカルで環境変数を使う

```bash
# .env.local を作成（Git に含めない）
touch .env.local

# 環境変数を記入
# .env.local
NEXT_PUBLIC_API_URL=http://localhost:3000
DATABASE_URL=postgresql://...
API_SECRET_KEY=secret123
```

```typescript
// src/app/page.tsx
export default function Home() {
  const apiUrl = process.env.NEXT_PUBLIC_API_URL;
  // ⚠️ NEXT_PUBLIC_ プレフィックス = ブラウザから アクセス可能（秘密にできない）
  // DATABASE_URL は NEXT_PUBLIC_ なし = サーバー側のみアクセス可（秘密情報OK）

  return <div>API URL: {apiUrl}</div>;
}
```

### 6.2 本番環境（Vercel）に環境変数を設定

```
1. Vercel ダッシュボード → プロジェクト選択
2. 「Settings」タブ → 「Environment Variables」
3. 「Add New」で環境変数追加
4. 例：
   - Key: DATABASE_URL
   - Value: postgresql://...
   - Environment: Production
5. 「Save」
6. デプロイを再実行
```

### 6.3 環境変数を CLI で設定

```bash
# 本番環境に環境変数を設定
vercel env add DATABASE_URL

# プロンプト: Value for DATABASE_URL?
# → postgresql://... を入力

# Environment を選択（複数選択可）
# ✓ Production
# ✓ Preview
# ○ Development
```

### 6.4 環境変数の確認

```bash
# 環境変数一覧を確認
vercel env list

# 出力:
# Environment Variable Name      Environments
# DATABASE_URL                   production, preview
# API_SECRET_KEY                 production
```

## 7. カスタムドメイン設定

### 7.1 デフォルト URL

```
自動生成: https://my-nextjs-app.vercel.app
```

### 7.2 カスタムドメインを追加

```
1. Vercel ダッシュボード → 「Settings」→「Domains」
2. 「Add Domain」をクリック
3. ドメイン名を入力（例：myapp.com）
4. 「Add」をクリック
5. DNS 設定を指示される
   ↓
6. ドメインのレジストラ（お名前.com / Route 53 等）で DNS を設定
   ↓
7. 自動検証（数分～数時間）
   ↓
本番環境が https://myapp.com でアクセス可能 ✅
```

### 7.3 DNS 設定例（Route 53 の場合）

```
Vercel が示すレコード（例）:
Name: myapp.com
Type: CNAME
Value: cname.vercel-dns.com

または A レコード:
Name: myapp.com
Type: A
Value: 76.76.19.165
```

## 8. デプロイロールバック

### 8.1 前のバージョンに戻す

```
1. Vercel ダッシュボード → 「Deployments」
2. 戻したいバージョンを選択
3. 「...」メニュー → 「Promote to Production」
4. 確認して「Promote」
   ↓
本番環境がロールバック ✅
```

### 8.2 CLI でロールバック

```bash
# デプロイ一覧を確認
vercel list

# 戻したいバージョンを指定
vercel promote [DEPLOYMENT_ID]

# 例:
vercel promote qjk25zkka23k2k3k
```
