## 1. Vercel Postgres

### 1.1 Vercel Postgres について

**Vercel Postgres** は、Vercel が提供する完全マネージド PostgreSQL ホスティング。

| 特徴 | 説明 |
|---|---|
| **自動スケーリング** | トラフィック増加時に自動調整 |
| **バックアップ** | 日次・自動 |
| **セキュリティ** | 暗号化・SSL デフォルト |
| **料金** | 従量課金（初月無料） |
| **リージョン** | Vercel と同じリージョン |

### 1.2 Vercel Postgres セットアップ

#### ステップ 1: プロジェクトに追加

```
Project > Storage > Databases > Create > Postgres
```

ダイアログで確認・作成。

#### ステップ 2: 接続文字列を確認

```
Storage > [Database Name] > .env.local
```

以下が自動生成：

```
POSTGRES_URL_NON_POOLING=postgresql://user:password@host:5432/db
POSTGRES_URL=postgresql://user:password@host:5432/db?sslmode=require
```

#### ステップ 3: ローカル `.env.local` に追加

```bash
# .env.local
POSTGRES_URL=postgresql://user:password@host/db
```

### 1.3 Node.js から接続

```bash
npm install @vercel/postgres
```

```javascript
// pages/api/db.js
import { sql } from '@vercel/postgres';

export default async function handler(req, res) {
  try {
    // クエリ実行
    const { rows } = await sql`
      SELECT * FROM users WHERE id = ${req.query.id}
    `;
    res.json(rows);
  } catch (error) {
    res.status(500).json({ error: error.message });
  }
}
```

### 1.4 Next.js 固有方法

```typescript
// app/api/users/route.ts
import { sql } from '@vercel/postgres';

export async function GET(request: Request) {
  try {
    const result = await sql`SELECT * FROM users LIMIT 10`;
    return Response.json(result.rows);
  } catch (error) {
    return Response.json({ error }, { status: 500 });
  }
}
```

### 1.5 テーブル初期化（マイグレーション）

```javascript
// scripts/init-db.js
import { sql } from '@vercel/postgres';

async function initDB() {
  try {
    // テーブル作成
    await sql`
      CREATE TABLE IF NOT EXISTS users (
        id SERIAL PRIMARY KEY,
        name VARCHAR(100) NOT NULL,
        email VARCHAR(100) UNIQUE NOT NULL,
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
      );
    `;
    console.log('✅ テーブルを作成しました');
  } catch (error) {
    console.error('❌ エラー:', error.message);
  }
}

initDB();
```

```bash
# 実行
node scripts/init-db.js
```

## 2. Vercel KV (Redis)

### 2.1 Vercel KV について

**Vercel KV** は Redis 互換のキー・バリュー ストア。キャッシュやセッション管理に最適。

| 用途 | 説明 |
|---|---|
| **キャッシュ** | API 応答キャッシュ |
| **セッション** | ユーザーセッション保存 |
| **レート制限** | API 呼び出し制限 |
| **リアルタイムデータ** | WebSocket メッセージ |

### 2.2 Vercel KV セットアップ

#### ステップ 1: 追加

```
Project > Storage > KV > Create
```

#### ステップ 2: 接続文字列確認

```
Storage > [KV Name] > .env.local
```

```
KV_URL=redis://:password@host:port
KV_REST_API_URL=https://host.kv.vercel.app
KV_REST_API_TOKEN=token
```

#### ステップ 3: SDK インストール

```bash
npm install @vercel/kv
```

### 2.3 基本操作

```javascript
// pages/api/cache.js
import { kv } from '@vercel/kv';

export default async function handler(req, res) {
  // 値を設定（TTL 1時間）
  await kv.set('user:123', { name: 'John' }, { ex: 3600 });

  // 値を取得
  const user = await kv.get('user:123');

  // 値を削除
  await kv.del('user:123');

  res.json({ user });
}
```

### 2.4 キャッシュ戦略の例

```javascript
// lib/cache.js
import { kv } from '@vercel/kv';

export async function getCachedData(key, fetchFn, ttl = 3600) {
  // キャッシュを確認
  const cached = await kv.get(key);
  if (cached) return cached;

  // キャッシュなければ取得
  const data = await fetchFn();

  // TTL 付きで保存
  await kv.set(key, data, { ex: ttl });

  return data;
}
```

```javascript
// pages/api/products.js
import { getCachedData } from '@/lib/cache';

export default async function handler(req, res) {
  const products = await getCachedData(
    'products:all',
    async () => {
      const response = await fetch('https://api.example.com/products');
      return response.json();
    },
    3600 // 1時間キャッシュ
  );

  res.json(products);
}
```

## 3. 外部データベース接続

### 3.1 MongoDB 接続

```bash
npm install mongodb
```

```javascript
// lib/mongodb.js
import { MongoClient } from 'mongodb';

const uri = process.env.MONGODB_URI;
const client = new MongoClient(uri);

export async function connectToDatabase() {
  if (!client.connected) {
    await client.connect();
  }
  return client.db('myapp');
}
```

```javascript
// pages/api/users.js
import { connectToDatabase } from '@/lib/mongodb';

export default async function handler(req, res) {
  const db = await connectToDatabase();
  const users = await db.collection('users').find({}).toArray();
  res.json(users);
}
```

### 3.2 Supabase 接続

```bash
npm install @supabase/supabase-js
```

```javascript
// lib/supabase.js
import { createClient } from '@supabase/supabase-js';

export const supabase = createClient(
  process.env.NEXT_PUBLIC_SUPABASE_URL,
  process.env.SUPABASE_SERVICE_ROLE_KEY
);
```

```javascript
// pages/api/todos.js
import { supabase } from '@/lib/supabase';

export default async function handler(req, res) {
  const { data, error } = await supabase
    .from('todos')
    .select('*')
    .eq('user_id', req.query.userId);

  if (error) return res.status(500).json(error);
  res.json(data);
}
```

### 3.3 Firebase Firestore 接続

```bash
npm install firebase-admin
```

```javascript
// lib/firebase.js
import * as admin from 'firebase-admin';

const serviceAccount = JSON.parse(process.env.FIREBASE_SERVICE_ACCOUNT);

admin.initializeApp({
```

### 3.4 外部 MySQL 接続

```bash
npm install mysql2/promise
```

```javascript

import mysql from 'mysql2/promise';

export async function getConnection() {
  return mysql.createConnection({
    host: process.env.MYSQL_HOST,
    user: process.env.MYSQL_USER,
    password: process.env.MYSQL_PASSWORD,
    database: process.env.MYSQL_DATABASE,
  });

```

```javascript

export default async function handler(req, res) {
  const connection = await getConnection();
  const [rows] = await connection.execute('SELECT * FROM orders');
  await connection.end();
  res.json(rows);
}
```

## 4. ORM ライブラリ連携

### 4.1 Prisma ORM

```bash
npm install @prisma/client
npm install -D prisma
npx prisma init
```

```prisma
// prisma/schema.prisma
datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

model User {
  id    Int     @id @default(autoincrement())
  name  String
  email String  @unique
  posts Post[]
}

model Post {
  id    Int     @id @default(autoincrement())
  title String
  author User    @relation(fields: [authorId], references: [id])
  authorId Int
}
```

```bash
# マイグレーション実行
npx prisma migrate dev --name init
```

```javascript
// lib/prisma.js
import { PrismaClient } from '@prisma/client';

let prisma;

if (process.env.NODE_ENV === 'production') {
  prisma = new PrismaClient();
} else {
  if (!global.prisma) {
    global.prisma = new PrismaClient();
  }
  prisma = global.prisma;
}

export default prisma;
```

```javascript
// pages/api/users.js
import prisma from '@/lib/prisma';

export default async function handler(req, res) {
  if (req.method === 'GET') {
    const users = await prisma.user.findMany();
    res.json(users);
  }

  if (req.method === 'POST') {
    const user = await prisma.user.create({
      data: {
        name: req.body.name,
        email: req.body.email,
      },
    });
    res.json(user);
  }
}
```

## 5. 接続プール

### 5.1 接続プール設定

Vercel Postgres はデフォルト接続プーリング済み。

外部 PostgreSQL の場合：

```javascript
// lib/pool.js
import { Pool } from 'pg';

export const pool = new Pool({
  connectionString: process.env.DATABASE_URL,
  max: 20,              // 最大接続数
  idleTimeoutMillis: 30000,  // アイドルタイムアウト
  connectionTimeoutMillis: 2000,  // 接続タイムアウト
});
```

### 5.2 PgBouncer（外部接続時）

PostgreSQL が 接続数制限に達した場合、**PgBouncer** コネクションプーラーを使用：

```
外部DB設定:
server = external-db.example.com:5432
auth_type = md5
pool_mode = transaction
max_client_conn = 1000
default_pool_size = 25
```

### 5.3 Vercel の接続文字列（プーリング有効）

```
POSTGRES_URL=postgresql://...?sslmode=require
（すでにプーリング適用）
```

## 6. マイグレーション管理

### 6.1 Prisma マイグレーション

```bash
# 新規マイグレーション作成
npx prisma migrate dev --name add_user_role

# 本番環境に適用
npx prisma migrate deploy

# マイグレーション履歴確認
npx prisma migrate status
```

### 6.2 Drizzle マイグレーション

```bash
# マイグレーションファイル生成
npx drizzle-kit generate:pg

# マイグレーション実行
npx drizzle-kit migrate
```

### 6.3 GitHub Actions でのマイグレーション自動化

```yaml
# .github/workflows/migrate.yml
name: Database Migration

on:
  push:
    branches: [main]

jobs:
  migrate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
        with:
          node-version: '18'
      - run: npm ci
      - run: npx prisma migrate deploy
        env:
          DATABASE_URL: ${{ secrets.DATABASE_URL }}
```

## 7. セキュリティ・接続文字列

### 7.1 接続文字列管理

```bash
# .env.local (git 除外)
POSTGRES_URL=postgresql://user:password@host/db

# 本番環境：Vercel ダッシュボード設定
```

### 7.2 SSL 接続（必須）

```javascript
// lib/db.js
import pg from 'pg';

const client = new pg.Client({
  connectionString: process.env.DATABASE_URL,
  ssl: {
    rejectUnauthorized: false,  // Vercel は自動
  },
});

await client.connect();
```

### 7.3 接続文字列の別形式

```
PostgreSQL URL フォーマット:
postgresql://[user[:password]@][netloc][:port][/dbname][?param1=value1&...]

例:
postgresql://user:pass@localhost:5432/mydb?sslmode=require
```

### 7.4 読み取り専用レプリカ設定

```javascript
// 本番環境では読み取り専用
const replicaUrl = process.env.DATABASE_REPLICA_URL;

const readClient = new pg.Client(replicaUrl);
```

## 8. トラブルシューティング

### よくある問題

| 問題 | 原因 | 解決策 |
|---|---|---|
| 接続タイムアウト | DB がダウン / ネットワーク遅延 | `SELECT 1;` で接続確認 |
| 認証エラー | パスワード間違い / ユーザー権限不足 | 接続文字列確認 |
| プール枯渇 | 接続が閉じられない | コネクション解放を確認 |
| マイグレーション失敗 | スキーマ競合 | ロック確認・再実行 |
| 接続文字列が見える | クライアント側にコンパイル | `.env.local` で管理 |

### デバッグコマンド

```bash
# Vercel から接続文字列プル
vercel env pull

# Prisma スキーマ確認
npx prisma db push

# DB 接続テスト
psql $POSTGRES_URL -c "SELECT 1;"

# ログ確認
vercel logs --follow
```
