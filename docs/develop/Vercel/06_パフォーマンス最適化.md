## 1. 画像最適化

### 1.1 Next.js Image コンポーネント

Vercel は自動的に画像を最適化。

```jsx
// pages/index.jsx
import Image from 'next/image';

export default function Home() {
  return (
    <Image
      src="/hero.jpg"
      alt="Hero"
      width={1200}
      height={600}
      priority  // LCP 画像は priority 指定
    />
  );
}
```

**メリット:**
- 自動 WebP 変換
- 遅延ロード（priority なし）
- レスポンシブ画像サービング
- AVIF サポート

### 1.2 外部画像 URL の最適化

```jsx
// next.config.js
export default {
  images: {
    remotePatterns: [
      {
        protocol: 'https',
        hostname: 'cdn.example.com',
        port: '',
        pathname: '/images/**',
      },
    ],
  },
};
```

```jsx
import Image from 'next/image';

<Image
  src="https://cdn.example.com/images/photo.jpg"
  alt="CDN image"
  width={800}
  height={600}
/>
```

### 1.3 画像ファイル形式

| 形式 | 用途 | サポート |
|---|---|---|
| **AVIF** | 最高圧縮率（次世代） | ⭐⭐⭐⭐⭐ |
| **WebP** | 高圧縮率 | ⭐⭐⭐⭐⭐ |
| **JPEG** | 標準形式 | ⭐⭐⭐⭐ |
| **PNG** | 透過画像 | ⭐⭐⭐⭐ |

Vercel は **自動的に AVIF → WebP → オリジナル** の順でサービング。

### 1.4 画像サイズ最適化の例

```jsx
// components/OptimizedImage.jsx
import Image from 'next/image';

export default function OptimizedImage({ src, alt }) {
  return (
    <Image
      src={src}
      alt={alt}
      width={1200}
      height={600}
      sizes="(max-width: 768px) 100vw, (max-width: 1200px) 50vw, 33vw"
      placeholder="blur"  // Blur-up 効果
      quality={75}  // 圧縮レベル（75 が推奨）
      priority={false}  // 遅延ロード
    />
  );
}
```

### 1.5 srcset 自動生成

```jsx
<Image
  src="/photo.jpg"
  alt="Photo"
  width={800}
  height={600}
  sizes="100vw"  // 自動的に複数サイズ生成
/>

// 生成される srcset
// /photo.jpg?w=640
// /photo.jpg?w=750
// /photo.jpg?w=828
// ...
```

## 2. Core Web Vitals

### 2.1 Core Web Vitals の 3つの指標

| 指標 | 意味 | 目標 |
|---|---|---|
| **LCP** | Largest Contentful Paint（最大要素の描画） | < 2.5s |
| **FID** | First Input Delay（入力遅延） | < 100ms |
| **CLS** | Cumulative Layout Shift（レイアウトシフト） | < 0.1 |

### 2.2 LCP 改善

```jsx
// ❌ 悪い例：大きな画像が遅れて読み込み
<Image src="/hero-large.jpg" />

// ✅ 良い例：priority で最初に読み込み
<Image
  src="/hero-large.jpg"
  priority
  width={1200}
  height={600}
/>
```

```jsx
// ✅ 良い例：font-display を使用
// pages/_document.jsx
<link
  href="https://fonts.googleapis.com/css2?family=Roboto&display=swap"
  rel="stylesheet"
/>
```

### 2.3 FID / INP 改善

```jsx
// ❌ 悪い例：メインスレッドブロッキング
function handleClick() {
  heavyComputation();  // 長時間実行
  updateUI();
}

// ✅ 良い例：Web Worker に移譲
function handleClick() {
  const worker = new Worker('/heavy-work.js');
  worker.onmessage = (e) => updateUI(e.data);
}
```

### 2.4 CLS 改善

```jsx
// ❌ 悪い例：高さ未指定でレイアウト崩れ
<img src="/image.jpg" alt="Image" />

// ✅ 良い例：width / height で予約
<img
  src="/image.jpg"
  alt="Image"
  width={800}
  height={600}
  style={{ width: '100%', height: 'auto' }}
/>
```

```jsx
// ✅ 良い例：Font Loading Observer
// フォント読み込み待機時にスペース確保
import { useEffect } from 'react';

export default function App() {
  useEffect(() => {
    document.fonts.ready.then(() => {
      // フォント準備完了後にレイアウト確定
    });
  }, []);
}
```

```
Project > Analytics > Web Vitals
```

リアルユーザーメトリクス（RUM）を表示。

## 3. キャッシング戦略

### 3.1 静的キャッシュ（ビルド時）

```javascript
// next.config.js
export default {
  onDemandEntries: {
    maxInactiveAge: 60 * 1000,  // 60秒キャッシュ保持
  },
};
```

### 3.2 ISR（Incremental Static Regeneration）

```jsx
// pages/products/[id].jsx
export default function Product({ product }) {
  return <h1>{product.name}</h1>;
}

export async function getStaticProps({ params }) {
  const product = await fetch(`/api/products/${params.id}`);

  return {
    props: { product },
    revalidate: 60,  // 60秒ごとに再生成
  };
}

export async function getStaticPaths() {
  return {
    paths: [{ params: { id: '1' } }],
    fallback: 'blocking',  // 動的ページは オンデマンド生成
  };
}
```

### 3.3 HTTP キャッシュヘッダー

```javascript
// pages/api/data.js
export default function handler(req, res) {
  res.setHeader('Cache-Control', 'public, max-age=3600, s-maxage=86400');
  // クライアント: 1時間キャッシュ
  // Vercel Edge: 24時間キャッシュ
  res.json({ data: 'content' });
}
```

### 3.4 Edge Caching

Vercel の Edge Network は自動的に以下をキャッシュ：

```
- Static HTML / CSS / JS
- Image Optimization API 応答
- API Routes (Cache-Control ヘッダー付き)
```

キャッシュバスティング：

```javascript
// キャッシュ無効化
res.setHeader('Cache-Control', 'no-cache, no-store, must-revalidate');

// または Vercel から手動削除
// Project > Deployments > [URL] > [Settings] > Cache > Clear
```

### 3.5 サーバーサイドキャッシング

```javascript
// lib/cache.js
const cache = new Map();

export function getCached(key, fn, ttl = 3600) {
  if (cache.has(key)) {
    const { data, expiry } = cache.get(key);
    if (Date.now() < expiry) return data;
    cache.delete(key);
  }

  return new Promise((resolve) => {
    fn().then((data) => {
      cache.set(key, { data, expiry: Date.now() + ttl * 1000 });
      resolve(data);
    });
  });
}
```

## 4. Edge Middleware

### 4.1 Edge Middleware とは

Vercel Edge Network で低遅延実行される Node.js 環境。

### 4.2 Middleware 基本

```typescript
// middleware.ts
import { NextResponse } from 'next/server';
import type { NextRequest } from 'next/server';

export function middleware(request: NextRequest) {
  // リクエスト処理
  const response = NextResponse.next();

  // ヘッダー追加
  response.headers.set('X-Edge-Location', 'sfo');

  return response;
}

export const config = {
  matcher: ['/api/:path*'],
};
```

### 4.3 リダイレクト・リライト

```typescript
// middleware.ts
import { NextResponse } from 'next/server';

export function middleware(request: NextRequest) {
  const url = request.nextUrl.clone();

  // /old-page → /new-page にリダイレクト
  if (url.pathname === '/old-page') {
    url.pathname = '/new-page';
    return NextResponse.redirect(url);
  }

  // /api/v1 → /api/v2 にリライト
  if (url.pathname.startsWith('/api/v1')) {
    url.pathname = url.pathname.replace('/v1', '/v2');
    return NextResponse.rewrite(url);
  }

  return NextResponse.next();
}
```

### 4.4 地域別処理

```typescript
// middleware.ts
import { NextResponse } from 'next/server';

export function middleware(request: NextRequest) {
  const country = request.geo?.country;

  // 日本からのアクセスは別サイト
  if (country === 'JP') {
    return NextResponse.rewrite(new URL('/jp', request.url));
  }

  return NextResponse.next();
}
```

### 4.5 レート制限

```typescript
// middleware.ts
import { NextResponse } from 'next/server';

const rateLimit = new Map();

export function middleware(request: NextRequest) {
  const ip = request.ip || 'unknown';

  if (!rateLimit.has(ip)) {
    rateLimit.set(ip, []);
  }

  const requests = rateLimit.get(ip);
  const now = Date.now();
  const recentRequests = requests.filter((t) => now - t < 60000);

  if (recentRequests.length > 100) {
    return new NextResponse('Rate limited', { status: 429 });
  }

  recentRequests.push(now);
  rateLimit.set(ip, recentRequests);

  return NextResponse.next();
}
```

### 5.2 動的インポート

```jsx
// ❌ 悪い例：すべて初期バンドルに含まれる
import HeavyComponent from './HeavyComponent';

export default function Page() {
  return <HeavyComponent />;
}

// ✅ 良い例：遅延ロード
import dynamic from 'next/dynamic';

const HeavyComponent = dynamic(() => import('./HeavyComponent'), {
  loading: () => <p>Loading...</p>,
});

export default function Page() {
  return <HeavyComponent />;
}
```

### 5.3 Tree Shaking

```javascript
// ❌ 悪い例：全体がバンドル
import lodash from 'lodash';

const result = lodash.map([1, 2, 3], x => x * 2);

// ✅ 良い例：必要な関数のみ
import map from 'lodash/map';

const result = map([1, 2, 3], x => x * 2);
```

### 5.4 CSS-in-JS の最適化

```jsx
// ❌ 悪い例：runtime CSS-in-JS (styled-components v5)
import styled from 'styled-components';

const Box = styled.div`
  color: red;
`;

// ✅ 良い例：Zero Runtime CSS (Tailwind / Panda CSS)
export default function Box() {
  return <div className="text-red-500">Box</div>;
}
```

## 8. トラブルシューティング

### よくある問題

| 問題 | 原因 | 解決策 |
|---|---|---|
| LCP が遅い | 大型画像遅延ロード | `priority` 指定 |
| CLS が大きい | フォント読み込み遅延 | `display=swap` |
| FID 高い | 重い JavaScript | 動的インポート・Web Worker |
| キャッシュされない | Cache-Control 未設定 | ヘッダー設定確認 |
| Edge Middleware 遅い | 重い処理 | ロジック最小化 |

### デバッグコマンド

```bash
# 本番環境を検証
vercel logs --follow

# Lighthouse スコア確認
lighthouse https://deployment.vercel.app --chrome-flags="--headless"

# バンドルサイズ確認
npm run build -- --analyze
```
