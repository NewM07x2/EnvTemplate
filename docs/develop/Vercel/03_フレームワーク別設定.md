## 1. 共通セットアップ

### 1.1 vercel.json の作成（オプション）

どのフレームワークでも、プロジェクトルートに `vercel.json` を作成することでデプロイ設定をカスタマイズできます。

```json
{
  "version": 2,
  "buildCommand": "npm run build",
  "outputDirectory": "dist",
  "installCommand": "npm install",
  "env": {
    "NODE_ENV": "production"
  },
  "regions": ["sfo1"],
  "functions": {
    "api/**": {
      "maxDuration": 60
    }
  }
}
```

### 1.2 .gitignore に追加

```gitignore
# Vercel
.vercel
.env.local
.env.*.local

# ビルド出力
dist/
build/
out/
.next/
.nuxt/

# ノードモジュール
node_modules/

# その他
.DS_Store
*.log
```

## 2. Next.js（推奨）

**最適度**: ⭐⭐⭐⭐⭐  
**セットアップ難易度**: ⭐（最も簡単）

### 2.1 プロジェクト作成

```bash
npx create-next-app@latest my-nextjs-app --typescript
cd my-nextjs-app
```

### 2.2 最小限の next.config.js

```javascript
// next.config.js
/** @type {import('next').NextConfig} */
const nextConfig = {
  reactStrictMode: true,
  // 画像最適化（Vercel の Image Optimization API を使用）
  images: {
    remotePatterns: [
      {
        protocol: 'https',
        hostname: 'example.com',
      },
    ],
  },
};

module.exports = nextConfig;
```

### 2.3 デプロイ

```bash
git push origin main
# 自動デプロイ開始 ✅
```

### 2.4 Next.js 固有機能

| 機能 | 説明 |
|---|---|
| **Image Optimization** | 自動で WebP 変換・リサイズ |
| **ISR** | キャッシュを期間限定リセット |
| **API Routes** | サーバーレス関数自動配置 |
| **Middleware** | Edge での処理 |
| **ドラフト機能** | プレビュー環境自動生成 |

### 2.5 環境変数設定

```bash
# .env.local
NEXT_PUBLIC_API_URL=https://api.example.com
DATABASE_URL=postgresql://...

# Vercel にも同じ環境変数を設定
vercel env add NEXT_PUBLIC_API_URL
vercel env add DATABASE_URL
```

## 3. React (SPA)

**最適度**: ⭐⭐⭐⭐  
**セットアップ難易度**: ⭐⭐

### 3.1 プロジェクト作成

```bash
npx create-react-app my-react-app
cd my-react-app
```

### 3.2 build 設定

React は デフォルトで `build/` にビルド出力されます。

```
package.json はデフォルトで正しく設定済み:
"build": "react-scripts build"
```

### 3.3 Vite + React の場合

```bash
npm create vite@latest my-react-app -- --template react-ts
cd my-react-app
npm install
npm run build
```

### 3.4 vercel.json での出力設定

```json
{
  "version": 2,
  "buildCommand": "npm run build",
  "outputDirectory": "dist"  // Vite の場合
}
```

### 3.5 SPA の注意点

React SPA は **クライアントサイド ルーティング** なため、Vercel で正しくルーティング設定が必要。

```json
{
  "version": 2,
  "routes": [
    {
      "src": "/(.*)",
      "dest": "/index.html"
    }
  ]
}
```

**または** `vercel.json` を使わず、設定は自動（React SPA 検出時）。

### 3.6 環境変数

```javascript
// React は REACT_APP_ プレフィックスが必須
// .env.local
REACT_APP_API_URL=https://api.example.com

// .js での使用
const apiUrl = process.env.REACT_APP_API_URL;
```

## 5. Nuxt

**最適度**: ⭐⭐⭐⭐  
**セットアップ難易度**: ⭐⭐⭐

Nuxt はフルスタックフレームワークなため、Vercel の最適化が より深く統合されています。

### 5.1 プロジェクト作成

```bash
npx nuxi init my-nuxt-app
cd my-nuxt-app
npm install
```

### 5.2 nuxt.config.ts

```typescript
// nuxt.config.ts
export default defineNuxtConfig({
  ssr: true,  // SSR 有効（デフォルト）
  nitro: {
    prerender: {
      crawlLinks: true,
      routes: ['/sitemap.xml', '/rss.xml'],
    },
  },
})
```

### 5.3 Vercel デプロイ

Nuxt 3.x は Vercel が自動検出するため、`vercel.json` 不要。

```bash
git push origin main
# 自動デプロイ ✅
```

### 5.4 Server API Routes

```typescript
// server/api/hello.ts
export default defineEventHandler(async (event) => {
  return {
    message: 'Hello from Nuxt API',
  }
})
```

これは自動的にサーバーレス関数にデプロイされます。

### 5.5 環境変数

```
.env
NUXT_PUBLIC_API_URL=https://api.example.com
```

## 6. SvelteKit

**最適度**: ⭐⭐⭐⭐  
**セットアップ難易度**: ⭐⭐

### 6.1 プロジェクト作成

```bash
npm create svelte@latest my-sveltekit-app
cd my-sveltekit-app
npm install
```

### 6.2 adapter-vercel をインストール

```bash
npm install -D @sveltejs/adapter-vercel
```

### 6.3 svelte.config.js を修正

```javascript
// svelte.config.js
import adapter from '@sveltejs/adapter-vercel';

export default {
  kit: {
    adapter: adapter({
      // オプション
      split: false,  // API Routes を分割するか
    }),
  },
};
```

### 6.4 Vercel デプロイ

```bash
git push origin main
# 自動デプロイ ✅
```

### 6.5 Server Routes（API）

```typescript
// src/routes/api/hello/+server.ts
import { json } from '@sveltejs/kit';

export async function GET() {
  return json({ message: 'Hello from SvelteKit' });
}
```

自動的にサーバーレス関数化。

### 6.6 環境変数

```
.env
PUBLIC_API_URL=https://api.example.com
SECRET_KEY=mykey123
```

## 9. Express / Node.js

**最適度**: ⭐⭐⭐  
**セットアップ難易度**: ⭐⭐⭐⭐

### 9.1 プロジェクト作成

```bash
mkdir my-express-app
cd my-express-app
npm init -y
npm install express
```

### 9.2 サーバーコード

```javascript
// api/index.js
import express from 'express';

const app = express();
app.use(express.json());

app.get('/api/hello', (req, res) => {
  res.json({ message: 'Hello from Express' });
});

export default app;
```

### 9.3 vercel.json

```json
{
  "version": 2,
  "builds": [
    {
      "src": "api/index.js",
      "use": "@vercel/node"
    }
  ],
  "routes": [
    {
      "src": "/(.*)",
      "dest": "api/index.js"

}
```

```json
{
  "name": "my-express-app",
  "version": "1.0.0",
  "type": "module",

    "dev": "node api/index.js"
  },
  "dependencies": {
    "express": "^4.18.0"
  }
}
```

### 9.5 デプロイ

```bash
git push origin main
```

## 10. トラブルシューティング

### よくある問題

| 問題 | 原因 | 解決策 |
|---|---|---|
| ビルド失敗 | Node.js バージョン不一致 | `vercel.json` で `nodeVersion` 指定 |
| 404 エラー（SPA） | ルーティング未設定 | `vercel.json` で `routes` を設定 |
| 環境変数が反映されない | `NEXT_PUBLIC_` プレフィックス忘れ | プレフィックスを確認・追加 |
| デプロイが遅い | 依存ライブラリが多い | `package-lock.json` で固定 |
| API が 500 エラー | 環境変数不足 | `vercel env list` で確認 |

### デバッグのコツ

```bash
# ローカルで本番環境を再現
vercel dev

# ビルドログを確認
vercel logs --follow

# デプロイ状況を詳細確認
vercel inspect [DEPLOYMENT_URL]
```
