## 1. ビルド失敗

### 1.1 Node.js バージョン不一致

**エラーメッセージ:**

```
Error: The engine "node" is incompatible with this module.
Expected version "16.x"
```

**原因**: ローカルと Vercel の Node.js バージョンが異なる

**解決策:**

```javascript
// vercel.json
{
  "env": {
    "NODE_ENV": "production",
    "NODE_OPTIONS": "--max-old-space-size=4096"
  },
  "builds": [
    {
      "src": "package.json",
      "use": "@vercel/node@3",
      "config": {
        "nodeVersion": "18.x"  // ← バージョン指定
      }
    }
  ]
}
```

または

```bash
# .nvmrc でローカルバージョン指定
echo "18.16.0" > .nvmrc

# Vercel に通知
vercel env add NODE_VERSION "18.x"
```

### 1.2 依存ライブラリのインストール失敗

**エラーメッセージ:**

```
npm ERR! code E404
npm ERR! 404 Not Found - GET https://registry.npmjs.org/missing-package
```

**原因**: `package.json` に不要なパッケージが記載 / 削除されたパッケージ

**解決策:**

```bash
# 1. ローカルで依存関係を確認
npm install

# 2. package-lock.json を再生成
rm package-lock.json
npm install

# 3. Git にプッシュ
git add package.json package-lock.json
git commit -m "Update dependencies"
git push origin main
```

### 1.3 メモリ不足でビルド失敗

**エラーメッセージ:**

```
FATAL ERROR: CALL_AND_RETRY_LAST Allocation failed - JavaScript heap out of memory
```

**原因**: 大型プロジェクトでメモリ不足

**解決策:**

```bash
# vercel.json で メモリ制限を緩和
{
  "builds": [
    {
      "src": "package.json",
      "use": "@vercel/node@3"
    }
  ],
  "env": {
    "NODE_OPTIONS": "--max-old-space-size=4096"  // 4GB に設定
  }
}
```

### 1.4 Next.js ビルド時の型エラー

**エラーメッセージ:**

```
error - ./app/page.tsx:5:32
Type 'string | undefined' is not assignable to type 'string'
```

**原因**: TypeScript の厳密モードエラー

**解決策:**

```typescript
// tsconfig.json
{
  "compilerOptions": {
    "strict": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true
  }
}
```

```typescript
// 修正例
interface Props {
  title: string;
}

export default function Page({ title = 'Default' }: Partial<Props>) {
  return <h1>{title}</h1>;
}
```

### 1.5 Prisma マイグレーション失敗

**エラーメッセージ:**

```
Migration failed to apply cleanly to the shadow database.
```

**原因**: スキーマが破損 / ロック状態

**解決策:**

```bash
# 1. ローカルでリセット（注意：データ削除）
npx prisma migrate reset

# 2. Vercel でスキーマリセット
npx prisma migrate deploy --skip-generate

# 3. ダッシュボードから DB リセット
# Vercel > Storage > Database > Reset Database
```

## 2. デプロイメントエラー

### 2.1 GitHub 連携が不安定

**現象**: PR マージ後にデプロイされない

**原因**: GitHub ウェブフック設定の問題

**解決策:**

```bash
# 1. Vercel アカウント再接続
vercel link --prod

# 2. GitHub リポジトリ設定確認
# Repository Settings > Installed GitHub Apps > Vercel

# 3. デプロイをリトライ
vercel redeploy [DEPLOYMENT_ID]
```

### 2.2 デプロイが 50 分でタイムアウト

**エラーメッセージ:**

```
Build timed out after 50 minutes
```

**原因**: ビルド処理が長い / 無限ループ

**解決策:**

```javascript
// next.config.js でビルド最適化
export default {
  // 不要なビルド処理を削除
  productionBrowserSourceMaps: false,
  poweredByHeader: false,
  compress: true,
  optimizeFonts: true,
};
```

```bash
# ビルド時間を測定
time npm run build
```

### 2.3 Vercel CLI でのデプロイ失敗

**エラー:**

```
Error: Project not found. Did you mean one of these?
```

**原因**: プロジェクト ID 設定ミス

**解決策:**

```bash
# 1. リンク
vercel link

# 2. .vercelignore でビルド除外
node_modules
.git
.env.local
```

### 2.4 キャッシュクリア後も古い版が表示

**原因**: CDN キャッシュが残存

**解決策:**

```bash
# 1. Vercel ダッシュボードからキャッシュクリア
# Project > Deployments > [URL] > Settings > Cache > Clear All

# 2. または CLI で
vercel --prod --skip-build

# 3. ブラウザキャッシュもクリア
# Ctrl+Shift+Delete > 全期間 > キャッシュ削除
```

## 3. ランタイムエラー

### 3.1 500 Internal Server Error

**原因**: サーバーサイドコード実行時エラー

**解決策:**

```bash
# 1. ログ確認
vercel logs --follow

# 例出力:
# [GET] /api/users - 500
# Error: ECONNREFUSED (DB 接続失敗)

# 2. 環境変数確認
vercel env list

# 3. ローカルで再現
vercel dev

# 4. エラーハンドリング追加
export default async function handler(req, res) {
  try {
    const data = await fetchData();
    res.json(data);
  } catch (error) {
    console.error('API Error:', error);
    res.status(500).json({ error: error.message });
  }
}
```

### 3.2 Cannot find module エラー

**エラーメッセージ:**

```
Error: Cannot find module '@app/utils'
```

**原因**: パスエイリアス設定の不一致

**解決策:**

```json
// tsconfig.json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@app/*": ["src/*"],
      "@utils/*": ["lib/utils/*"]
    }
  }
}
```

```javascript
// next.config.js でも確認
module.exports = {
  webpack: (config) => {
    config.resolve.alias = {
      ...config.resolve.alias,
      '@app': require('path').resolve(__dirname, 'src/'),
    };
    return config;
  },
};
```

### 3.3 モジュールが ESM / CJS の不一致

**エラーメッセージ:**

```
ERR_MODULE_NOT_FOUND: Cannot find module
Error [ERR_MODULE_NOT_FOUND]: No such file...
```

**原因**: CommonJS と ES Modules の混在

**解決策:**

```json
// package.json
{
  "type": "module",  // ES Modules として宣言
  "engines": {
    "node": "18.x"
  }
}
```

```javascript
// ファイル拡張子を明示
import { helper } from './lib/helper.js';  // .js を追加
```

### 3.4 動的インポート失敗

**エラー:**

```
TypeError: Dynamic require is not available
```

**原因**: サーバーレス環境で `require()` を使用

**解決策:**

```javascript
// ❌ 悪い例
const module = require(`./modules/${name}`);

// ✅ 良い例
import * as modules from './modules';
const module = modules[name];
```

## 4. 環境変数の問題

### 4.1 環境変数が読み込まれない

**現象**: `process.env.MY_VAR` が undefined

**原因:**
- Vercel ダッシュボードに未設定
- プレフィックス忘れ（NEXT_PUBLIC_ など）
- デプロイ後キャッシュ

**解決策:**

```bash
# 1. 環境変数を確認
vercel env list

# 2. ローカルにプル（同期）
vercel env pull

# 3. 環境を指定して設定
vercel env add MY_VAR "value" --environment=production

# 4. デプロイ再実行
vercel --prod --skip-build
```

```javascript
// 4. Next.js コンポーネントで確認
export default function Page() {
  console.log(process.env.NEXT_PUBLIC_API_URL);  // ✅ クライアント側可
  console.log(process.env.SECRET_KEY);  // ❌ サーバーサイドのみ
  return <div>{process.env.NEXT_PUBLIC_API_URL}</div>;
}
```

### 4.2 プロダクションとプレビューの環境変数が異なる

**原因**: 環境別設定の不一致

**解決策:**

```bash
# Production 環境のみに設定
vercel env add API_URL "https://api.example.com" --environment=production

# Preview 環境に設定
vercel env add API_URL "https://staging-api.example.com" --environment=preview

# 確認
vercel env list --environment=production
vercel env list --environment=preview
```

### 4.3 シークレット変数が見える

**原因**: クライアント側に埋め込まれた

**現象**: DevTools > Network で環境変数が見える

**解決策:**

```javascript
// ❌ 悪い例：クライアント側でシークレット使用
export default function Page() {
  const secret = process.env.DATABASE_PASSWORD;  // ❌ ブラウザに晒される
  return <div>{secret}</div>;
}

// ✅ 良い例：サーバーサイドのみ
export async function getServerSideProps() {
  const secret = process.env.DATABASE_PASSWORD;  // ✅ サーバーのみ
  const data = await connectDB(secret);
  return { props: { data } };
}
```

## 5. パフォーマンス問題

### 5.1 ページ読み込みが遅い

**原因**: 大型 JavaScript / 画像未最適化

**解決策:**

```bash
# 1. Lighthouse で計測
npx lighthouse https://my-site.vercel.app

# 2. バンドルサイズ確認
npm run build -- --analyze

# 3. Core Web Vitals 確認
# Vercel > Analytics > Web Vitals
```

```jsx
// 画像最適化
import Image from 'next/image';

<Image
  src="/hero.jpg"
  width={1200}
  height={600}
  priority  // LCP 画像
  quality={75}
/>

// 動的インポート
import dynamic from 'next/dynamic';
const HeavyComponent = dynamic(() => import('./Heavy'));
```

### 5.2 API 応答が遅い

**原因**: DB クエリ遅い / N+1 問題

**解決策:**

```javascript
// 1. キャッシング実装
import { kv } from '@vercel/kv';

export default async function handler(req, res) {
  const cached = await kv.get('users:all');
  if (cached) return res.json(cached);

  const users = await db.query('SELECT * FROM users');
  await kv.set('users:all', users, { ex: 3600 });
  res.json(users);
}

// 2. クエリ最適化
// ❌ N+1 問題
const users = await db.query('SELECT * FROM users');
for (const user of users) {
  user.posts = await db.query('SELECT * FROM posts WHERE user_id = ?', user.id);
}

// ✅ JOIN で解決
const users = await db.query(`
  SELECT u.*, p.*
  FROM users u
  LEFT JOIN posts p ON u.id = p.user_id
`);
```

## 6. データベース接続

### 6.1 接続タイムアウト

**エラーメッセージ:**

```
Error: connect ETIMEDOUT
```

**原因**: DB がダウン / ネットワーク遅延 / ファイアウォール

**解決策:**

```bash
# 1. DB が動作しているか確認
psql $DATABASE_URL -c "SELECT 1;"

# 2. 接続文字列確認
vercel env list

# 3. Vercel Postgres の場合
# Storage > [Database] > Status を確認

# 4. 接続タイムアウト値を増やす
# vercel.json
{
  "functions": {
    "api/**": {
      "maxDuration": 60  // 60秒
    }
  }
}
```

### 6.2 認証エラー（ユーザー名・パスワード）

**エラー:**

```
fatal: password authentication failed for user "postgres"
```

**原因**: 接続文字列のユーザー名・パスワード間違い

**解決策:**

```bash
# 1. Vercel ダッシュボードから接続文字列確認
# Project > Storage > Database > Connection Details

# 2. コピーして `.env.local` に設定
vercel env pull

# 3. 接続テスト
psql $POSTGRES_URL -c "SELECT 1;"
```

### 6.3 プール接続数超過

**エラー:**

```
too many connections
```

**原因**: 同時接続が制限超過

**解決策:**

```javascript
// 接続プール設定
import { Pool } from 'pg';

const pool = new Pool({
  max: 10,  // 最大接続数を削減
  idleTimeoutMillis: 30000,
  connectionTimeoutMillis: 2000,
});

// または singleton pattern でインスタンス共有
let connection;

export async function getConnection() {
  if (!connection) {
    connection = new Pool({ max: 5 });
  }
  return connection;
}
```

## 7. CORS・ネットワーク

### 7.1 CORS エラー

**エラーメッセージ:**

```
Access to XMLHttpRequest from origin has been blocked by CORS policy
```

**原因**: クロスオリジンリクエストで CORS ヘッダー不足

**解決策:**

```javascript
// API Route に CORS ヘッダーを追加
export default function handler(req, res) {
  res.setHeader('Access-Control-Allow-Origin', '*');
  res.setHeader('Access-Control-Allow-Methods', 'GET,POST,PUT,DELETE');
  res.setHeader('Access-Control-Allow-Headers', 'Content-Type');

  if (req.method === 'OPTIONS') {
    return res.status(200).end();
  }

  // API ロジック
  res.json({ message: 'OK' });
}
```

### 7.2 Mixed Content エラー

**エラー:**

```
Mixed Content: The page was loaded over HTTPS but requested an insecure resource
```

**原因**: HTTPS サイトから HTTP リソースを読み込み

**解決策:**

```javascript
// 環境変数を HTTPS に
// .env.local
NEXT_PUBLIC_API_URL=https://api.example.com  // https:// に修正

// または Middleware でリダイレクト
export function middleware(request) {
  if (request.nextUrl.protocol === 'http:') {
    return NextResponse.redirect(`https://${request.nextUrl.host}${request.nextUrl.pathname}`);
  }
}
```

## 8. ログ・デバッグ方法

### 8.1 本番環境ログ確認

```bash
# リアルタイムログ
vercel logs --follow

# フィルター付き
vercel logs --follow --environment=production

# エラーのみ
vercel logs --follow | grep "error"
```

### 8.2 ローカルで本番環境を再現

```bash
# 本番環境を模擬
vercel env pull  # 環境変数をプル
vercel dev      # ローカルで起動

# http://localhost:3000 でテスト
```

### 8.3 デプロイメント詳細確認

```bash
# デプロイメント検査
vercel inspect [DEPLOYMENT_URL]

# ビルド出力確認
vercel logs --environment=production [DEPLOYMENT_URL]

# Vercel ダッシュボード
# Project > Deployments > [URL] > Logs
```

### 8.4 DevTools で実行時エラー確認

```javascript
// 本番環境でもエラーを追跡
window.addEventListener('error', (event) => {
  console.error('Runtime Error:', event);
  // 外部ログサービスに送信
  fetch('/api/log-error', {
    method: 'POST',
    body: JSON.stringify(event),
  });
});
```
