## 1. 基本セットアップ

### 1.1 初期環境構築スクリプト

```bash
#!/bin/bash
# scripts/setup-dev.sh

set -e

echo "🚀 Vercel 開発環境セットアップ開始..."

# Node.js バージョン確認
node_version=$(node -v | cut -d'v' -f2)
echo "✅ Node.js: $node_version"

# 依存ライブラリインストール
echo "📦 依存ライブラリをインストール中..."
npm install

# 環境変数ファイル作成
if [ ! -f .env.local ]; then
  echo "📝 .env.local を作成中..."
  cp .env.example .env.local
  echo "⚠️  .env.local を編集してください"
fi

# Vercel リンク
if [ ! -d .vercel ]; then
  echo "🔗 Vercel プロジェクトをリンク中..."
  npx vercel link
fi

# 環境変数をローカルにプル
echo "📥 環境変数をプル中..."
vercel env pull

# Git フック設定
echo "🪝 Git フック設定中..."
npx husky install
npx husky add .husky/pre-commit "npm run lint"
npx husky add .husky/pre-push "npm test"

echo "✅ セットアップ完了！"
echo "🏃 npm run dev でサーバーを起動してください"
```

```bash
chmod +x scripts/setup-dev.sh
./scripts/setup-dev.sh
```

### 1.2 package.json 開発スクリプト

```json

  "scripts": {
```

## 2. マルチプロジェクト構成

### 2.1 モノレポ構造（Turborepo）

```
my-workspace/
├── apps/
│   ├── web/              # Next.js フロントエンド
│   │   ├── package.json
│   │   ├── vercel.json
│   │   └── src/
│   ├── api/              # API サーバー（Express）
│   │   ├── package.json
│   │   └── src/
│   └── admin/            # 管理画面
│       ├── package.json
│       └── src/
├── packages/
│   ├── shared/           # 共有ライブラリ
│   ├── ui/               # UI コンポーネント
│   └── types/            # 型定義
├── turbo.json
└── package.json
```

### 2.2 Turborepo 設定

```json
// turbo.json
{
  "version": "1",
  "tasks": {
    "dev": {
      "cache": false,
      "persistent": true
    },
    "build": {
      "dependsOn": ["^build"],
      "outputs": ["dist/**", "build/**"]
    },
    "test": {
      "outputs": ["coverage/**"],
      "cache": true
    },
    "lint": {
      "outputs": [".eslintcache"],
      "cache": true
    }
  }
}
```

### 2.3 複数アプリケーション同時開発

```bash
# 全アプリケーションを開発モードで起動
turbo run dev

# 特定アプリケーションのみ
turbo run dev --filter=web
turbo run dev --filter=api

# ターミナル分割して起動
# ウィンドウ 1
cd apps/web && npm run dev

# ウィンドウ 2
cd apps/api && npm run dev

# ウィンドウ 3
vercel dev --debug
```

## 3. Docker 開発環境

### 3.1 Docker Compose セットアップ

```yaml
# docker-compose.yml
version: '3.9'

services:
  app:
    build:
      context: .
      dockerfile: Dockerfile.dev
    ports:
      - "3000:3000"
      - "9229:9229"  # Node.js debugger
    environment:
      - NODE_ENV=development
      - DATABASE_URL=postgresql://user:password@db:5432/dev_db
      - REDIS_URL=redis://redis:6379
    volumes:
      - .:/app
      - /app/node_modules
    depends_on:
      - db
      - redis
    command: npm run dev

  db:
    image: postgres:15
    environment:
      POSTGRES_USER: user
      POSTGRES_PASSWORD: password
      POSTGRES_DB: dev_db
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data

  redis:
    image: redis:7
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data

volumes:
  postgres_data:
  redis_data:
```

### 3.2 Dockerfile.dev

```dockerfile
FROM node:18-alpine

WORKDIR /app

# キャッシュ最適化
COPY package*.json ./
RUN npm ci

# ソースコード
COPY . .

# ポート
EXPOSE 3000 9229

# ホットリロード有効
CMD ["npm", "run", "dev"]
```

### 3.3 Docker での開発

```bash
# 起動
docker-compose up -d

# ログ確認
docker-compose logs -f app

# シェルアクセス
docker-compose exec app sh

# 停止
docker-compose down

# DB リセット
docker-compose down -v
docker-compose up db
```

## 4. チームオンボーディング

### 4.1 新メンバー向けセットアップガイド

```markdown
# 開発環境セットアップガイド

## 前提条件
- Node.js 18+
- npm 9+
- Git
- VS Code (推奨)

## セットアップ手順

### 1. リポジトリをクローン
\`\`\`bash
git clone https://github.com/org/project.git
cd project
\`\`\`

### 2. 自動セットアップスクリプト実行
\`\`\`bash
npm run setup
\`\`\`

このスクリプトが以下を実行します:
- Node.js バージョン確認
- npm パッケージインストール
- .env.local 作成
- Vercel プロジェクトリンク
- 環境変数プル
- Git フック設定

### 3. 手動設定（初回のみ）
.env.local を編集して Vercel で取得した環境変数を設定

### 4. 開発サーバー起動
\`\`\`bash
npm run dev
\`\`\`

http://localhost:3000 でアクセス可能

## よくある問題

### "Port 3000 is already in use"
\`\`\`bash
# 別ポートで起動
PORT=3001 npm run dev
\`\`\`

### ".env.local が見つからない"
\`\`\`bash
vercel env pull
\`\`\`

### "Node.js バージョンエラー"
\`\`\`bash
nvm install 18
nvm use 18
\`\`\`

## サポート
質問は #dev-help Slack チャネルへ
```

### 4.2 チーム向けガイド資料

```markdown
# チーム開発ガイド

## ブランチ戦略 (Git Flow)

\`\`\`
main (本番)
├─ release/v1.0.0
└─ develop (開発)
   ├─ feature/user-auth
   ├─ feature/dashboard
   └─ bugfix/search-bug
\`\`\`

## コミットメッセージ規約

\`\`\`
feat: 新機能説明
fix: バグ修正説明
docs: ドキュメント更新
refactor: コード整理
test: テスト追加
chore: 依存関係更新など

例:
git commit -m "feat: ユーザー認証機能実装"
\`\`\`

## PR レビュー チェックリスト

- [ ] コード品質（lint パス）
- [ ] テストカバレッジ 80% 以上
- [ ] パフォーマンス影響なし
- [ ] セキュリティ脆弱性なし
- [ ] API ドキュメント更新済み

## デプロイメント プロセス

1. Feature ブランチで開発
2. PR 作成（自動でステージング デプロイ）
3. コードレビュー＆テスト
4. develop にマージ
5. release ブランチで最終テスト
6. main にマージ（本番デプロイ）
```

## 7. Git Worktree（複数ブランチ同時作業）

### 7.1 複数ブランチを同時に開く

```bash
# メインワークツリー確認
git worktree list

# 新しいブランチで別ワークツリー作成
git worktree add -b feature/new-feature ../project-feature main

# ファイル構成
project/
├── .git/
├── .gitworktrees/
│   └── project-feature/  # 新しいワークツリー
└── (main ブランチファイル)
```

### 7.2 複数ワークツリーで作業

```bash
# ウィンドウ 1: main ブランチで開発
cd project
npm run dev

# ウィンドウ 2: feature ブランチで作業
cd ../project-feature
npm run dev  # 別ポート

# 別プロセスなので互いに影響なし
PORT=3001 npm run dev
```

### 7.3 ワークツリー削除

```bash
# ワークツリー削除
git worktree remove ../project-feature

# 強制削除
git worktree remove --force ../project-feature

# クリーンアップ
git worktree prune
```

## 8. トラブルシューティング

### よくある問題

| 問題 | 原因 | 解決策 |
|---|---|---|
| Port 3000 使用中 | 別プロセスが起動中 | `PORT=3001 npm run dev` |
| Module not found | node_modules が古い | `npm ci` で再インストール |
| TypeScript エラー | 型定義キャッシュ | `tsc --noEmit` で確認 |
| .env.local なし | Vercel リンク未実行 | `vercel env pull` |
| Vercel リンク失敗 | 認証情報なし | `vercel login` で再認証 |

### デバッグコマンド

```bash
# 環境確認
npm list
node --version
npm --version

# キャッシュクリア
npm cache clean --force
rm -rf node_modules package-lock.json
npm install

# Vercel 再リンク
rm -rf .vercel
vercel link
vercel env pull

# 開発サーバーリセット
kill -9 $(lsof -t -i:3000)
npm run dev
```
