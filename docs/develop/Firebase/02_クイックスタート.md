## 1. Firebase プロジェクトの作成

### ① Firebase Console にアクセス

[https://console.firebase.google.com/](https://console.firebase.google.com/) を開き、Google アカウントでログインします。

### ② プロジェクトを作成

1. **「プロジェクトを追加」** をクリック
2. プロジェクト名を入力（例: `my-app-dev`）
3. Google Analytics の有効化を選択（学習用は無効でも可）
4. **「プロジェクトを作成」** をクリック

> 💡 **命名規則の例**: `{アプリ名}-{環境}` → `myapp-dev` / `myapp-prod`

### ③ Web アプリを登録

1. プロジェクトの概要ページで **`</>`（Web）** アイコンをクリック
2. アプリのニックネームを入力
3. **「アプリを登録」** をクリック
4. 表示される `firebaseConfig` オブジェクトをコピーして保存（後で使用）

```javascript
// コピーした設定はこのような形式
const firebaseConfig = {
  apiKey: "AIzaSy...",
  authDomain: "myapp-dev.firebaseapp.com",
  projectId: "myapp-dev",
  storageBucket: "myapp-dev.appspot.com",
  messagingSenderId: "123456789",
  appId: "1:123456789:web:abcdef"
};
```

## 3. Firebase の初期化

### ディレクトリ構成

```
src/
└── lib/
    └── firebase/
        ├── config.ts     ← Firebase の初期化
        ├── auth.ts       ← Authentication のヘルパー
        └── firestore.ts  ← Firestore のヘルパー
```

### 初期化ファイル (`src/lib/firebase/config.ts`)

```typescript
import { initializeApp, getApps, getApp } from 'firebase/app'
import { getAuth } from 'firebase/auth'
import { getFirestore } from 'firebase/firestore'
import { getStorage } from 'firebase/storage'

const firebaseConfig = {
  apiKey: process.env.NEXT_PUBLIC_FIREBASE_API_KEY,
  authDomain: process.env.NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN,
  projectId: process.env.NEXT_PUBLIC_FIREBASE_PROJECT_ID,
  storageBucket: process.env.NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET,
  messagingSenderId: process.env.NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID,
  appId: process.env.NEXT_PUBLIC_FIREBASE_APP_ID,
}

// 重複初期化を防ぐ（Next.js の Hot Reload 対策）
const app = getApps().length > 0 ? getApp() : initializeApp(firebaseConfig)

export const auth = getAuth(app)
export const db = getFirestore(app)
export const storage = getStorage(app)
export default app
```

## 4. 動作確認

Firestore に接続してデータを 1 件取得するテストを書きます。

```typescript
// src/app/page.tsx（動作確認用）
import { collection, getDocs } from 'firebase/firestore'
import { db } from '@/lib/firebase/config'

export default async function HomePage() {
  try {
    // 'test' コレクションの取得を試みる（存在しなくてもエラーにならない）
    const querySnapshot = await getDocs(collection(db, 'test'))
    console.log('Firestore 接続成功。ドキュメント数:', querySnapshot.size)
  } catch (error) {
    console.error('Firestore 接続エラー:', error)
  }

  return <div>Firebase 接続テスト完了（コンソールを確認してください）</div>
}
```

開発サーバーを起動してブラウザのコンソールに「Firestore 接続成功」と表示されれば OK です。

```bash
npm run dev
```

## 5. 環境変数の管理

### `.env.local`（開発環境）

```bash
# Firebase コンソール > プロジェクトの設定 > 全般 > マイアプリ から取得
NEXT_PUBLIC_FIREBASE_API_KEY=AIzaSy...
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=myapp-dev.firebaseapp.com
NEXT_PUBLIC_FIREBASE_PROJECT_ID=myapp-dev
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=myapp-dev.appspot.com
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=123456789
NEXT_PUBLIC_FIREBASE_APP_ID=1:123456789:web:abcdef

# Admin SDK 用（サーバーサイドのみ・絶対にクライアントに渡さないこと）
FIREBASE_ADMIN_PROJECT_ID=myapp-dev
FIREBASE_ADMIN_CLIENT_EMAIL=firebase-adminsdk@myapp-dev.iam.gserviceaccount.com
FIREBASE_ADMIN_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----\n..."
```

> ⚠️ `NEXT_PUBLIC_` が付いた変数はブラウザから見えます。Firebase の場合、API Key はブラウザ公開前提の設計です（セキュリティルールで保護）。  
> **Admin SDK のキーは絶対にクライアントに渡さないこと。**

### `.gitignore` への追記確認

```bash
# .gitignore に以下が含まれているか確認
.env.local
.env.*.local
```

### ローカルエミュレーターの活用（推奨）

本番 Firebase を使わず、ローカルでテストできます。

```bash
# エミュレーターの初期化
firebase init emulators

# エミュレーターの起動（Auth・Firestore・Functions）
firebase emulators:start
```

```typescript
// エミュレーター接続の設定（開発環境のみ）
import { connectAuthEmulator } from 'firebase/auth'
import { connectFirestoreEmulator } from 'firebase/firestore'

if (process.env.NODE_ENV === 'development') {
  connectAuthEmulator(auth, 'http://localhost:9099')
  connectFirestoreEmulator(db, 'localhost', 8080)
}
```
