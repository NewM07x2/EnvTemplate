## 1. Firebase Emulator Suite とは

Firebase Emulator Suite は、Firebase の各サービスをローカル PC 上で再現する開発ツールです。

```
┌─────────────────────────────────────────────────────────┐
│               ローカル開発環境                             │
│                                                           │
│  Next.js アプリ                                           │
│  ┌─────────────┐    接続    ┌─────────────────────────┐  │
│  │ Firebase    │  ────────→ │ Firebase Emulator Suite  │  │
│  │ Client SDK  │            │                          │  │
│  │ Admin SDK   │            │  ・Firestore Emulator    │  │
│  └─────────────┘            │  ・Auth Emulator         │  │
│                              │  ・Storage Emulator      │  │
│                              │  ・Functions Emulator    │  │
│                              │  ・Hosting Emulator      │  │
│                              │  ・Pub/Sub Emulator      │  │
│                              └─────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
```

### 使うメリット

| メリット | 説明 |
|---------|------|
| 🆓 コスト0 | 本番 Firebase へのアクセスが発生しない |
| 🔒 安全 | 本番データを誤って変更・削除するリスクなし |
| ⚡ 高速 | ネットワーク遅延なしでローカル完結 |
| 🧪 テスト向き | セキュリティルールのユニットテストに使用可 |
| 🔌 オフライン開発 | インターネット接続なしで開発できる |

## 2. 初期セットアップ

### Firebase CLI のインストール

```bash
npm install -g firebase-tools

# バージョン確認
firebase --version  # 13.x 以上を推奨
```

### Firebase プロジェクトへのログインと初期化

```bash
# Google アカウントでログイン
firebase login

# プロジェクトディレクトリで初期化
cd your-project
firebase init emulators
```


## 4. Next.js からエミュレーターに接続する

### 環境変数でエミュレーターを切り替える

```bash
# .env.local（ローカル開発用）
NEXT_PUBLIC_USE_FIREBASE_EMULATOR=true

# .env.production（本番用・エミュレーター無効）
NEXT_PUBLIC_USE_FIREBASE_EMULATOR=false
```

### Client SDK の接続設定（`src/lib/firebase/client.ts`）

```typescript
import { initializeApp, getApps, getApp } from 'firebase/app';
import { getAuth, connectAuthEmulator } from 'firebase/auth';
import { getFirestore, connectFirestoreEmulator } from 'firebase/firestore';
import { getStorage, connectStorageEmulator } from 'firebase/storage';
import { getFunctions, connectFunctionsEmulator } from 'firebase/functions';

const firebaseConfig = {
  apiKey:            process.env.NEXT_PUBLIC_FIREBASE_API_KEY!,
  authDomain:        process.env.NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN!,
  projectId:         process.env.NEXT_PUBLIC_FIREBASE_PROJECT_ID!,
  storageBucket:     process.env.NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET!,
  messagingSenderId: process.env.NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID!,
  appId:             process.env.NEXT_PUBLIC_FIREBASE_APP_ID!,
};

const app = getApps().length > 0 ? getApp() : initializeApp(firebaseConfig);

export const auth      = getAuth(app);
export const db        = getFirestore(app);
export const storage   = getStorage(app);
export const functions = getFunctions(app, 'asia-northeast1');

// エミュレーターへの接続（開発環境のみ）
if (
  process.env.NEXT_PUBLIC_USE_FIREBASE_EMULATOR === 'true' &&
  typeof window !== 'undefined'
) {
  // 重複接続を防ぐフラグ（ホットリロード対策）
  const globalWithEmulator = global as typeof global & {
    _emulatorConnected?: boolean;
  };

  if (!globalWithEmulator._emulatorConnected) {
    connectAuthEmulator(auth, 'http://127.0.0.1:9099', { disableWarnings: true });
    connectFirestoreEmulator(db, '127.0.0.1', 8080);
    connectStorageEmulator(storage, '127.0.0.1', 9199);
    connectFunctionsEmulator(functions, '127.0.0.1', 5001);
    globalWithEmulator._emulatorConnected = true;
    console.log('🔥 Firebase Emulators に接続しました');
  }
}
```

### Admin SDK の接続設定（`src/lib/firebase/admin.ts`）

```typescript
import { initializeApp, getApps, cert } from 'firebase-admin/app';
import { getAuth } from 'firebase-admin/auth';
import { getFirestore } from 'firebase-admin/firestore';

if (getApps().length === 0) {
  if (process.env.NEXT_PUBLIC_USE_FIREBASE_EMULATOR === 'true') {
    // エミュレーター用: サービスアカウントキー不要
    process.env.FIRESTORE_EMULATOR_HOST = '127.0.0.1:8080';
    process.env.FIREBASE_AUTH_EMULATOR_HOST = '127.0.0.1:9099';

    initializeApp({
      projectId: process.env.NEXT_PUBLIC_FIREBASE_PROJECT_ID!,
    });
  } else {
    // 本番環境用
    initializeApp({
      credential: cert({
        projectId:   process.env.FIREBASE_ADMIN_PROJECT_ID!,
        clientEmail: process.env.FIREBASE_ADMIN_CLIENT_EMAIL!,
        privateKey:  process.env.FIREBASE_ADMIN_PRIVATE_KEY!.replace(/\\n/g, '\n'),
      }),
    });
  }
}

export const adminAuth = getAuth();
export const adminDb   = getFirestore();
```

### package.json にスクリプトを追加

```json
{
  "scripts": {
    "dev": "next dev",
    "dev:emulator": "firebase emulators:start --import=./emulator-data --export-on-exit=./emulator-data & next dev",
    "emulators": "firebase emulators:start --import=./emulator-data --export-on-exit=./emulator-data",
    "build": "next build",
    "start": "next start"
  }
}
```

> **`--import` / `--export-on-exit`**: エミュレーターのデータを `./emulator-data` フォルダに保存・復元します。セッション間でデータが維持されます。

## 5. Emulator UI の使い方

エミュレーター起動後、ブラウザで `http://localhost:4000` にアクセスします。

```
http://localhost:4000         → Emulator UI トップ
http://localhost:4000/firestore → Firestore データブラウザ
http://localhost:4000/auth      → 認証ユーザー一覧
http://localhost:4000/storage   → Storage ファイルブラウザ
http://localhost:4000/functions → Functions ログ
```

### Emulator UI でできること

| 機能 | 操作 |
|------|------|
| ドキュメントの閲覧・編集・削除 | Firestore タブ |
| テストユーザーの作成・削除 | Auth タブ |
| ファイルのアップロード・削除 | Storage タブ |
| 関数の実行ログ確認 | Functions タブ |
| データの一括クリア | 各タブの「Clear all data」ボタン |

## 6. テストデータのシードとエクスポート

### シードスクリプトの作成

```typescript
// scripts/seed.ts
import { initializeApp } from 'firebase/app';
import { getFirestore, connectFirestoreEmulator, collection, addDoc, Timestamp } from 'firebase/firestore';
import { getAuth, connectAuthEmulator, createUserWithEmailAndPassword } from 'firebase/auth';

const app = initializeApp({
  apiKey:    'fake-api-key', // エミュレーター用はダミー値で OK
  projectId: 'your-project-id',
});

const db   = getFirestore(app);
const auth = getAuth(app);

connectFirestoreEmulator(db, '127.0.0.1', 8080);
connectAuthEmulator(auth, 'http://127.0.0.1:9099');

async function seed() {
  console.log('🌱 シードデータを投入中...');

  // テストユーザーの作成
  const { user } = await createUserWithEmailAndPassword(
    auth,
    'test@example.com',
    'password123'
  );
  console.log(`✅ テストユーザー作成: ${user.uid}`);

  // テスト投稿の作成
  const postsRef = collection(db, 'posts');
  for (let i = 1; i <= 5; i++) {
    await addDoc(postsRef, {
      title:     `テスト投稿 ${i}`,
      content:   `これはテスト投稿 ${i} の内容です。`,
      authorId:  user.uid,
      published: true,
      createdAt: Timestamp.now(),
    });
  }

  console.log('✅ シードデータの投入完了！');
  process.exit(0);
}

seed().catch(console.error);
```

```bash
# シードスクリプトの実行（エミュレーター起動中に実行）
npx ts-node scripts/seed.ts
```

### データのエクスポートとインポート

```bash
# 現在のエミュレーターデータをエクスポート
firebase emulators:export ./emulator-data

# エクスポートしたデータを読み込んでエミュレーターを起動
firebase emulators:start --import=./emulator-data

# 終了時に自動エクスポート
firebase emulators:start --import=./emulator-data --export-on-exit=./emulator-data
```

> **チームでの共有**: `emulator-data` フォルダを Git 管理に含めれば、チーム全員が同じテストデータでローカル開発できます。

## 7. CI/CD でエミュレーターを使う

### GitHub Actions での設定例

```yaml
# .github/workflows/test.yml
name: Firebase Tests

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Node.js セットアップ
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      - name: 依存関係のインストール
        run: npm ci

      - name: Firebase CLI のインストール
        run: npm install -g firebase-tools

      - name: Java のセットアップ（エミュレーターに必要）
        uses: actions/setup-java@v4
        with:
          distribution: 'temurin'
          java-version: '17'

      - name: エミュレーターでテスト実行
        run: |
          firebase emulators:exec \
            --only firestore,auth,storage \
            --project demo-test \
            "npx jest"
        env:
          NEXT_PUBLIC_FIREBASE_PROJECT_ID: demo-test
          NEXT_PUBLIC_USE_FIREBASE_EMULATOR: "true"
```

> **`firebase emulators:exec`**: エミュレーターを起動してコマンドを実行し、完了後に自動停止します。

## 8. よくある問題と解決策

### ❌ エミュレーターが起動しない（Java が見つからない）

```
Error: Could not spawn `java -version`: spawn java ENOENT
```

```bash
# 解決策: Java 11 以上をインストール
# Windows: https://adoptium.net/ からダウンロード
# macOS: brew install openjdk@17
# 確認
java -version
```

---

### ❌ ポートが既に使用中

```
⚠  Port 8080 is not open on localhost, could not start Firestore Emulator.
```

```bash
# Windows: ポートを使用しているプロセスを確認・終了
netstat -ano | findstr :8080
taskkill /PID <PID番号> /F

# または firebase.json でポートを変更
```

---

### ❌ ホットリロード時に「Firestore has already been started」エラー

```typescript
// 解決策: グローバルフラグで重複接続を防ぐ（上記の client.ts を参照）
const globalWithEmulator = global as typeof global & { _emulatorConnected?: boolean };
if (!globalWithEmulator._emulatorConnected) {
  connectFirestoreEmulator(db, '127.0.0.1', 8080);
  globalWithEmulator._emulatorConnected = true;
}
```

---

### ❌ エミュレーターへの接続時に Auth エラーが出る

```
Error: Firebase: Error (auth/network-request-failed)
```

```bash
# 解決策: 127.0.0.1 と localhost の混在を避ける
# connectAuthEmulator の URL を統一する
connectAuthEmulator(auth, 'http://127.0.0.1:9099');  # ← 'localhost' でなく '127.0.0.1'
```

---

### ❌ 関数のエミュレーターが TypeScript を認識しない

```bash
# functions ディレクトリで TypeScript をビルドしてから起動
cd functions
npm run build  # または npx tsc

# ウォッチモードで起動（コード変更時に自動リビルド）
firebase emulators:start --only functions
# 別ターミナルで
cd functions && npm run build:watch
```

## まとめ

| コマンド | 説明 |
|---------|------|
| `firebase emulators:start` | エミュレーター起動（Ctrl+C で停止） |
| `firebase emulators:start --only firestore,auth` | 特定のエミュレーターのみ起動 |
| `firebase emulators:start --import=./emulator-data` | 保存データを読み込んで起動 |
| `firebase emulators:export ./emulator-data` | 現在のデータをエクスポート |
| `firebase emulators:exec "npm test"` | エミュレーター起動→テスト実行→停止 |
