## 1. Firestore のデータ移行の基本方針

Firestore には SQL の `ALTER TABLE` のような「スキーマ変更コマンド」はありません。データ構造の変更は **アプリケーションコードとデータ移行スクリプト** で対応します。

```
Firestore のデータ変更の 3 つのアプローチ

① レイジーマイグレーション（段階的）
   変更前データを読み取るたびに新形式に変換して再書き込み
   → 小規模・低リスクだが全データが更新されるまで時間がかかる

② バッチマイグレーション（一括）
   スクリプトで全ドキュメントを一括変換
   → 確実だが大量の読み書きコストが発生する

③ デュアルライト（並行運用）
   新旧両方の形式で書き込みを続けながら段階移行
   → ゼロダウンタイムだが複雑度が上がる
```

## 2. スキーマ変更パターン

### パターン①: フィールドの追加

フィールドの追加は最も簡単です。既存ドキュメントにフィールドがない場合のデフォルト値をアプリコードで処理します。

```typescript
// 変更前: { name: string, email: string }
// 変更後: { name: string, email: string, role: string } を追加

// アプリ側でデフォルト値を処理（マイグレーション不要）
type User = {
  name: string;
  email: string;
  role?: string; // Optional にしておく
};

function getUserRole(user: User): string {
  return user.role ?? 'member'; // なければデフォルト値を返す
}
```

### パターン②: フィールドのリネーム

```typescript
// 変更前: { firstName: string, lastName: string }
// 変更後: { displayName: string }

// マイグレーションスクリプトで変換が必要
async function migrateUserNames() {
  const snapshot = await adminDb.collection('users').get();
  const batch = adminDb.batch();
  let count = 0;

  for (const doc of snapshot.docs) {
    const data = doc.data();
    if (data.firstName !== undefined) { // 未移行のドキュメントのみ処理
      batch.update(doc.ref, {
        displayName: `${data.firstName} ${data.lastName}`,
        firstName: FieldValue.delete(), // 旧フィールドを削除
        lastName:  FieldValue.delete(),
      });
      count++;

      // Firestore バッチは 500 件が上限
      if (count % 500 === 0) {
        await batch.commit();
        console.log(`${count} 件処理済み`);
      }
    }
  }

  if (count % 500 !== 0) await batch.commit();
  console.log(`完了: 合計 ${count} 件`);
}
```

### パターン③: ネスト構造の変更

```typescript
// 変更前: { address: string }
// 変更後: { address: { street: string, city: string, zipCode: string } }

async function migrateAddressField() {
  const snapshot = await adminDb.collection('users').get();
  const batch = adminDb.batch();
  let batchCount = 0;
  let totalCount = 0;

  for (const doc of snapshot.docs) {
    const data = doc.data();

    // address が文字列のドキュメントのみ移行
    if (typeof data.address === 'string') {
      batch.update(doc.ref, {
        address: {
          street:  data.address, // 既存の文字列をstreetに移動
          city:    '',
          zipCode: '',
        },
      });
      batchCount++;
      totalCount++;

      if (batchCount === 500) {
        await batch.commit();
        batchCount = 0;
        console.log(`${totalCount} 件処理済み`);
      }
    }
  }

  if (batchCount > 0) await batch.commit();
  console.log(`完了: 合計 ${totalCount} 件`);
}
```

## 3. バッチマイグレーションスクリプト

### 汎用マイグレーションフレームワーク

```typescript
// scripts/migrate.ts
import { initializeApp, cert } from 'firebase-admin/app';
import { getFirestore, FieldValue } from 'firebase-admin/firestore';
import * as dotenv from 'dotenv';

dotenv.config({ path: '.env.local' });

initializeApp({
  credential: cert({
    projectId:   process.env.FIREBASE_ADMIN_PROJECT_ID!,
    clientEmail: process.env.FIREBASE_ADMIN_CLIENT_EMAIL!,
    privateKey:  process.env.FIREBASE_ADMIN_PRIVATE_KEY!.replace(/\\n/g, '\n'),
  }),
});

const db = getFirestore();

// マイグレーション定義
interface Migration {
  id: string;        // 一意の ID（例: '2024-01-15_add-role-field'）
  description: string;
  up: () => Promise<void>;   // 移行処理
  down?: () => Promise<void>; // ロールバック処理
}

// マイグレーション実行 + 履歴記録
async function runMigration(migration: Migration) {
  const migrationRef = db.collection('_migrations').doc(migration.id);
  const migrationDoc = await migrationRef.get();

  if (migrationDoc.exists) {
    console.log(`⏭  ${migration.id}: 実行済みのためスキップ`);
    return;
  }

  console.log(`▶  ${migration.id}: 実行開始...`);
  const startTime = Date.now();

  try {
    await migration.up();
    await migrationRef.set({
      id:          migration.id,
      description: migration.description,
      executedAt:  new Date(),
      durationMs:  Date.now() - startTime,
    });
    console.log(`✅ ${migration.id}: 完了 (${Date.now() - startTime}ms)`);
  } catch (error) {
    console.error(`❌ ${migration.id}: エラー`, error);
    throw error;
  }
}

// ── マイグレーション定義 ────────────────────────────────────
const migrations: Migration[] = [
  {
    id: '2024-01-15_add-role-to-users',
    description: 'users コレクションに role フィールドを追加',
    async up() {
      const snapshot = await db.collection('users').get();
      const batchOps: Promise<void>[] = [];
      let batch = db.batch();
      let count = 0;

      for (const doc of snapshot.docs) {
        if (doc.data().role === undefined) {
          batch.update(doc.ref, { role: 'member' });
          count++;
          if (count % 500 === 0) {
            batchOps.push(batch.commit().then(() => undefined));
            batch = db.batch();
          }
        }
      }
      if (count % 500 !== 0) batchOps.push(batch.commit().then(() => undefined));
      await Promise.all(batchOps);
      console.log(`  → ${count} 件更新`);
    },
  },
  {
    id: '2024-02-01_rename-createdDate-to-createdAt',
    description: 'posts の createdDate フィールドを createdAt にリネーム',
    async up() {
      const snapshot = await db.collection('posts').get();
      let batch = db.batch();
      let count = 0;

      for (const doc of snapshot.docs) {
        const data = doc.data();
        if (data.createdDate !== undefined) {
          batch.update(doc.ref, {
            createdAt:   data.createdDate,
            createdDate: FieldValue.delete(),
          });
          count++;
          if (count % 500 === 0) {
            await batch.commit();
            batch = db.batch();
          }
        }
      }
      if (count % 500 !== 0) await batch.commit();
      console.log(`  → ${count} 件更新`);
    },
  },
];

// ── エントリーポイント ─────────────────────────────────────
async function main() {
  console.log('🔥 マイグレーション開始');
  for (const migration of migrations) {
    await runMigration(migration);
  }
  console.log('🎉 すべてのマイグレーション完了');
  process.exit(0);
}

main().catch((err) => {
  console.error('致命的エラー:', err);
  process.exit(1);
});
```

## 4. Firestore のエクスポート・インポート

### Google Cloud CLI を使ったエクスポート（推奨）

```bash
# Google Cloud CLI のインストール
# https://cloud.google.com/sdk/docs/install

# プロジェクトを設定
gcloud config set project your-project-id

# Cloud Storage バケットにエクスポート
gcloud firestore export gs://your-project-id.appspot.com/backups/2024-01-15

# 特定のコレクションのみエクスポート
gcloud firestore export gs://your-project-id.appspot.com/backups/2024-01-15 \
  --collection-ids=users,posts,comments

# インポート
gcloud firestore import gs://your-project-id.appspot.com/backups/2024-01-15
```

### Admin SDK を使った手動エクスポート

```typescript
// scripts/export.ts
import { initializeApp, cert } from 'firebase-admin/app';
import { getFirestore } from 'firebase-admin/firestore';
import * as fs from 'fs';

// ... initializeApp

const db = getFirestore();

async function exportCollection(collectionName: string) {
  const snapshot = await db.collection(collectionName).get();
  const data = snapshot.docs.map((doc) => ({ id: doc.id, ...doc.data() }));

  const filename = `export_${collectionName}_${new Date().toISOString().slice(0, 10)}.json`;
  fs.writeFileSync(filename, JSON.stringify(data, null, 2));
  console.log(`✅ ${collectionName}: ${data.length} 件 → ${filename}`);
}

async function main() {
  await exportCollection('users');
  await exportCollection('posts');
  process.exit(0);
}
main();
```

### 定期バックアップ（Cloud Functions × Cloud Scheduler）

```typescript
// functions/src/backup.ts
import { onSchedule } from 'firebase-functions/v2/scheduler';
import { GoogleAuth } from 'google-auth-library';

export const dailyBackup = onSchedule(
  { schedule: '0 2 * * *', timeZone: 'Asia/Tokyo' }, // 毎日 02:00
  async () => {
    const projectId = process.env.GCLOUD_PROJECT;
    const bucket    = `gs://${projectId}-backups`;
    const timestamp = new Date().toISOString().slice(0, 10);

    const auth   = new GoogleAuth({ scopes: ['https://www.googleapis.com/auth/cloud-platform'] });
    const client = await auth.getClient();

    await client.request({
      url:    `https://firestore.googleapis.com/v1/projects/${projectId}/databases/(default):exportDocuments`,
      method: 'POST',
      data:   { outputUriPrefix: `${bucket}/${timestamp}` },
    });

    console.log(`✅ バックアップ完了: ${bucket}/${timestamp}`);
  }
);
```

## 5. 他サービスから Firebase への移行

### MySQL / PostgreSQL → Firestore

```typescript
// scripts/migrate-from-mysql.ts
import mysql from 'mysql2/promise';
import { initializeApp, cert } from 'firebase-admin/app';
import { getFirestore, Timestamp } from 'firebase-admin/firestore';

// ... initializeApp

const db = getFirestore();

async function migrateUsers() {
  const conn = await mysql.createConnection({
    host: 'localhost', database: 'myapp', user: 'root', password: 'password',
  });

  const [rows] = await conn.execute('SELECT * FROM users');
  const users = rows as Array<{
    id: number; name: string; email: string; created_at: Date;
  }>;

  let batch = db.batch();
  let count = 0;

  for (const user of users) {
    const docRef = db.collection('users').doc(String(user.id));
    batch.set(docRef, {
      name:      user.name,
      email:     user.email,
      createdAt: Timestamp.fromDate(user.created_at),
      // MySQL の id は Firestore のドキュメント ID として使用
    });
    count++;

    if (count % 500 === 0) {
      await batch.commit();
      batch = db.batch();
      console.log(`${count} 件処理済み`);
    }
  }

  if (count % 500 !== 0) await batch.commit();
  await conn.end();
  console.log(`✅ ユーザー移行完了: 合計 ${count} 件`);
}

migrateUsers();
```

### Supabase → Firestore

```typescript
// scripts/migrate-from-supabase.ts
import { createClient } from '@supabase/supabase-js';
import { getFirestore, Timestamp } from 'firebase-admin/firestore';

const supabase = createClient(
  process.env.SUPABASE_URL!,
  process.env.SUPABASE_SERVICE_ROLE_KEY! // Service Role Key で全データ取得
);

const db = getFirestore();

async function migratePosts() {
  let page = 0;
  const pageSize = 1000;
  let totalCount = 0;

  while (true) {
    const { data, error } = await supabase
      .from('posts')
      .select('*')
      .range(page * pageSize, (page + 1) * pageSize - 1);

    if (error) throw error;
    if (!data || data.length === 0) break;

    let batch = db.batch();
    let batchCount = 0;

    for (const post of data) {
      const docRef = db.collection('posts').doc(post.id);
      batch.set(docRef, {
        title:     post.title,
        content:   post.content,
        authorId:  post.author_id,       // snake_case → camelCase
        published: post.is_published,
        createdAt: Timestamp.fromDate(new Date(post.created_at)),
        updatedAt: Timestamp.fromDate(new Date(post.updated_at)),
      });
      batchCount++;

      if (batchCount === 500) {
        await batch.commit();
        batch = db.batch();
        batchCount = 0;
      }
    }

    if (batchCount > 0) await batch.commit();
    totalCount += data.length;
    console.log(`${totalCount} 件処理済み`);
    page++;
  }

  console.log(`✅ 投稿移行完了: 合計 ${totalCount} 件`);
}
```

## 6. Authentication のユーザー移行

### 他サービスから Firebase Auth へのユーザー移行

Firebase Admin SDK の `importUsers` を使うとパスワードハッシュごとユーザーを移行できます。

```typescript
// scripts/migrate-auth.ts
import { getAuth, UserImportRecord } from 'firebase-admin/auth';

async function importUsersFromCsv() {
  // 移行するユーザーのリスト（実際は CSV や DB から取得）
  const users: UserImportRecord[] = [
    {
      uid:          'user-001',
      email:        'alice@example.com',
      emailVerified: true,
      displayName:  'Alice',
      // BCrypt ハッシュのパスワードをそのまま移行可能
      passwordHash: Buffer.from('$2b$10$...'),
      passwordSalt: Buffer.from('...'),
    },
    {
      uid:   'user-002',
      email: 'bob@example.com',
      emailVerified: false,
      displayName:  'Bob',
      // パスワードなし（ソーシャルログイン専用ユーザーなど）
    },
  ];

  const result = await getAuth().importUsers(users, {
    hash: {
      algorithm: 'BCRYPT', // 元のハッシュアルゴリズムに合わせる
    },
  });

  console.log(`✅ ${result.successCount} 件移行成功`);
  if (result.failureCount > 0) {
    console.error('失敗:', result.errors);
  }
}
```

### 対応しているハッシュアルゴリズム

| アルゴリズム | 定数 |
|------------|------|
| BCrypt | `BCRYPT` |
| SCRYPT | `SCRYPT` |
| PBKDF2_SHA256 | `PBKDF2_SHA256` |
| SHA256 | `SHA256` |
| MD5 | `MD5` |
| HMAC_SHA256 | `HMAC_SHA256` |

## 7. Cloud Storage のファイル移行

```typescript
// scripts/migrate-storage.ts
import { getStorage } from 'firebase-admin/storage';
import axios from 'axios';

const bucket = getStorage().bucket();

async function downloadAndUpload(sourceUrl: string, destPath: string) {
  // 元の URL からファイルをダウンロード
  const response = await axios.get(sourceUrl, { responseType: 'arraybuffer' });
  const buffer = Buffer.from(response.data);

  // Firebase Storage にアップロード
  const file = bucket.file(destPath);
  await file.save(buffer, {
    contentType: response.headers['content-type'],
    metadata: {
      metadata: { migratedFrom: sourceUrl, migratedAt: new Date().toISOString() },
    },
  });

  return await file.getSignedUrl({ action: 'read', expires: '03-01-2500' });
}

// Supabase Storage → Firebase Storage
async function migrateFiles() {
  const files = [
    { src: 'https://xxx.supabase.co/storage/v1/object/public/avatars/user1.jpg', dest: 'avatars/user1.jpg' },
    // ...
  ];

  for (const { src, dest } of files) {
    try {
      const [url] = await downloadAndUpload(src, dest);
      console.log(`✅ ${dest}: ${url}`);
    } catch (err) {
      console.error(`❌ ${dest}:`, err);
    }
  }
}
```

## 8. ゼロダウンタイム移行のパターン

本番環境でサービスを止めずにデータ構造を変更するパターンです。

```
フェーズ 1: デュアルライト
  ・新形式と旧形式の両方に書き込む
  ・読み取りは旧形式を優先（新形式がなければ旧形式を返す）

フェーズ 2: バックフィル（バックグラウンドマイグレーション）
  ・旧形式のみのドキュメントをバックグラウンドで順次変換

フェーズ 3: 新形式へ切り替え
  ・全データが新形式に移行完了後、読み取りを新形式に切り替え

フェーズ 4: 旧フィールドの削除
  ・一定期間後、旧形式のフィールドを削除
```

```typescript
// フェーズ 1: デュアルライトの実装例
async function savePost(postId: string, data: {
  title: string;
  authorId: string;
}) {
  await db.collection('posts').doc(postId).set({
    title:      data.title,
    authorId:   data.authorId,  // 新形式
    author_id:  data.authorId,  // 旧形式（互換性のため一時的に残す）
    schemaVersion: 2,           // バージョン管理
  });
}

// 読み取り時の互換処理
function normalizePost(data: FirebaseFirestore.DocumentData) {
  return {
    title:    data.title,
    authorId: data.authorId ?? data.author_id, // 新形式がなければ旧形式を使用
  };
}
```

## 9. ロールバック戦略

### バックアップからのリストア

```bash
# エクスポート（移行前に必ず実行）
gcloud firestore export gs://your-project-id-backups/pre-migration-$(date +%Y%m%d)

# ロールバック（移行失敗時）
gcloud firestore import gs://your-project-id-backups/pre-migration-20240115
```

### マイグレーションスクリプトに `down()` を実装

```typescript
const migration: Migration = {
  id: '2024-01-15_add-role',
  description: 'role フィールドを追加',
  async up() {
    // 移行処理
    const snapshot = await db.collection('users').get();
    // ... role: 'member' を追加
  },
  async down() {
    // ロールバック処理
    const snapshot = await db.collection('users').get();
    let batch = db.batch();
    let count = 0;
    for (const doc of snapshot.docs) {
      batch.update(doc.ref, { role: FieldValue.delete() });
      count++;
      if (count % 500 === 0) { await batch.commit(); batch = db.batch(); }
    }
    if (count % 500 !== 0) await batch.commit();
    // 移行履歴を削除
    await db.collection('_migrations').doc(migration.id).delete();
  },
};
```

## まとめ

| 状況 | 推奨アプローチ |
|------|-------------|
| フィールドの追加 | アプリ側でデフォルト値処理（マイグレーション不要） |
| フィールドのリネーム・削除 | バッチマイグレーションスクリプト |
| 本番稼働中の大規模変更 | ゼロダウンタイム移行（デュアルライト） |
| 定期バックアップ | Cloud Functions + Cloud Scheduler |
| 他 DB からの移行 | Admin SDK のバッチ書き込み（500件単位） |
| ユーザーの移行 | `importUsers()` でパスワードハッシュごと移行 |
