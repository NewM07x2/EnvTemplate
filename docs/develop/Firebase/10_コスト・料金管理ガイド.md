## 1. Firebase の料金プラン

| プラン | 名称 | 月額 | 特徴 |
|-------|------|------|------|
| **Spark** | 無料プラン | $0 | 無料枠内のみ利用可。Cloud Functions は使用不可 |
| **Blaze** | 従量課金プラン | 無料枠 + 超過分を従量 | Cloud Functions が使用可能。無料枠は Spark と同じ |

> **Blaze プランへの移行タイミング**: Cloud Functions や外部 API 呼び出しが必要になったとき。無料枠を超えなければ追加料金は発生しないため、早めに移行しても問題ありません。

## 3. コストが発生しやすいケースと対策

### ❗ ケース①: onSnapshot の多用による読み取り爆発

```typescript
// NG: ページに10個の onSnapshot があり、10ユーザーが同時接続
// → 1変更で 10 × 10 = 100 回の読み取りが発生
useEffect(() => {
  const unsub1 = onSnapshot(collection(db, 'posts'), callback);
  const unsub2 = onSnapshot(collection(db, 'users'), callback);
  // ...
}, []);
```

```typescript
// ✅ 対策: 必要最小限の範囲に限定
useEffect(() => {
  // コレクション全体でなく、自分の投稿のみ購読
  const q = query(
    collection(db, 'posts'),
    where('authorId', '==', currentUser.uid),
    limit(20) // 件数を制限
  );
  const unsub = onSnapshot(q, callback);
  return () => unsub();
}, [currentUser.uid]);
```

---

### ❗ ケース②: セキュリティルールの `get()` 呼び出し

```
// NG: リクエストのたびに users ドキュメントを読み取る（隠れた課金）
function isAdmin() {
  return get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin';
}
```

```
// ✅ 対策: カスタムクレームを使ってルールを簡略化（読み取り不要）
function isAdmin() {
  return request.auth.token.role == 'admin';
}
```

---

### ❗ ケース③: Cloud Functions の無駄な起動

```typescript
// NG: onDocumentWritten は create/update/delete すべてで起動する
export const onPostChanged = onDocumentWritten('posts/{postId}', async (event) => {
  // 重い処理（毎回の write で起動）
  await someHeavyOperation();
});
```

```typescript
// ✅ 対策: 必要なトリガーだけに限定
export const onPostCreated = onDocumentCreated('posts/{postId}', async (event) => {
  await someHeavyOperation();
});
```

### ❗ ケース④: Storage の画像を最適化せずに配信

画像を元サイズのまま配信すると、ダウンロード転送量が急増します。

```typescript
// ✅ 対策: Cloud Functions で画像を自動リサイズ
import { onObjectFinalized } from 'firebase-functions/v2/storage';
import sharp from 'sharp';

export const resizeImage = onObjectFinalized(async (event) => {
  const { bucket, name } = event.data;
  if (!name || name.includes('_thumb')) return; // 再帰防止

  const storageRef = getStorage().bucket(bucket);
  const [inputBuffer] = await storageRef.file(name!).download();

  const thumbBuffer = await sharp(inputBuffer)
    .resize(400, 400, { fit: 'inside' })
    .webp({ quality: 80 })
    .toBuffer();

  const thumbPath = name!.replace(/(\.[^.]+)$/, '_thumb.webp');
  await storageRef.file(thumbPath).save(thumbBuffer, {
    contentType: 'image/webp',
  });
});
```

## 4. コスト削減のベストプラクティス

### Firestore

| 対策 | 効果 |
|------|------|
| `limit()` でクエリ件数を制限 | 不要な読み取りを削減 |
| フィールドを絞る（`select()`） | 転送量を削減（読み取り回数は変わらない） |
| ページネーション（`startAfter`） | 一度に大量取得しない |
| オフラインキャッシュを活用 | 同じデータの再取得を防ぐ |
| `onSnapshot` の範囲を最小化 | リアルタイム更新の読み取りを削減 |
| バッチ書き込み（`writeBatch`） | 書き込み操作をまとめる（回数は同じだがトランザクション削減） |

### Cloud Functions

| 対策 | 効果 |
|------|------|
| メモリ・CPU を最小必要量に設定 | コンピュート時間コストを削減 |
| タイムアウトを短く設定 | 無限ループを防ぐ |
| idempotency（べき等性）を実装 | 再試行による二重課金を防ぐ |
| `runWith({ maxInstances: N })` | 急激なスケールアウトを防ぐ |

### Cloud Storage

| 対策 | 効果 |
|------|------|
| 画像をアップロード前にリサイズ・圧縮 | 容量・転送量を削減 |
| 使われていないファイルを定期削除 | ストレージ容量を削減 |
| CDN キャッシュ（Firebase Hosting 経由）を活用 | ダウンロード転送量を削減 |

## 5. 予算アラートの設定

Google Cloud コンソール から予算アラートを設定できます。

### 設定手順

1. [Google Cloud Console](https://console.cloud.google.com/) にアクセス
2. 左メニュー「お支払い」→「予算とアラート」
3. 「予算を作成」をクリック
4. 以下を設定：

```
予算名:        Firebase 月次予算
スコープ:      プロジェクト = [your-project-id]
予算タイプ:    指定した金額
金額:          $10（まずは小さい金額から）

アラートのしきい値:
  ✅ 50% に達したとき（$5）
  ✅ 90% に達したとき（$9）
  ✅ 100% に達したとき（$10）

通知:
  ✅ メールで通知（請求先アカウントの管理者）
```

## 6. コスト見積もりの方法

### Firebase 料金計算ツール

[Firebase Pricing Calculator](https://firebase.google.com/pricing) で月間使用量を入力して概算コストを確認できます。

### 見積もり例（中規模 Web アプリ）

```
ユーザー数:          1,000 MAU
DAU:                 200

Firestore:
  読み取り:          200 DAU × 50 reads/day × 30日 = 300,000 回/月
                     ↑ 無料枠 50,000 回超過 → 超過分 250,000 回 = $0.15
  書き込み:          200 DAU × 5 writes/day × 30日 = 30,000 回/月
                     ↑ 無料枠 20,000 回超過 → 超過分 10,000 回 = $0.018
  ストレージ:        100 MB → 無料枠内

Cloud Functions:
  呼び出し:          200,000 回/月 → 無料枠内 (200万回)

Cloud Storage:
  画像:              2 GB → 無料枠内 (5GB)

合計: 約 $0.17/月
```

### 使用量のモニタリング

```bash
# Firebase CLI でプロジェクトの使用状況を確認
firebase use [project-id]
firebase projects:list
```

[Firebase コンソール](https://console.firebase.google.com/) → 「使用量」タブで各サービスのリアルタイム使用量を確認できます。

## 7. 請求書の確認と分析

### 課金の確認場所

| 確認場所 | 内容 |
|---------|------|
| Firebase コンソール → 使用量 | サービス別のリアルタイム使用量 |
| Google Cloud Console → お支払い | 詳細な請求書・コスト分析 |
| Google Cloud Console → レポート | サービス・期間別のコスト推移グラフ |

### 予想外の課金が発生した場合のチェックリスト

- [ ] Firestore の読み取り数が急増していないか → `onSnapshot` の範囲を確認
- [ ] Cloud Functions が無限ループしていないか → ログを確認
- [ ] セキュリティルールで `get()` を多用していないか → カスタムクレームに移行
- [ ] Storage のダウンロード量が多くないか → 画像最適化・CDN キャッシュを確認
- [ ] SMS 認証（電話番号）の利用が多くないか → 不正利用を確認

## まとめ

| 優先度 | 対策 |
|-------|------|
| 🔴 高 | 予算アラートを設定する（まず $10 以下） |
| 🔴 高 | `onSnapshot` の範囲を必要最小限に絞る |
| 🟡 中 | セキュリティルールの `get()` をカスタムクレームに置き換える |
| 🟡 中 | 画像アップロード時にリサイズ・WebP 変換する |
| 🟢 低 | Firestore クエリに `limit()` を付ける |
| 🟢 低 | Cloud Functions のメモリ設定を最適化する |
