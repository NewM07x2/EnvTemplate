## 1. レイアウトの基本概念

### 1.1 レイアウトとは

レイアウトはダイアグラム内のノードとリンクを**自動的に配置する**アルゴリズムです。

```typescript
// ダイアグラム全体にレイアウトを適用
const diagram = new go.Diagram(divId, {
  layout: new go.TreeLayout(), // ← ここで指定
});

// レイアウトなし（手動配置）の場合は省略する
const diagram2 = new go.Diagram(divId, {
  // layout を指定しない → ノードは位置データに基づいて配置
});
```

### 1.2 レイアウトの共通プロパティ

| プロパティ | 型 | 説明 |
|---|---|---|
| `isOngoing` | `boolean` | ノード追加/削除時に自動再計算するか（デフォルト: `true`） |
| `isInitial` | `boolean` | 初回表示時にレイアウトを実行するか（デフォルト: `true`） |
| `isViewportSized` | `boolean` | ビューポートサイズに合わせるか |
| `arrangementOrigin` | `Point` | レイアウトの起点座標 |

### 1.3 手動でレイアウトを実行

```typescript
// 明示的にレイアウトを再実行
diagram.layoutDiagram(true);

// 特定のレイアウトを一回だけ適用
diagram.layout = new go.ForceDirectedLayout();
diagram.layoutDiagram(true);
```

## 2. TreeLayout（ツリーレイアウト）

**用途**: 組織図、ファイルツリー、マインドマップ、意思決定ツリー

```
        ┌───┐
        │ A │
        └─┬─┘
    ┌─────┼─────┐
  ┌─┴─┐ ┌─┴─┐ ┌─┴─┐
  │ B │ │ C │ │ D │
  └─┬─┘ └───┘ └─┬─┘
  ┌─┴─┐       ┌─┴─┐
  │ E │       │ F │
  └───┘       └───┘
```

### 2.1 基本設定

```typescript
const diagram = new go.Diagram(divId, {
  layout: new go.TreeLayout({
    angle: 90,            // 方向（0=右, 90=下, 180=左, 270=上）
    layerSpacing: 50,     // 層間の距離
    nodeSpacing: 20,      // 同一層内のノード間距離
    nodeIndent: 0,        // インデント量
    nodeIndentPastParent: 0.5, // 親からのインデント率
  }),
});
```

### 2.2 angle（方向）のバリエーション

| angle | 方向 | 用途 |
|---|---|---|
| `0` | 左→右 | ファイルツリー、マインドマップ |
| `90` | 上→下 | 組織図（最も一般的） |
| `180` | 右→左 | RTL レイアウト |
| `270` | 下→上 | ボトムアップ表示 |

### 2.3 TreeStyle（配置スタイル）

```typescript
// 標準（ルートが中央）
new go.TreeLayout({
  treeStyle: go.TreeStyle.Layered, // デフォルト
})

// 最後の子をルートの直下に並べる
new go.TreeLayout({
  treeStyle: go.TreeStyle.LastParents,
  alternateAngle: 90,
  alternateLayerSpacing: 30,
})

// 交互に展開
new go.TreeLayout({
  treeStyle: go.TreeStyle.Alternating,
})

// ルートの右側のみ
new go.TreeLayout({
  treeStyle: go.TreeStyle.RootOnly,
})
```

### 2.4 alignment（揃え方）

```typescript
// 子ノードの揃え方
new go.TreeLayout({
  alignment: go.TreeAlignment.CenterChildren,    // 子を中央揃え（デフォルト）
  // alignment: go.TreeAlignment.Start,           // 上/左揃え
  // alignment: go.TreeAlignment.End,             // 下/右揃え
  // alignment: go.TreeAlignment.Bus,             // バス型（両側に子を配置）
  // alignment: go.TreeAlignment.TopLeftBus,      // 上左バス型
  // alignment: go.TreeAlignment.BottomRightBus,  // 下右バス型
})
```

### 2.5 完全な例: 水平ツリー

```typescript
const diagram = new go.Diagram(divId, {
  layout: new go.TreeLayout({
    angle: 0,                // 左→右
    layerSpacing: 80,
    nodeSpacing: 10,
    alignment: go.TreeAlignment.Start,
    arrangement: go.TreeArrangement.FixedRoots,
    // 折りたたまれたサブツリーのソート
    sorting: go.TreeSorting.Ascending,
    comparer: (a: go.TreeVertex, b: go.TreeVertex) => {
      const aName = a.node?.data.name ?? "";
      const bName = b.node?.data.name ?? "";
      return aName.localeCompare(bName);
    },
  }),
});
```

## 3. ForceDirectedLayout（力学モデル）

**用途**: ネットワーク図、ソーシャルグラフ、関係性の可視化

```
      ┌───┐
   ┌──┤ A ├──┐
   │  └───┘  │
 ┌─┴─┐    ┌─┴─┐
 │ B │────│ C │
 └─┬─┘    └─┬─┘
   │  ┌───┐  │
   └──┤ D ├──┘
      └───┘
```

ノードをバネと電荷で物理シミュレーションし、自然な配置を生成します。

### 3.1 基本設定

```typescript
const diagram = new go.Diagram(divId, {
  layout: new go.ForceDirectedLayout({
    maxIterations: 300,           // 反復回数（多いほど精密）
    defaultSpringLength: 80,      // バネの自然長
    defaultSpringStiffness: 0.05, // バネの硬さ
    defaultElectricalCharge: 100, // ノードの反発力
    epsilonDistance: 1,           // 収束判定の閾値
    infinityDistance: 1000,       // 影響範囲
    arrangesToOrigin: true,       // 原点に集まるか
  }),
});
```

### 3.2 主要プロパティ

| プロパティ | デフォルト | 説明 |
|---|---|---|
| `maxIterations` | `100` | シミュレーションの反復回数 |
| `defaultSpringLength` | `50` | リンクの理想的な長さ |
| `defaultSpringStiffness` | `0.05` | バネの硬さ（0〜1） |
| `defaultElectricalCharge` | `150` | ノード間の反発力 |
| `epsilonDistance` | `1` | 移動量がこの値以下で収束と判定 |
| `infinityDistance` | `1000` | 電荷の影響が及ぶ最大距離 |
| `randomNumberGenerator` | `null` | 乱数生成器（再現性のため） |

### 3.3 パラメータ調整のガイド

```typescript
// ━━━ 密集したグラフ → ノード間隔を広げる ━━━
new go.ForceDirectedLayout({
  defaultElectricalCharge: 300,  // 反発力を上げる
  defaultSpringLength: 120,      // バネを長くする
})

// ━━━ 疎なグラフ → コンパクトにする ━━━
new go.ForceDirectedLayout({
  defaultElectricalCharge: 50,
  defaultSpringLength: 40,
  defaultSpringStiffness: 0.1,   // バネを硬くする
})

// ━━━ 大量ノード → 速度優先 ━━━
new go.ForceDirectedLayout({
  maxIterations: 50,             // 反復を減らす
  epsilonDistance: 5,            // 早めに収束判定
})
```

## 4. LayeredDigraphLayout（層状有向グラフ）

**用途**: フローチャート、BPMN、データパイプライン、ビルド依存関係

```
  ┌───┐   ┌───┐
  │ A │   │ B │    Layer 0
  └─┬─┘   └─┬─┘
    │   ┌───┐│
    └──►│ C ├┘     Layer 1
        └─┬─┘
    ┌─────┼─────┐
  ┌─┴─┐ ┌─┴─┐ ┌─┴─┐
  │ D │ │ E │ │ F │  Layer 2
  └───┘ └───┘ └───┘
```

### 4.1 基本設定

```typescript
const diagram = new go.Diagram(divId, {
  layout: new go.LayeredDigraphLayout({
    direction: 90,              // 方向（0=右, 90=下, 180=左, 270=上）
    layerSpacing: 50,           // 層間距離
    columnSpacing: 30,          // 同一層内の列間距離
    setsPortSpots: true,        // ポートの位置を自動設定
    // サイクル処理方式
    cycleRemoveOption: go.LayeredDigraphCycleRemove.DepthFirst,
    // 層割り当てアルゴリズム
    layeringOption: go.LayeredDigraphLayering.OptimalLinkLength,
    // 初期化方式
    initializeOption: go.LayeredDigraphInit.DepthFirstIn,
    // 交差最小化の反復回数
    iterations: 4,
    // 攻撃性（ノード配置の最適化の度合い）
    aggressiveOption: go.LayeredDigraphAggressive.More,
  }),
});
```

### 4.2 direction（方向）

| direction | フロー方向 | 用途例 |
|---|---|---|
| `0` | 左→右 | プロセスフロー |
| `90` | 上→下 | 組織図、フローチャート |
| `180` | 右→左 | RTL言語対応 |
| `270` | 下→上 | ボトムアップフロー |

### 4.3 PackOption（同一層内の圧縮方式）

```typescript
new go.LayeredDigraphLayout({
  packOption:
    go.LayeredDigraphPack.Expand |    // 可能な限り幅を広げる
    go.LayeredDigraphPack.Straighten, // リンクをなるべく直線にする
})
```

## 5. CircularLayout（円形レイアウト）

**用途**: ネットワークトポロジ、リングバッファ、循環参照の可視化

```
        ┌───┐
     ┌──┤ A ├──┐
   ┌─┴─┐└───┘┌─┴─┐
   │ E │      │ B │
   └─┬─┘      └─┬─┘
   ┌─┴─┐      ┌─┴─┐
   │ D │──────│ C │
   └───┘      └───┘
```

### 5.1 基本設定

```typescript
const diagram = new go.Diagram(divId, {
  layout: new go.CircularLayout({
    radius: 150,                  // 円の半径（NaN = 自動計算）
    startAngle: 0,                // 開始角度（度）
    sweepAngle: 360,              // 展開角度（360 = 完全な円）
    spacing: 10,                  // ノード間の最小距離
    arrangement: go.CircularArrangement.ConstantSpacing,
    direction: go.CircularDirection.Clockwise,
    sorting: go.CircularSorting.Forwards,
    // ノード直径の計算基準
    nodeDiameterFormula: go.CircularNodeDiameterFormula.Pythagorean,
  }),
});
```

### 5.2 arrangement（配置方式）

| 値 | 説明 |
|---|---|
| `ConstantSpacing` | ノード間を等間隔にする |
| `ConstantAngle` | 各ノードの角度を等間隔にする |
| `ConstantDistance` | ノード中心間の距離を等しくする |
| `Packed` | できるだけコンパクトに詰める |

### 5.3 半円・扇形の配置

```typescript
// 半円（上半分のみ）
new go.CircularLayout({
  startAngle: 180,
  sweepAngle: 180,
  radius: 200,
})

// 扇形（90度）
new go.CircularLayout({
  startAngle: 0,
  sweepAngle: 90,
  radius: 200,
})
```

## 6. GridLayout（グリッドレイアウト）

**用途**: カンバン、カードギャラリー、アイコン一覧、パレット

```
  ┌───┐ ┌───┐ ┌───┐
  │ A │ │ B │ │ C │
  └───┘ └───┘ └───┘
  ┌───┐ ┌───┐
  │ D │ │ E │
  └───┘ └───┘
```

### 6.1 基本設定

```typescript
const diagram = new go.Diagram(divId, {
  layout: new go.GridLayout({
    wrappingColumn: 3,              // 折り返す列数（NaN = 自動）
    wrappingWidth: NaN,             // 折り返す幅（px）
    cellSize: new go.Size(100, 80), // セルサイズ
    spacing: new go.Size(10, 10),   // セル間の余白
    alignment: go.GridAlignment.Location,
    arrangement: go.GridArrangement.LeftToRight,
    sorting: go.GridSorting.Ascending,
    comparer: (a: go.Part, b: go.Part) => {
      return (a.data.name ?? "").localeCompare(b.data.name ?? "");
    },
  }),
});
```

### 6.2 主要プロパティ

| プロパティ | デフォルト | 説明 |
|---|---|---|
| `wrappingColumn` | `NaN` | 折り返し列数（NaN = wrappingWidth で自動計算） |
| `wrappingWidth` | `NaN` | 折り返し幅 |
| `cellSize` | `Size(1,1)` | 各セルの最小サイズ |
| `spacing` | `Size(10,10)` | セル間の水平・垂直余白 |
| `alignment` | `Location` | `Location`（位置揃え）/ `Position`（座標揃え） |
| `arrangement` | `LeftToRight` | `LeftToRight` / `RightToLeft` |
| `sorting` | `Forwards` | `Forwards` / `Reverse` / `Ascending` / `Descending` |

## 7. レイアウト比較表

| レイアウト | 最適なデータ構造 | 計算量 | ノード数の目安 |
|---|---|---|---|
| **TreeLayout** | ツリー構造 | O(n) | 〜10,000 |
| **ForceDirectedLayout** | 任意のグラフ | O(n²)※ | 〜500 |
| **LayeredDigraphLayout** | 有向グラフ | O(n log n) | 〜5,000 |
| **CircularLayout** | 環状・完全グラフ | O(n log n) | 〜200 |
| **GridLayout** | リスト/コレクション | O(n log n) | 〜10,000 |

> ※ ForceDirectedLayout は反復シミュレーションのため大量ノードには不向き

### 選択フローチャート

```
データ構造は？
├── 親子関係あり → TreeLayout
├── フロー（方向性あり） → LayeredDigraphLayout
├── 関連性（方向なし） → ForceDirectedLayout
├── 環状 → CircularLayout
└── 単純なリスト → GridLayout
```

## 8. レイアウトの動的切替

```typescript
import * as go from "gojs";

// ━━━ レイアウト定義 ━━━
const layouts: Record<string, go.Layout> = {
  tree: new go.TreeLayout({ angle: 90, layerSpacing: 50 }),
  force: new go.ForceDirectedLayout({ maxIterations: 200 }),
  layered: new go.LayeredDigraphLayout({ direction: 90 }),
  circular: new go.CircularLayout({ radius: 150 }),
  grid: new go.GridLayout({ wrappingColumn: 4 }),
};

// ━━━ レイアウト切替関数 ━━━
function switchLayout(diagram: go.Diagram, layoutName: string): void {
  const layout = layouts[layoutName];
  if (!layout) {
    console.warn(`Unknown layout: ${layoutName}`);
    return;
  }

  diagram.startTransaction("change layout");
  diagram.layout = layout;
  diagram.layoutDiagram(true); // 即座に再計算
  diagram.commitTransaction("change layout");

  // 全体が見えるようにズーム調整
  diagram.zoomToFit();
}

// ━━━ UI から呼び出し ━━━
// <select onchange="switchLayout(diagram, this.value)">
//   <option value="tree">ツリー</option>
//   <option value="force">力学モデル</option>
//   <option value="layered">層状</option>
//   <option value="circular">円形</option>
//   <option value="grid">グリッド</option>
// </select>
```

### アニメーション付きの切替

```typescript
function switchLayoutAnimated(diagram: go.Diagram, layoutName: string): void {
  const layout = layouts[layoutName];
  if (!layout) return;

  // アニメーション有効化
  diagram.animationManager.isEnabled = true;

  diagram.startTransaction("change layout");
  diagram.layout = layout;
  diagram.commitTransaction("change layout");

  // ズーム調整もアニメーション
  diagram.commandHandler.zoomToFit();
}
```

## 9. グループ内レイアウト

グループ（サブグラフ）ごとに異なるレイアウトを適用できます。

```typescript
// ━━━ グループテンプレート ━━━
diagram.groupTemplate = new go.Group("Auto", {
  // グループ内部のレイアウトを個別指定
  layout: new go.TreeLayout({
    angle: 0,
    layerSpacing: 40,
  }),
}).add(
  new go.Shape("RoundedRectangle", {
    fill: "rgba(33, 150, 243, 0.1)",
    stroke: "#2196F3",
    strokeWidth: 1,
    parameter1: 10,
  }),
  new go.Panel("Vertical").add(
    new go.TextBlock({
      font: "bold 14px sans-serif",
      margin: new go.Margin(8, 8, 4, 8),
    }).bind("text"),
    new go.Placeholder({ padding: 10 })
  )
);

// ━━━ ダイアグラム全体は LayeredDigraph ━━━
diagram.layout = new go.LayeredDigraphLayout({
  direction: 90,
  layerSpacing: 80,
});

// ━━━ データ ━━━
diagram.model = new go.GraphLinksModel(
  [
    { key: "g1", text: "フロントエンド", isGroup: true },
    { key: "g2", text: "バックエンド",   isGroup: true },
    { key: 1, text: "React",     group: "g1" },
    { key: 2, text: "Next.js",   group: "g1" },
    { key: 3, text: "Express",   group: "g2" },
    { key: 4, text: "Prisma",    group: "g2" },
  ],
  [
    { from: 2, to: 3 },
    { from: 3, to: 4 },
  ]
);
```

## 10. レイアウトのパフォーマンス

### 10.1 大量ノード時の最適化

```typescript
// ━━━ 仮想化（Virtualization）を有効化 ━━━
const diagram = new go.Diagram(divId, {
  // 表示領域外のノードを描画しない
  "animationManager.isEnabled": false,

  // 初回レイアウト完了後にアニメーション再有効化
  "InitialLayoutCompleted": () => {
    diagram.animationManager.isEnabled = true;
  },
});

// ━━━ レイアウトの制御 ━━━
// 大量追加時はレイアウトを一時停止
diagram.layout.isOngoing = false; // 自動再計算を無効化

// バッチ処理
diagram.startTransaction("batch add");
for (let i = 0; i < 1000; i++) {
  diagram.model.addNodeData({ key: i, text: `Node ${i}` });
}
diagram.commitTransaction("batch add");

// 手動でレイアウト実行
diagram.layout.isOngoing = true;
diagram.layoutDiagram(true);
```

### 10.2 レイアウト種別ごとの最適化Tips

| レイアウト | 最適化ポイント |
|---|---|
| **TreeLayout** | `arrangement: FixedRoots` で不要な再配置を防ぐ |
| **ForceDirectedLayout** | `maxIterations` を減らす、`epsilonDistance` を大きくする |
| **LayeredDigraphLayout** | `iterations` を減らす（交差最小化の反復回数） |
| **CircularLayout** | `nodeDiameterFormula: Pythagorean` で高速計算 |
| **GridLayout** | `cellSize` を固定してセルサイズ計算をスキップ |
