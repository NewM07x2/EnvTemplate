## 1. GoJS とは

**GoJS** は、Northwoods Software 社が開発した **JavaScript / TypeScript 向けのダイアグラム描画ライブラリ** です。Web ブラウザ上でインタラクティブなダイアグラム（組織図、フローチャート、ネットワーク図、マインドマップ等）を構築できます。

| 項目 | 内容 |
|------|------|
| **正式名称** | GoJS |
| **開発元** | Northwoods Software（1998年設立） |
| **言語** | JavaScript / TypeScript |
| **描画方式** | HTML5 Canvas（SVG ではない） |
| **依存関係** | なし（ゼロデペンデンシ） |
| **最新バージョン** | 3.1.x（2025年10月リリース） |
| **npm パッケージ** | `gojs` |
| **GitHub** | [NorthwoodsSoftware/GoJS](https://github.com/NorthwoodsSoftware/GoJS) |
| **公式サイト** | [gojs.net](https://gojs.net/) |

### なぜ GoJS を使うのか

- **200以上のサンプル**が公式に提供されており、すぐに使い始められる
- **フレームワーク非依存**で React / Vue / Angular / Svelte のいずれでも使える
- **ドラッグ＆ドロップ、ズーム、Undo/Redo** などの操作が標準搭載
- **自動レイアウト**で複雑なグラフも美しく配置
- 企業向け（3M、Airbus、Amazon、NASA 等が採用）

## 2. 主な特徴

### 描画・表示

| 特徴 | 説明 |
|------|------|
| **Canvas ベース描画** | HTML5 Canvas を使用し、大量ノードでも高パフォーマンス |
| **仮想化** | 画面外のノードは描画しない（大規模グラフ対応） |
| **SVG / 画像エクスポート** | ダイアグラムを SVG や PNG として書き出し可能 |
| **印刷対応** | ダイアグラムの印刷 API を提供 |

### インタラクション

| 特徴 | 説明 |
|------|------|
| **ドラッグ＆ドロップ** | ノードの移動、リンクの接続が標準装備 |
| **Undo / Redo** | UndoManager を有効にするだけで使える |
| **キーボードショートカット** | Ctrl+C/V/Z、Delete 等がデフォルト対応 |
| **コンテキストメニュー** | 右クリック / ロングプレスメニュー |
| **ツールチップ** | ホバー時の情報表示 |
| **テキスト編集** | ノード内テキストのインライン編集 |
| **選択・マルチ選択** | クリック / 範囲選択 / Ctrl+クリック |

### データ管理

| 特徴 | 説明 |
|------|------|
| **Model-View アーキテクチャ** | データ（Model）と表示（Diagram）を分離 |
| **データバインディング** | モデルデータとノード表示の自動同期 |
| **JSON シリアライズ** | `model.toJson()` / `Model.fromJson()` で保存・復元 |
| **トランザクション** | 変更はトランザクション単位で管理 |

### レイアウト

| 特徴 | 説明 |
|------|------|
| **TreeLayout** | ツリー（組織図、ディレクトリ構造） |
| **ForceDirectedLayout** | 力学モデル（ネットワーク図） |
| **LayeredDigraphLayout** | 層状有向グラフ（フローチャート） |
| **CircularLayout** | 円形配置 |
| **GridLayout** | グリッド配置 |
| **カスタムレイアウト** | 独自レイアウトの実装が可能 |

## 3. 中核コンセプト

GoJS を理解するための核心的な概念です。

### 用語の整理

```
┌──────────────────────────────────────────────────────────┐
│                      Diagram（図）                        │
│                                                          │
│  ┌─────────────┐     Link     ┌─────────────┐           │
│  │    Node      │ ──────────→ │    Node      │           │
│  │  ┌────────┐  │             │  ┌────────┐  │           │
│  │  │ Panel  │  │             │  │ Panel  │  │           │
│  │  │┌──────┐│  │             │  │┌──────┐│  │           │
│  │  ││Shape ││  │             │  ││Shape ││  │           │
│  │  │└──────┘│  │             │  │└──────┘│  │           │
│  │  │┌──────┐│  │             │  │┌──────┐│  │           │
│  │  ││Text  ││  │             │  ││Text  ││  │           │
│  │  ││Block ││  │             │  ││Block ││  │           │
│  │  │└──────┘│  │             │  │└──────┘│  │           │
│  │  └────────┘  │             │  └────────┘  │           │
│  └─────────────┘             └─────────────┘           │
│                                                          │
│  Model（データ層）                                        │
│  ┌──────────────────────────────────────────┐            │
│  │ nodeDataArray: [{ key, text, ... }, ...]  │            │
│  │ linkDataArray: [{ from, to, ... }, ...]   │            │
│  └──────────────────────────────────────────┘            │
└──────────────────────────────────────────────────────────┘
```

| 概念 | 説明 | クラス |
|------|------|--------|
| **Diagram** | ダイアグラム全体。HTML div にホストされる | `go.Diagram` |
| **Model** | ノードとリンクのデータを保持 | `go.Model`, `go.GraphLinksModel`, `go.TreeModel` |
| **Node** | グラフの頂点（ノード） | `go.Node` |
| **Link** | ノード間を結ぶ線 | `go.Link` |
| **Group** | ノードをグルーピングするコンテナ | `go.Group` |
| **Panel** | 複数の GraphObject を配置するコンテナ | `go.Panel` |
| **Shape** | 図形（矩形、円、カスタム形状） | `go.Shape` |
| **TextBlock** | テキスト表示 | `go.TextBlock` |
| **Picture** | 画像表示 | `go.Picture` |
| **Part** | Node / Link / Group の基底クラス | `go.Part` |
| **GraphObject** | 全ビジュアル要素の基底クラス | `go.GraphObject` |

### GraphObject の継承階層

```
GraphObject（抽象基底）
├── Shape         … 図形を描画
├── TextBlock     … テキストを描画
├── Picture       … 画像を描画
├── Placeholder   … Group 内のメンバー表示領域
└── Panel         … 子要素を配置するコンテナ
    ├── Part      … Diagram 上に配置可能なパネル
    │   ├── Node  … グラフのノード
    │   │   └── Group … ノードのグループ
    │   ├── Link  … ノード間の接続線
    │   └── Adornment … 選択ハンドル等の装飾
    └── (その他のパネル)
```

## 4. アーキテクチャ（Model-View）

GoJS は **Model-View アーキテクチャ** を採用しています。

```
┌─────────────┐                    ┌─────────────────┐
│   Model      │  ← データ保存 →   │  JSON / DB      │
│ (データ層)   │                    └─────────────────┘
│              │
│ nodeDataArray│    データバインディング
│ linkDataArray│  ─────────────────→  ┌─────────────┐
│              │                      │   Diagram    │
└─────────────┘                      │ (表示層)     │
                                     │              │
      変更を通知                      │  Node        │
   ◄──────────────────────────────── │  Link        │
      ユーザー操作                    │  Group       │
                                     └─────────────┘
```

| レイヤ | 役割 | 注意点 |
|--------|------|--------|
| **Model** | 純粋な JavaScript オブジェクトの配列を保持 | 独自プロパティを自由に追加できる |
| **Diagram** | Model のデータを視覚化する | テンプレートに基づいてノード/リンクを描画 |
| **Template** | Model データ → 見た目を定義するマッピング | `nodeTemplate`, `linkTemplate` |
| **Binding** | Model のプロパティ → GraphObject のプロパティ | `new go.Binding("text", "name")` |

### 3 種類の Model

| Model | 用途 | リンクの定義方法 |
|-------|------|-----------------|
| `go.Model` | リンクなし（ノードのみ） | — |
| `go.GraphLinksModel` | 汎用（任意のリンク） | `linkDataArray` に `{ from, to }` |
| `go.TreeModel` | ツリー構造 | ノードデータに `parent` プロパティ |

## 6. 対応フレームワーク

GoJS はゼロデペンデンシですが、主要フレームワークとの統合がサポートされています。

| フレームワーク | 統合方法 | パッケージ |
|---------------|---------|-----------|
| **React** | `gojs-react` コンポーネントライブラリ | `gojs-react` |
| **Angular** | `gojs-angular` コンポーネントライブラリ | `gojs-angular` |
| **Vue 3** | 公式サンプル | — （直接統合） |
| **Svelte** | div + onMount で初期化 | — （直接統合） |
| **Next.js** | `gojs-react` + dynamic import（SSR回避） | `gojs-react` |
| **Nuxt** | ClientOnly + onMounted で初期化 | — |
| **Electron** | そのまま使用可能 | — |
| **Node.js** | サーバーサイドレンダリング（限定的） | `gojs` |

### React での使用例

```bash
npm install gojs gojs-react
```

```tsx
import { ReactDiagram } from 'gojs-react'
import * as go from 'gojs'

function initDiagram(): go.Diagram {
  const diagram = new go.Diagram({
    'undoManager.isEnabled': true,
    layout: new go.TreeLayout({ angle: 90 }),
  })
  // テンプレート定義...
  return diagram
}

export function MyDiagram() {
  return (
    <ReactDiagram
      initDiagram={initDiagram}
      divClassName="diagram-component"
      nodeDataArray={[
        { key: 1, text: 'Alpha' },
        { key: 2, text: 'Beta', parent: 1 },
      ]}
    />
  )
}
```

## 7. ライセンス体系

### ライセンスの種類

| ライセンス | 用途 | 費用 |
|-----------|------|------|
| **評価版** | 開発・検証（ウォーターマーク付き） | 無料 |
| **開発ライセンス** | 商用開発 | 有料（開発者単位） |
| **配布ライセンス** | 本番環境への配布 | 有料 |

### 重要なポイント

- **評価版（無料）** はウォーターマーク（透かし）が表示される
- 評価版のまま開発・検証は自由に行える
- **本番リリース時にライセンスキーが必要**
- ライセンスキーは `Diagram` の初期化時に設定する

```typescript
// ライセンスキーの設定
;(go as any).licenseKey = "your-license-key-here"

const diagram = new go.Diagram("myDiagramDiv")
```

## 8. 競合ライブラリとの比較

| 特徴 | GoJS | D3.js | Mermaid | React Flow | JointJS | Cytoscape.js |
|------|------|-------|---------|------------|---------|-------------|
| **用途** | ダイアグラム全般 | データ可視化全般 | テキストベース図 | ノードベースUI | ダイアグラム | グラフ理論 |
| **描画方式** | Canvas | SVG/Canvas | SVG | SVG (React) | SVG | Canvas |
| **インタラクション** | ◎ 豊富 | △ 要自作 | ✕ なし | ○ 良い | ○ 良い | ○ 良い |
| **自動レイアウト** | ◎ 6種類+ | △ 要自作 | ◎ 自動 | △ 限定的 | ○ プラグイン | ◎ 豊富 |
| **学習コスト** | 中 | 高 | 低 | 低 | 中 | 中 |
| **TypeScript** | ◎ 公式対応 | ○ | ○ | ◎ | ○ | ○ |
| **ライセンス** | 商用（有料） | BSD | MIT | MIT | MPL 2.0 | MIT |
| **サンプル数** | 200+ | 多数 | — | 多数 | 多数 | 多数 |
| **大規模データ** | ◎ 仮想化あり | △ | ✕ | ○ | ○ | ○ |

### GoJS を選ぶべきケース

| ケース | 理由 |
|-------|------|
| **業務用ダイアグラムエディタ** | 豊富なインタラクション、Undo/Redo、JSON保存 |
| **組織図・ツリー構造** | TreeLayout が強力 |
| **フローチャートエディタ** | リンク描画、ドラッグ＆ドロップ、レイアウト |
| **大量ノードの描画** | Canvas 仮想化でパフォーマンスが良い |
| **エンタープライズ品質が必要** | 25年以上の実績、手厚いサポート |

### GoJS を避けるべきケース

| ケース | 代替 |
|-------|------|
| データ可視化（チャート・グラフ） | D3.js / Chart.js |
| ドキュメント内の静的な図 | Mermaid |
| シンプルなノードUI | React Flow |
| グラフ理論の計算 | Cytoscape.js |
| 無料ライセンスが必須 | React Flow / Cytoscape.js |
