## 1. ライセンスキーの設定

### 1.1 ライセンス体系

| ライセンス | 用途 | ウォーターマーク | 費用 |
|---|---|---|---|
| 評価版（デフォルト） | 開発・検証 | あり（右上に "GoJS Evaluation" 表示） | 無料 |
| 開発ライセンス | 本番デプロイ | なし | 有料（[公式](https://nwoods.com/sales/index.html)参照） |

### 1.2 ライセンスキーの適用

```typescript
// ━━━ アプリケーションの初期化時に設定 ━━━
import * as go from "gojs";

// ライセンスキーを設定（Diagram 作成前に実行）
(go.Diagram as any).licenseKey = "your-license-key-here";

// その後、通常通り Diagram を作成
const diagram = new go.Diagram("myDiagramDiv");
```

### 1.3 ライセンスキーの管理方法

```typescript
// ❌ NG: ソースコードにハードコード
(go.Diagram as any).licenseKey = "AAAA-BBBB-CCCC-DDDD";

// ✅ OK: 環境変数から読み取り
(go.Diagram as any).licenseKey = process.env.GOJS_LICENSE_KEY;

// ✅ OK: 設定ファイルから読み取り（.env に記載）
// .env
// VITE_GOJS_LICENSE_KEY=your-license-key-here
(go.Diagram as any).licenseKey = import.meta.env.VITE_GOJS_LICENSE_KEY;
```

## 4. 大規模ダイアグラムの対応

### 4.1 仮想化（Virtualization）

表示領域外のノードを描画しないことでパフォーマンスを向上させます。

```typescript
const diagram = new go.Diagram(divId, {
  // 仮想化を有効にする設定
  "animationManager.isEnabled": false,
  scrollMode: go.ScrollMode.InfiniteScroll,

  // 固定境界を使用
  fixedBounds: new go.Rect(0, 0, 10000, 10000),
});

// ━━━ 仮想化対応のモデル ━━━
// 大量データはページネーションで読み込み
async function loadLargeData(diagram: go.Diagram): Promise<void> {
  const pageSize = 100;
  let page = 0;

  while (true) {
    const response = await fetch(`/api/nodes?page=${page}&size=${pageSize}`);
    const { nodes, hasMore } = await response.json();

    diagram.startTransaction("load page");
    for (const node of nodes) {
      diagram.model.addNodeData(node);
    }
    diagram.commitTransaction("load page");

    if (!hasMore) break;
    page++;
  }
}
```

### 4.2 ノード数の目安

| ノード数 | 推奨レイアウト | 注意事項 |
|---|---|---|
| 〜100 | 全レイアウト OK | 問題なし |
| 100〜500 | TreeLayout / LayeredDigraph | ForceDirected は遅くなる可能性 |
| 500〜1,000 | TreeLayout | アニメーション無効化推奨 |
| 1,000〜5,000 | TreeLayout / Grid | 仮想化を検討 |
| 5,000〜 | 仮想化 + ページネーション | 全件表示は非推奨 |

### 4.3 概要＋詳細パターン

```typescript
// 全体概要（グルーピング表示）→ ダブルクリック → 詳細表示
diagram.nodeTemplate = new go.Node("Auto", {
  doubleClick: (e, node) => {
    const n = node as go.Node;
    if (n.data.isGroup) {
      // 詳細ダイアグラムに遷移
      loadDetailView(n.data.groupId);
    }
  },
}).add(/* ... */);
```

## 5. SVG / 画像エクスポート

### 5.1 SVG エクスポート

```typescript
function exportToSVG(diagram: go.Diagram): string {
  const svg = diagram.makeSvg({
    scale: 1,
    background: "white",
    // 特定領域のみエクスポート
    // position: new go.Point(0, 0),
    // size: new go.Size(800, 600),
  });

  if (!svg) return "";
  return new XMLSerializer().serializeToString(svg);
}

// SVG ファイルとしてダウンロード
function downloadSVG(diagram: go.Diagram, filename = "diagram.svg"): void {
  const svgString = exportToSVG(diagram);
  const blob = new Blob([svgString], { type: "image/svg+xml" });
  const url = URL.createObjectURL(blob);

  const a = document.createElement("a");
  a.href = url;
  a.download = filename;
  a.click();

  URL.revokeObjectURL(url);
}
```

### 5.2 PNG エクスポート

```typescript
function downloadPNG(
  diagram: go.Diagram,
  filename = "diagram.png",

    background: "white",
```

## 6. 印刷対応

```typescript
function printDiagram(diagram: go.Diagram): void {
  // 印刷用ウィンドウを開く
  const printWindow = window.open("", "_blank");
  if (!printWindow) return;

  // SVG を生成
  const svg = diagram.makeSvg({
    scale: 1,
    background: "white",
  });

  if (!svg) return;

  // 印刷用 HTML を構築
  printWindow.document.write(`
    <!DOCTYPE html>
    <html>
    <head>
      <title>ダイアグラム印刷</title>
      <style>
        @media print {
          body { margin: 0; }
          svg { width: 100%; height: auto; }
        }
        @page {
          size: A4 landscape;
          margin: 10mm;
        }
      </style>
    </head>
    <body>
      ${new XMLSerializer().serializeToString(svg)}
    </body>
    </html>
  `);

  printWindow.document.close();
  printWindow.focus();

  // 少し遅延してから印刷ダイアログを開く
  setTimeout(() => {
    printWindow.print();
    printWindow.close();
  }, 500);
}
```

### ページ分割印刷

```typescript
function printMultiPage(diagram: go.Diagram): void {
  const bounds = diagram.documentBounds;
  const pageWidth = 800;
  const pageHeight = 600;
  const pages: SVGElement[] = [];

  for (let y = bounds.y; y < bounds.y + bounds.height; y += pageHeight) {
    for (let x = bounds.x; x < bounds.x + bounds.width; x += pageWidth) {
      const svg = diagram.makeSvg({
        position: new go.Point(x, y),
        size: new go.Size(pageWidth, pageHeight),
        background: "white",
      });
      if (svg) pages.push(svg);
    }
  }

  // pages 配列を使って複数ページを印刷
  console.log(`${pages.length} ページを生成`);
}
```

## 9. テスト戦略

### 9.1 ユニットテスト（モデルのテスト）

```typescript
// model.test.ts
import * as go from "gojs";
import { describe, it, expect } from "vitest";

describe("Diagram Model", () => {
  it("ノードを追加できる", () => {
    const model = new go.GraphLinksModel();
    model.addNodeData({ key: 1, text: "Node 1" });
    expect(model.nodeDataArray).toHaveLength(1);
    expect(model.findNodeDataForKey(1)).toBeDefined();
  });

  it("リンクを追加できる", () => {
    const model = new go.GraphLinksModel(
      [{ key: 1 }, { key: 2 }],
      []
    );
    model.addLinkData({ from: 1, to: 2 });
    expect(model.linkDataArray).toHaveLength(1);
  });

  it("JSON にシリアライズ・復元できる", () => {
    const model = new go.GraphLinksModel(
      [{ key: 1, text: "A" }, { key: 2, text: "B" }],
      [{ from: 1, to: 2 }]
    );

    const json = model.toJson();
    const restored = go.Model.fromJson(json) as go.GraphLinksModel;

    expect(restored.nodeDataArray).toHaveLength(2);
    expect(restored.linkDataArray).toHaveLength(1);
  });

  it("Undo/Redo が動作する", () => {
    const model = new go.GraphLinksModel();
    model.undoManager.isEnabled = true;

    model.startTransaction("add");
    model.addNodeData({ key: 1, text: "Node 1" });
    model.commitTransaction("add");

    expect(model.nodeDataArray).toHaveLength(1);

    model.undoManager.undo();
    expect(model.nodeDataArray).toHaveLength(0);

    model.undoManager.redo();
    expect(model.nodeDataArray).toHaveLength(1);
  });
});
```

## 11. トラブルシューティング

### よくある問題と解決策

| 問題 | 原因 | 解決策 |
|---|---|---|
| ダイアグラムが表示されない | div のサイズが 0 | CSS で `width` と `height` を明示的に指定 |
| "GoJS Evaluation" が消えない | ライセンスキー未設定 | `Diagram.licenseKey` を設定 |
| ノードが全部左上に重なる | レイアウト未指定 + 位置データなし | `layout` を設定するか `loc` プロパティを指定 |
| 双方向バインドが動かない | `makeTwoWay()` 忘れ | `new go.Binding("text").makeTwoWay()` |
| Undo が動かない | UndoManager 無効 | `"undoManager.isEnabled": true` |
| 変更がすぐ反映されない | トランザクション未使用 | `startTransaction` / `commitTransaction` で囲む |
| パフォーマンスが悪い | アニメーション + 大量ノード | `animationManager.isEnabled = false` |
| React で再レンダリングされない | Model の参照が同じ | 新しい Model インスタンスを作成 |
| SSR でエラーが出る | Canvas API が未定義 | `ssr: false` または `typeof window` チェック |
| テキスト編集で日本語入力が途切れる | IME 非対応 | GoJS 3.x で改善済み。古いバージョンの場合はアップデート |

### デバッグのコツ


## 12. セキュリティ考慮事項

### 12.1 XSS 対策

```typescript
// GoJS はデフォルトでテキストをエスケープしますが、
// ユーザー入力をそのまま model に渡す場合は注意

// ❌ NG: HTML をそのまま表示する仕組みはないが念のため
function sanitizeInput(input: string): string {
  return input
    .replace(/&/g, "&amp;")
    .replace(/</g, "&lt;")
    .replace(/>/g, "&gt;")
    .replace(/"/g, "&quot;")
    .replace(/'/g, "&#x27;");
}

// ✅ OK: バリデーション付きで model に追加
function addUserNode(diagram: go.Diagram, userInput: string): void {
  const sanitized = sanitizeInput(userInput.trim());
  if (sanitized.length === 0 || sanitized.length > 200) return;

  diagram.startTransaction("add");
  diagram.model.addNodeData({
    key: Date.now(),
    text: sanitized,
  });
  diagram.commitTransaction("add");
}
```

### 12.2 JSON 復元時のバリデーション

```typescript
// サーバーから受け取った JSON を検証してから復元
function safeLoadModel(jsonString: string): go.Model | null {
  try {
    const parsed = JSON.parse(jsonString);

    // 基本構造のバリデーション
    if (!parsed.class || !parsed.nodeDataArray) {
      console.error("Invalid model JSON structure");
      return null;
    }

    // ノード数の制限
    if (parsed.nodeDataArray.length > 10000) {
      console.error("Too many nodes");
      return null;
    }

    return go.Model.fromJson(jsonString);
  } catch (e) {
    console.error("Failed to parse model JSON:", e);
    return null;
  }
}
```

### 12.3 ライセンスキーの保護

- ライセンスキーをフロントエンドに埋め込む必要がありますが、以下の点に注意：
  - **`.env` ファイル** で管理し、Git にコミットしない
  - **ビルド時に注入**する（`import.meta.env` や `process.env`）
  - ライセンスキーはドメイン制限付きのため、第三者のサイトでは使用不可
